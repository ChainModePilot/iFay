# 4. Canales de comunicación

La comunidad iFay proporciona múltiples canales de comunicación para satisfacer las necesidades de colaboración en diferentes escenarios.

## GitHub Discussions

GitHub Discussions es la plataforma principal de discusión asíncrona de la comunidad iFay, adecuada para:

- Discusión pública de SEPs (Specification Enhancement Proposals)
- Exploración en profundidad de cuestiones técnicas
- Solicitudes de funcionalidades y discusiones de diseño
- Comunicación diaria dentro de los grupos de trabajo

Recomendamos que las discusiones técnicas importantes se realicen en GitHub Discussions para garantizar que el contenido de las discusiones sea rastreable y buscable.

## GitHub Issues

GitHub Issues se utiliza para el seguimiento de tareas específicas y la gestión de problemas:

- Informes de errores
- Tareas de desarrollo específicas
- Sugerencias de mejora de documentación
- Problemas con casos de prueba de iFACTS

Al enviar un Issue, proporcione pasos de reproducción detallados o información de contexto para ayudar a los mantenedores a localizar y abordar el problema rápidamente.

## Discord

Discord es la plataforma de comunicación en tiempo real de la comunidad iFay. Cada grupo de trabajo tiene su propio canal dedicado:

- `#general`: Discusión general
- `#wg-fayid`: Grupo de trabajo FayID
- `#wg-ego`: Grupo de trabajo Ego
- `#wg-protocols`: Grupo de trabajo de protocolos
- `#wg-fayger`: Grupo de trabajo FayGer
- `#wg-ifacts`: Grupo de trabajo iFACTS
- `#wg-docs`: Grupo de trabajo de documentación
- `#wg-community`: Grupo de trabajo de la comunidad

Discord es adecuado para intercambios rápidos y discusiones informales. Sin embargo, tenga en cuenta que las decisiones técnicas importantes deben registrarse en GitHub para garantizar la transparencia y la trazabilidad.

## Reuniones regulares

- **Reunión mensual de Core Maintainers**: Se celebra una vez al mes, se realiza públicamente y está abierta a la observación de todos los miembros de la comunidad. Las agendas y actas de las reuniones se publican en GitHub.
- **Reuniones de grupos de trabajo**: Cada grupo de trabajo puede establecer su propia frecuencia y formato de reuniones, con la recomendación de al menos una reunión de sincronización por mes.

## Lista de correo

La lista de correo se utiliza para anuncios y notificaciones importantes:

- Notificaciones de lanzamiento de nuevas versiones
- Notificaciones de cambio de estado de SEP
- Anuncios de decisiones importantes de gobernanza
- Notificaciones de eventos comunitarios

Suscribirse a la lista de correo garantiza que no se perderá información importante.

## Código de conducta

Todos los canales de comunicación siguen el Código de Conducta de iFay. Nos comprometemos a fomentar un entorno comunitario amigable, inclusivo y profesional. Mantenga el respeto y la constructividad en todas las interacciones.

## Idioma

El inglés es el idioma de trabajo principal de la comunidad iFay, pero damos la bienvenida a la participación en otros idiomas. Si se siente más cómodo usando español, chino, alemán u otro idioma, no dude en hacerlo — los miembros de la comunidad ayudarán con la traducción para garantizar que sus ideas se transmitan con precisión.

Nuestro objetivo es garantizar que el idioma no sea una barrera para participar en la comunidad iFay.
