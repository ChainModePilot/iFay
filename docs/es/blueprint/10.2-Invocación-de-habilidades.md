# 10.2 Invocación de habilidades

## Definición en una línea

La Invocación de habilidades es la **boca y teléfono** de iFay — si la Operación simulada es iFay usando sus manos para operar interfaces, la Invocación de habilidades es iFay "llamando" directamente a un servicio — sin interfaz necesaria, llamando directamente a APIs o activando tareas.

## Por qué es necesario

Imagina que necesitas traducir un documento. Tienes dos opciones:

**Opción 1**: Abrir un navegador, ir a un sitio web de traducción, pegar el texto, esperar la traducción, copiar el resultado. Este es el enfoque de "Operación simulada" — operando a través de la interfaz paso a paso.

**Opción 2**: Llamar directamente al teléfono de la empresa de traducción, decir "traduce este pasaje por mí", y te dicen el resultado directamente. Sin sitio web que abrir, sin interfaz que operar. Este es el enfoque de "Invocación de habilidades" — **llamar directamente al servicio**.

¿Por qué se necesita la segunda opción? Porque en muchos casos, operar una interfaz es un intermediario innecesario. Lo que realmente quieres no es "abrir un sitio web de traducción" sino "traducir este pasaje al inglés". La Invocación de habilidades permite a iFay saltarse la interfaz y lograr directamente tu objetivo.

## Su posición en la arquitectura

```
Arquitectura de cuatro capas de iFay
├── Capa Social
├── Capa de Interacción          ← La Invocación de habilidades está aquí
│   ├── Percepción (Sentido)
│   │   └── ...
│   └── Acción
│       ├── Operación simulada   ← Manos: opera interfaces
│       ├── 👉 Invocación de habilidades  ← Boca: "llama" directamente a servicios
│       └── Comportamiento autónomo       ← Voluntad: decide cuándo actuar
├── Capa de Cognición
│   └── Habilidades
│       └── Habilidades registradas  ← La Invocación solo puede llamar habilidades registradas
└── Capa Ego
```

La Invocación de habilidades se encuentra en el **subsistema de Acción de la Capa de Interacción**, junto con la Operación simulada y el Comportamiento autónomo. Su prerrequisito es que la habilidad que se llama debe estar ya registrada — las habilidades no registradas no pueden ser invocadas.

## Cómo funciona

El flujo de trabajo de la Invocación de habilidades sigue una cadena clara: **Intención → Coincidencia → Llamada → Registro**.

**1. Intención** — iFay recibe una intención de tarea (de la instrucción del Human Prime, la activación del Comportamiento autónomo o la solicitud de otro módulo).

**2. Coincidencia** — iFay compara la intención contra el inventario de habilidades registradas para encontrar la habilidad más adecuada para llamar.

**3. Llamada** — iFay invoca la habilidad coincidente. Dado que la habilidad fue preautorizada durante el registro, la llamada se ejecuta inmediatamente con cero retraso.

**4. Registro** — La invocación y su resultado se registran. Si la GMChain del ecosistema iFay está activa, la contribución también se registra para liquidación con MeriToken.

## Relaciones con otros módulos

| Módulo relacionado | Relación | Analogía con el cuerpo humano |
|-------------------|----------|-------------------------------|
| **Habilidades registradas** | La Invocación solo puede llamar habilidades registradas; el registro es el prerrequisito para la invocación | Debes tener licencia antes de ejercer |
| **Operación simulada** | Ambos en el subsistema de Acción, pero enfoques diferentes: la Invocación llama directamente, la Operación simulada opera a través de interfaces | Hacer una llamada telefónica vs. ir en persona |
| **Comportamiento autónomo** | El Comportamiento autónomo decide "cuándo llamar", la Invocación maneja la llamada real | La voluntad decide → la boca habla |
| **Gestión de Credenciales** | La Invocación necesita credenciales para autenticación durante la ejecución | Necesitas mostrar identificación al hacer trámites |
| **GMChain** | Las invocaciones de habilidades pueden registrarse en GMChain como contribuciones | Registros de trabajo para evaluación de desempeño |

## Historias de escenarios

### Escenario 1: Traducción instantánea sin abrir ninguna app

Recibes un correo en japonés. Le dices a iFay: "Tradúceme este correo."

iFay no abre un navegador ni navega a un sitio web de traducción. En cambio, invoca directamente la habilidad de API de traducción registrada — la que fue preautorizada durante el registro. Dos segundos después, el resultado de la traducción se te presenta. Sin ventanas emergentes pidiendo contraseñas, sin ruedas de autenticación girando — todo fue preparado durante el registro.

### Escenario 2: Orquestación multi-habilidad para una tarea compleja

Dices: "Ayúdame a preparar la reunión con el cliente de mañana."

iFay orquesta múltiples invocaciones de habilidades:
1. Invoca la API de calendario para recuperar los detalles de la reunión de mañana
2. Invoca la API de correo para encontrar correspondencia reciente con este cliente
3. Invoca un flujo de trabajo de generación de documentos para compilar un resumen de la reunión
4. Invoca la API de servicios de oficina para reservar la sala de reuniones y pedir refrigerios

Cada invocación es directa e inmediata — sin interfaces que navegar, sin formularios que llenar. Toda la preparación se completa en menos de un minuto.

## Para desarrolladores

La Invocación de habilidades pertenece a la **Fase 2 (Toma de control directo del cliente)** como módulo central.

- **ID de requisito**: Requisito 10 (Invocación de habilidades)
- **Especificación de interfaz**: Interfaz `InvokeSkillService` con métodos centrales: `invoke()` (invocar habilidad), `matchIntent()` (coincidir intención con habilidad) y `getInvocationLog()` (log de invocación)
- **Prerrequisito**: Las habilidades deben registrarse vía `RegisteredSkillManager` antes de poder ser invocadas
- **Módulos relacionados**: Habilidades registradas (`RegisteredSkillManager`) proporciona el inventario de habilidades; Gestión de Credenciales (`CredentialManager`) proporciona autenticación
- **Pruebas de conformidad**: iFACTS L1 verifica la coincidencia de intención y capacidad de invocación de habilidades; L2 verifica la integración de interfaz con el módulo de Habilidades registradas; L3 verifica la cadena completa "expresión de intención → invocación de habilidad → registro de contribución"
