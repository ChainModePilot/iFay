# 4. 커뮤니케이션 채널

iFay 커뮤니티는 다양한 시나리오에서의 협업 요구를 충족하기 위해 여러 커뮤니케이션 채널을 제공합니다.

## GitHub Discussions

GitHub Discussions는 iFay 커뮤니티의 주요 비동기 토론 플랫폼으로, 다음에 적합합니다:

- SEP(사양 개선 제안)의 공개 토론
- 기술적 질문에 대한 심층 탐구
- 기능 요청 및 설계 토론
- 워킹 그룹 내 일상적인 커뮤니케이션

중요한 기술적 토론은 GitHub Discussions에서 진행하시기를 권장합니다. 이를 통해 토론 내용의 추적 및 검색이 가능해집니다.

## GitHub Issues

GitHub Issues는 구체적인 작업 추적 및 이슈 관리에 사용됩니다:

- 버그 보고
- 구체적인 개발 작업
- 문서 개선 제안
- iFACTS 테스트 케이스 이슈

Issue를 제출할 때는 메인테이너가 문제를 신속하게 파악하고 처리할 수 있도록 상세한 재현 단계 또는 컨텍스트 정보를 제공해 주십시오.

## Discord

Discord는 iFay 커뮤니티의 실시간 커뮤니케이션 플랫폼입니다. 각 워킹 그룹에는 전용 채널이 있습니다:

- `#general`: 일반 토론
- `#wg-fayid`: FayID 워킹 그룹
- `#wg-ego`: Ego 워킹 그룹
- `#wg-protocols`: 프로토콜 워킹 그룹
- `#wg-fayger`: FayGer 워킹 그룹
- `#wg-ifacts`: iFACTS 워킹 그룹
- `#wg-docs`: 문서 워킹 그룹
- `#wg-community`: 커뮤니티 워킹 그룹

Discord는 빠른 교류와 비공식적인 토론에 적합합니다. 다만, 중요한 기술적 결정은 투명성과 추적 가능성을 보장하기 위해 GitHub에 기록해야 한다는 점에 유의해 주십시오.

## 정기 회의

- **코어 메인테이너 월례 회의**: 월 1회 개최되며, 공개적으로 진행되어 모든 커뮤니티 구성원이 참관할 수 있습니다. 회의 안건과 회의록은 GitHub에 게시됩니다.
- **워킹 그룹 회의**: 각 워킹 그룹은 자체적으로 회의 빈도와 형식을 정할 수 있으며, 최소 월 1회 동기화 회의를 권장합니다.

## 메일링 리스트

메일링 리스트는 중요한 공지 및 알림에 사용됩니다:

- 새 버전 릴리스 알림
- SEP 상태 변경 알림
- 주요 거버넌스 결정 공지
- 커뮤니티 이벤트 알림

메일링 리스트를 구독하시면 중요한 정보를 놓치지 않을 수 있습니다.

## 행동 강령

모든 커뮤니케이션 채널은 iFay 행동 강령을 따릅니다. 우리는 친근하고, 포용적이며, 전문적인 커뮤니티 환경을 조성하기 위해 노력하고 있습니다. 모든 상호작용에서 존중과 건설적인 태도를 유지해 주십시오.

## 언어

영어는 iFay 커뮤니티의 주요 작업 언어이지만, 다른 언어로의 참여도 환영합니다. 한국어, 중국어, 독일어 또는 다른 언어가 더 편하시다면 자유롭게 사용해 주십시오. 커뮤니티 구성원이 번역을 지원하여 여러분의 아이디어가 정확하게 전달되도록 하겠습니다.

우리의 목표는 언어가 iFay 커뮤니티 참여의 장벽이 되지 않도록 하는 것입니다.
