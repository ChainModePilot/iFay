# 6. iFay는 인간 인격의 인스턴시에이트(복제)

iFay의 개성은 무에서 생성되는 것이 아닙니다—휴먼 프라임(Human Prime, 줄여서 Prime)으로부터 인스턴시에이트(복제)됩니다. 모든 iFay는 휴먼 프라임(Human Prime)의 인격 확장이며, 이 확장에는 구조화된 주입 과정이 필요합니다.

이 페이지는 휴먼 프라임(Human Prime)이 세 가지 차원에서 자신의 특성을 iFay에 주입하는 방법을 설명합니다: **성격**이 Ego 모델을 형성하고, **데이터**가 개인 데이터 힙을 채우고, **권한**이 크리덴셜 위임을 수립합니다. 세 차원은 하나도 빠질 수 없습니다—성격은 iFay에게 "나는 누구인가"를 알게 하고, 데이터는 iFay에게 "나의 과거"를 부여하며, 권한은 iFay가 "나를 대신하여 행동"할 수 있게 합니다.

---

## 휴먼 프라임(Human Prime) 성격(→ Ego 모델)

휴먼 프라임(Human Prime)의 성격은 Ego 모델의 훈련 및 설정 입력입니다. Ego 모델은 iFay의 내장 마이크로 모델로, 가치 지향, 관심 선호, 습관, 인지 경계, 스킬 경계, 권한 경계, 작업 스타일 등의 차원에서 iFay가 휴먼 프라임(Human Prime)과 정렬을 유지하도록 제약합니다.

### 세 가지 주입 방식

| 방식 | 설명 |
|------|------|
| **대화 서술** | 휴먼 프라임(Human Prime)이 iFay와 가이드식 대화를 진행하며, iFay는 표준 연구 설문을 통해 대화 주제 범위를 제약하여 유효한 성격 데이터를 수집합니다. 이것은 잡담이 아닙니다—매 라운드의 대화가 특정 성격 차원을 중심으로 전개됩니다. |
| **설문 작성** | 휴먼 프라임(Human Prime)이 구조화된 성격 설문을 직접 작성하고, 시스템이 답변을 Ego 모델의 훈련 파라미터로 변환합니다. |
| **직접 설정** | 고급 사용자가 Ego 모델의 제약 파라미터를 직접 편집하여 iFay의 행동 경계를 정밀하게 제어할 수 있습니다. |

### 다중 버전 Ego

사람은 다면적입니다. 업무에서는 엄격하고 집중하며, 친구 앞에서는 가볍고 유머러스하고, 가족 곁에서는 부드럽고 인내심 있습니다. Ego 모델은 여러 버전을 지원하며, 각 버전은 휴먼 프라임(Human Prime) 인격의 한 측면에 대응합니다. 어느 시점에서든 하나의 Ego 버전만 활성 상태이며, 수동 전환과 컨텍스트 기반 자동 전환을 지원합니다.

### 시나리오

> Isabel이 방금 자신의 iFay를 활성화했습니다. 시스템이 그녀를 대화로 안내합니다—임의의 잡담이 아니라 표준 연구 설문을 기반으로 설계된 가이드식 교류입니다.
>
> "표현이 모호한 업무 이메일을 받았을 때, 직접 답장하여 명확히 하는 쪽인가요, 아니면 먼저 비공식적으로 상대방을 찾아가는 쪽인가요?"
>
> Isabel은 후자를 선택하고, 비공식적 상황에서 잠재적 오해를 해결하는 것을 더 선호한다고 보충합니다. iFay는 이 선호를 기록하고, Ego 모델의 "소통 스타일" 차원의 파라미터로 인코딩합니다.
>
> 20분 후, Isabel의 iFay는 그녀의 소통 선호, 의사결정 스타일, 위험 태도를 초보적으로 이해했습니다. 이러한 특성은 iFay가 일상 업무를 대신 처리할 때의 행동 준칙이 됩니다.

---

## 휴먼 프라임(Human Prime) 데이터(→ 개인 데이터 힙)

데이터는 iFay의 "기억"입니다. 데이터가 없는 iFay는 기억상실증에 걸린 사람과 같습니다—자신이 누구인지는 알지만, 무엇을 경험했는지 기억하지 못합니다. 개인 데이터 힙은 iFay의 모든 프라이빗 데이터를 통합 관리하는 컴포넌트이며, 휴먼 프라임(Human Prime) 데이터는 그 초기화 입력입니다.

### 네 가지 데이터 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| **콘텐츠(Content)** | 구조화 또는 비구조화 콘텐츠 자산 | 사진, 동영상, 글, 창작물 |
| **데이터(Data)** | 개인 데이터 | 앱 사용 기록, 건강 데이터, 소비 기록 |
| **지식 베이스(Knowledge Base)** | 조직화된 지식 체계 | 전문 노트, 학습 자료, 업무 문서 |
| **정보 피드(Info Feed)** | 실시간 정보 구독 | 뉴스 소스, 업계 동향, 소셜 미디어 팔로우 |

### 세 가지 주입 방식

1. **휴먼 프라임(Human Prime) 데이터 인가**: iFay가 휴먼 프라임(Human Prime)의 사진, 앱 데이터, 통신 기록 등에 접근하도록 인가합니다. 시스템은 OS 레이어와 긴밀하게 통합되어, 시스템 감독 하에 시스템 정보와 다른 앱의 프라이빗 데이터를 읽는 것을 지원합니다.
2. **파일 가져오기**: 문서, 노트, 채팅 기록 등 장기 기억 파일을 가져옵니다.
3. **정보 소스 지정**: 도서, 정보 소스, 지식 베이스를 지정하여 iFay가 이러한 소스에서 지속적으로 정보를 획득하게 합니다.

모든 데이터는 표준 형식으로 개인 데이터 힙에 저장되며, iFay만 직접 접근할 수 있습니다. 이것은 일회성 초기화 작업이지만, 장기 성장을 지원합니다—iFay는 휴먼 프라임(Human Prime)의 생활과 함께 지속적으로 새로운 데이터를 축적합니다.

### 시나리오

> 천밍은 지난 10년간의 디지털 생활을 자신의 iFay에 맡기기로 결정했습니다.
>
> 먼저 iFay가 핸드폰 앨범에 접근하도록 인가했습니다—30,000장 이상의 사진과 동영상으로, 대학 졸업부터 아이 출생까지 모든 중요한 순간을 기록하고 있습니다. 그런 다음 위챗 채팅 기록, 업무 이메일 아카이브, Notion 노트 라이브러리를 가져왔습니다.
>
> iFay가 이 데이터를 처리한 후, 천밍이 작년에 어디로 여행했는지뿐만 아니라 그의 소셜 서클, 업무 리듬, 생활 습관도 이해하게 되었습니다. 천밍의 동료가 지난번 팀빌딩 식당 이름을 물었을 때, iFay는 몇 초 만에 사진의 지오태그와 채팅 기록에서 답을 찾을 수 있었습니다—마치 천밍 자신이 떠올린 것처럼.

## 휴먼 프라임(Human Prime) 권한(→ 크리덴셜 위임)

권한은 iFay가 "당신을 대신해 무엇을 할 수 있는지"를 결정합니다. 권한이 없는 iFay는 열쇠 없는 집사와 같습니다—무엇을 해야 하는지 알지만, 어떤 문도 열 수 없습니다.

### 세 가지 권한 출처

iFay의 권한에는 세 가지 출처가 있으며, 이들 사이에 포함 관계는 없습니다:

| 출처 | 설명 | 예시 |
|------|------|------|
| **휴먼 프라임(Human Prime) 직접 인가** | 휴먼 프라임(Human Prime)이 기존 크리덴셜을 iFay에 위임하여 사용 | Netflix 계정, 은행 조회 권한 위임 |
| **휴먼 프라임(Human Prime) 권한 교환** | 휴먼 프라임(Human Prime)의 권한 교환으로 파생된 권한 | 휴먼 프라임(Human Prime)의 기업 계정으로 교환한 API 접근권 |
| **iFay 자주 획득** | iFay가 자체 행동을 통해 획득한 독립 권한 | GMChain 기여로 획득한 API 접근권, Fay 소셜 네트워크에서 구축한 신뢰 크리덴셜 |

### 포함 관계 없음

핵심 설계: 휴먼 프라임(Human Prime) 권한과 iFay 권한 사이에는 **포함 관계가 없습니다**. iFay는 휴먼 프라임(Human Prime)이 갖지 않은 권한을 가질 수 있고(예: iFay가 기여로 획득한 API 접근권), 휴먼 프라임(Human Prime)도 iFay에 위임하지 않은 권한을 가질 수 있습니다(예: 휴먼 프라임(Human Prime)의 개인 은행 계좌).

### 7가지 크리덴셜 유형

iFay는 다음 7가지 크리덴셜 유형의 관리를 지원합니다: FayID, 계정 비밀번호, 인증서, 인가, 액세스 토큰, 스마트 컨트랙트, 디지털 토큰(MeriToken). 휴먼 프라임(Human Prime)이 크리덴셜을 위임할 때, 원본 크리덴셜은 대응하는 사본 크리덴셜로 교환되며, iFay는 사본을 사용하여 로그인과 인증을 수행하여 원본 크리덴셜의 안전을 보장합니다.

### 시나리오

> 이웨이는 Netflix 계정을 iFay에 위임했습니다—이것은 **휴먼 프라임(Human Prime) 직접 인가**입니다. iFay가 받은 것은 이웨이의 원본 비밀번호가 아니라 대응하는 사본 크리덴셜입니다. 이제 iFay는 이웨이의 시청 선호에 따라 자동으로 재생 목록을 관리할 수 있습니다.
>
> 동시에, 이웨이의 iFay는 GMChain을 통해 다른 Fay에게 정보 조립 서비스를 제공하며 상당한 MeriToken을 축적했습니다. 이 기여 덕분에 iFay는 고급 날씨 데이터 API의 접근권을 획득했습니다—이것은 **iFay 자주 획득** 권한이며, 이웨이 본인은 이 API의 접근권을 갖고 있지 않습니다.
>
> 하지만 이웨이의 개인 투자 계좌는 iFay에 위임하지 않기로 했습니다. 이것은 전혀 문제없습니다—휴먼 프라임(Human Prime)과 iFay의 권한 경계는 독립적입니다.
