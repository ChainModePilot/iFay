# 10.1 시뮬레이션 조작

## 한 줄 정의

시뮬레이션 조작은 iFay의 **양손**입니다—사람처럼 클릭, 드래그, 스크롤, 타이핑하여 기존 소프트웨어 인터페이스를 조작할 수 있으며, 소프트웨어가 AI를 위해 어떤 개조도 할 필요 없습니다.

## 왜 필요한가

시뮬레이션 조작은 iFay가 세상의 모든 소프트웨어가 AI를 위해 재설계되기를 기다리지 않고, **지금 바로 일을 도울 수 있게** 합니다.

시뮬레이션 조작과 전통적 "자동화 스크립트"(RPA 등)의 본질적 차이: RPA는 사전 작성된 스크립트에 따라 단계별로 실행하며, 인터페이스가 조금만 변해도 멈춥니다. iFay의 시뮬레이션 조작은 **감지 기반**입니다: 일인칭 추적기를 통해 화면을 "보고", 현재 인터페이스 상태를 이해한 후 다음 단계를 결정합니다.

## 작동 방식

**1. 감지—먼저 보고, 그 다음 손을 움직임**

조작 실행 전, iFay는 일인칭 추적기를 통해 현재 인터페이스를 "봅니다". HTML 코드를 파싱하는 것이 아니라 사람처럼 **화면에 무엇이 표시되는지 봅니다**.

**2. 조작—사람처럼 상호작용**

클릭, 드래그, 스크롤, 타이핑, 엣지 제스처, 멀티 핑거 제스처를 지원합니다.

**3. 피드백—완료 후 결과 확인**

매 조작 후 일인칭 추적기를 통해 인터페이스를 다시 "보고" 조작 성공 여부를 확인합니다. 이 "조작→관찰→조정" 순환이 RPA 스크립트와의 근본적 차이입니다.

## 다른 모듈과의 관계

| 관련 모듈 | 관계 | 인체 비유 |
|---------|------|---------|
| **일인칭 추적기** | 시뮬레이션 조작의 "눈", 매 조작 전후 인터페이스 상태를 감지 | 손 ↔ 눈(손-눈 협응) |
| **스킬 호출** | 같은 동작 서브시스템이지만 방식이 다름 | 손으로 조작 vs 전화로 처리 |
| **자율 행동** | 자율 행동이 "언제 손을 움직일지" 결정, 시뮬레이션 조작이 "어떻게 움직일지" 담당 | 의지가 행동 결정 → 손이 동작 실행 |
| **크리덴셜 관리** | 로그인 시 크리덴셜 관리가 제공하는 계정 비밀번호 사용 | 손으로 열쇠를 들고 문 열기 |

## 개발자를 위해

시뮬레이션 조작은 **1단계(인간 조작 시뮬레이션)**의 핵심 모듈입니다.

- **요구사항 번호**: 요구사항 5(시뮬레이션 조작)
- **인터페이스 규격**: `SimulatedOperation` 인터페이스, `execute()`, `explore()`, `getPostActionState()` 3개 핵심 메서드 포함
- **지원 조작 유형**: `click`, `drag`, `scroll`, `gesture`, `type`
- **핵심 설계**: 스크립트 기반이 아닌 감지 기반; 매 조작 후 추적기를 통해 상태 변화 감지, "조작→감지→조정" 폐쇄 루프 형성
- **적합성 테스트**: iFACTS L1이 각 조작 유형의 실행 능력을 검증, L2가 일인칭 추적기와의 손-눈 협응 인터페이스를 검증
- **iFay Ready 관련**: Bronze 등급 인증은 앱이 iFay의 시뮬레이션 조작을 통한 제어를 지원할 것을 요구
