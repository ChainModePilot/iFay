# 9.2 센서

## 한 줄 정의

센서는 iFay의 **신경계**입니다—온도, 위치, 심박수, 광선까지 주변 환경의 모든 변화를 감지할 수 있게 하며, 마치 전신에 퍼진 신경 말단과 같습니다.

## 왜 필요한가

일인칭 추적기가 iFay의 눈과 귀라면, 센서는 iFay의 **전체 신경 네트워크**입니다.

핸드폰과 스마트워치를 생각해 보세요: GPS는 위치를, 가속도계는 걷기인지 달리기인지를, 광선 센서는 실내인지 실외인지를, 심박 센서는 심박이 빠른지를 알려줍니다. iFay의 센서 모듈은 이 모든 디바이스의 센서를 **통합 접속**하여 iFay에게 완전한 신경계를 부여합니다.

## 아키텍처에서의 위치

```
iFay 4레이어 아키텍처
├── 소셜 레이어
├── 인터랙션 레이어          ← 센서는 여기
│   ├── 감지(Sense)
│   │   ├── 일인칭 추적기   ← 밖을 봄, 화면을 봄
│   │   ├── 👉 센서        ← 환경 감지(온도, 위치, 운동…)
│   │   └── 자기인식         ← 안을 봄, 당신을 읽음
│   └── 동작(Action)
├── 코그니션 레이어
│   ├── 사고
│   │   ├── 개인 데이터 힙      ← 센서 데이터가 최종 저장되는 곳
│   └── 스킬
│       ├── 디바이스 드라이버 허브    ← 센서의 하드웨어 인터페이스를 관리
└── 에고 레이어
```

센서는 **인터랙션 레이어의 감지 서브시스템**에 위치합니다. 센서 자체는 **감도 조절**만 담당하며(언제 더 많이 수집하고 언제 덜 수집할지 결정), 실제 하드웨어 연결과 데이터 저장은 각각 코그니션 레이어의 **디바이스 드라이버 허브**와 **개인 데이터 힙**이 담당합니다.

## 작동 방식

**1. 브릿지—디바이스의 센서 연결**

센서 모듈은 "번역관"처럼, 모든 다른 디바이스, 다른 유형의 센서 데이터를 iFay가 이해할 수 있는 형식으로 통일 번역합니다. **CAP(Control Authority Protocol)**와 **DTP(Data Tunnel Protocol)**를 기반으로 브릿지를 구현합니다.

**2. 조절—동적 감도**

iFay의 센서도 동적 감도 조절이 가능합니다:
- 사무실에서 조용히 일할 때 GPS는 매초 위치를 업데이트할 필요 없음
- 하지만 운전 내비게이션 시 GPS는 고빈도 업데이트 필요
- 수면 시 심박 센서는 샘플링 빈도를 낮출 수 있음
- 하지만 심박 이상 감지 시 즉시 샘플링 빈도를 높임

**3. 확장—미래 센서도 접속 가능**

센서 모듈의 설계는 개방적입니다—새로운 센서 유형이 나타나면 디바이스 드라이버 허브를 통해 새 드라이버를 등록하기만 하면 됩니다.

## 다른 모듈과의 관계

| 관련 모듈 | 관계 | 인체 비유 |
|---------|------|---------|
| **디바이스 드라이버 허브** | 센서의 실제 하드웨어 인터페이스를 관리 | 신경 말단 → 신경 전도 경로 |
| **개인 데이터 힙** | 센서가 수집한 데이터가 최종 저장됨 | 감각 신호 → 기억 저장 |
| **일인칭 추적기** | 같은 감지 서브시스템이지만 분업이 다름 | 눈 vs 전신 신경 |
| **자기인식** | 센서가 환경 데이터를 제공하고, 자기인식이 이를 활용하여 휴먼 프라임(Human Prime) 상태를 추론 | 신경계가 감각 제공 → 뇌가 감정 해석 |
| **CAP / DTP 프로토콜** | 센서가 이 두 프로토콜을 기반으로 데이터 브릿지 구현 | 신경 신호의 전송 프로토콜 |

## 개발자를 위해

센서 모듈은 **2단계(클라이언트 직접 접수)**의 핵심 모듈로, CAP(Control Authority Protocol)와 DTP(Data Tunnel Protocol)에 의존합니다.

- **요구사항 번호**: 요구사항 7(센서 모듈)
- **인터페이스 규격**: `SensorModule` 인터페이스, `registerSource()`, `adjustSensitivity()`, `getDataStream()`, `getActiveStatus()` 4개 핵심 메서드 포함
- **적합성 테스트**: iFACTS L1이 감도 조절 능력을 검증, L2가 디바이스 드라이버 허브 및 개인 데이터 힙과의 인터페이스 대접을 검증
