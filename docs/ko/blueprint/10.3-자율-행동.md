# 10.3 자율 행동

## 한 줄 정의

자율 행동은 iFay의 **자주 의지**입니다—시뮬레이션 조작은 손, 스킬 호출은 입, 자율 행동은 iFay 스스로 "언제 손을 움직이고, 언제 입을 열지" 결정합니다. iFay를 수동적 실행자에서 능동적 행동자로 변화시킵니다.

## 왜 필요한가

진정으로 좋은 비서는 **능동적으로** 일합니다:
- 매주 월요일 아침 자동으로 지난주 업무 요약 정리(**정시 작업**)
- 당신이 미간을 찌푸리고 한숨 쉬는 것을 보고 어려움이 있는지 물어봄(**눈치 후 행동**)
- "택배 알림 받을 때마다 물류 확인해 줘"라고 했던 것을 계속 기억하고 매번 자동 확인(**지속 실행 습관**)

중요한 안전 메커니즘: iFay가 자율적으로 한 일의 결과가 맞지 않으면, **멈추고 물어봅니다**.

## 작동 방식

**세 가지 트리거 방식:**

**1. 정시 작업—시간이 되면 실행**

가장 간단한 트리거 방식으로, 설정한 알람과 같습니다.

**2. 자기인식 추론—당신이 필요하다고 감지하면 능동적으로 실행**

가장 "지능적인" 트리거 방식입니다. 자기인식 모듈을 통해 상태와 환경을 관찰하고, 무엇이 필요한지 추론한 후 능동적으로 행동합니다.

**3. 영구 스킬—백그라운드에서 계속 실행되는 습관**

일부 스킬은 일회성이 아니라 지속적으로 실행됩니다. 이 영구 스킬은 **등록 스킬**(외부 서비스가 제공하는 지속 모니터링 능력)과 **내부 스킬**(iFay 자체의 고유 능력)에서 옵니다.

**동작→피드백→재동작 순환**

어떤 트리거 방식이든, 자율 행동은 핵심 순환을 따릅니다:
1. **동작**: 자율적으로 결정한 작업 실행
2. **피드백**: 실행 결과 관찰, 기대에 부합하는지 판단
3. **재동작**: 부합하면 다음 단계 계속; 부합하지 않으면 전략 조정 또는 일시 정지하여 확인 요청

**안전 밸브: 일시 정지 후 확인**

자율 행동의 결과가 의도와 일치하지 않으면, **모든 후속 자율 동작을 일시 정지**하고 물어봅니다.

## 개발자를 위해

자율 행동은 **4단계(iFay + coFay 전면 의인화)**의 핵심 모듈입니다.

- **요구사항 번호**: 요구사항 14(자율 행동)
- **인터페이스 규격**: `SelfDrivenBehavior` 인터페이스, `scheduleTask()`, `handleInference()`, `pauseAndConfirm()`, `getLoopStatus()` 4개 핵심 메서드 포함
- **세 가지 트리거 소스**: `scheduled`(정시 작업), `self_awareness`(자기인식 추론), `registered_skill` / `internal_skill`(영구 스킬)
- **적합성 테스트**: iFACTS L1이 세 가지 트리거 소스의 응답 능력을 검증, L4가 자율 행동의 안전 제약(일시 정지 확인 메커니즘)을 검증
