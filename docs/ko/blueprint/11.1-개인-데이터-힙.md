# 11.1 개인 데이터 힙

## 한 줄 정의

개인 데이터 힙은 iFay의 **기억**입니다—사진, 채팅 기록, 업무 문서, 건강 데이터…핸드폰, 컴퓨터, 클라우드 드라이브 곳곳에 흩어져 있습니다. 개인 데이터 힙이 이를 통합 관리하여, iFay가 언제든 과거의 어떤 기억이든 "떠올릴" 수 있게 합니다.

## 왜 필요한가

iFay가 당신의 디지털 분신이 되려면, 당신의 모든 것을 "기억"할 수 있어야 합니다—하지만 모든 데이터를 한 곳으로 옮기라고 요구할 수는 없습니다. 개인 데이터 힙의 해결책은: **데이터는 원래 위치에 두되, iFay가 통일된 방식으로 읽고 씁니다**. 마치 뇌가 기억이 어떤 뉴런에 저장되어 있는지 신경 쓰지 않고, "떠올릴" 수만 있으면 되는 것처럼.

## 작동 방식

**1. 통일—하나의 인터페이스로 모든 데이터 관리**

통일된 읽기/쓰기 인터페이스를 제공합니다—iFay의 다른 모듈은 "3년 전 항저우에서 찍은 사진을 찾아줘"라고만 하면, 개인 데이터 힙이 자동으로 해당 저장 위치에서 찾아줍니다.

**2. 분류—4가지 데이터 유형**

- **콘텐츠(Content)**: 당신이 만든 것—사진, 동영상, 글, 노트, 코드
- **데이터(Data)**: 구조화된 정보—건강 데이터, 재무 기록, 위치 이력, 센서 데이터
- **지식 베이스(Knowledge Base)**: 정리된 지식—전문 노트, 학습 자료, 업무 경험 정리
- **정보 피드(Info Feed)**: 지속 업데이트되는 정보—뉴스 구독, 소셜 미디어 동태, 이메일 알림

**3. 영구—데이터의 안전한 보관**

DTP(Data Tunnel Protocol)를 통해 데이터의 영구 저장과 보호를 담당합니다.

**4. 풍부화—원시 데이터를 더 가치 있게**

개인 데이터 힙은 수동적으로 데이터를 저장할 뿐만 아니라, 데이터에 대한 **풍부화와 개인화 처리**도 가능합니다.

## 개발자를 위해

개인 데이터 힙은 **2단계(클라이언트 직접 접수)**의 핵심 모듈입니다.

- **요구사항 번호**: 요구사항 11(개인 데이터 힙)
- **인터페이스 규격**: `PersonalDataHeap` 인터페이스, `read()`, `write()`, `enrich()`, `persist()` 4개 핵심 메서드 포함
- **4가지 데이터 유형**: Content, Data, Knowledge Base, Info Feed
- **관련 프로토콜**: DTP(Data Tunnel Protocol)
- **적합성 테스트**: iFACTS L1이 4가지 데이터 유형의 읽기/쓰기 능력을 검증, L3가 "데이터 수집→저장→풍부화→검색"의 완전한 링크를 검증
