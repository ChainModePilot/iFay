# 7. iFay 프로필

iFay Profile은 iFay의 완전한 "신분증"입니다—시맨틱 해석 가능한 6차원 속성 표입니다. 인간이 iFay를 식별하고, 시스템이 해석하고, Fay 간 상호 인식에 사용됩니다.

휴먼 프라임(Human Prime) 프로필(정렬 의식 / Aligned Consciousness)은 iFay Profile의 하위 집합이자 입력 소스입니다. 휴먼 프라임(Human Prime) 프로필은 "휴먼 프라임(Human Prime)이 어떤 사람인가"를 기술하고, iFay Profile은 "이 iFay가 어떤 존재인가"를 기술합니다—휴먼 프라임(Human Prime)의 인격 특성을 포함하지만, iFay 자체의 스킬, 하드웨어, 권한 등 독립 속성도 포함합니다.

---

## iFay 신원

iFay 신원은 FayID와 메타 정보로 구성됩니다. FayID는 전역 고유, 인간 가독, 기계 가독의 영구 식별자로, 모든 상호작용 시나리오에서 iFay가 식별되고 추적되는 기반입니다.

---

## Ego 모델

Profile의 Ego 차원은 현재 활성 Ego 버전 참조를 기록합니다.

Ego 모델은 플러그인 방식의 전환 가능한 엣지 소형 모델로, 어떤 단말 디바이스에든 로드하면 해당 디바이스가 휴먼 프라임(Human Prime)의 개성을 갖게 됩니다. iFay는 여러 Ego 버전을 지원하여 휴먼 프라임(Human Prime)의 다면적 인격에 적응하지만—어느 시점에서든 하나의 버전만 활성 상태이며, "인격 분열"을 방지합니다.

전환 방식은 두 가지: 휴먼 프라임(Human Prime) 수동 트리거, 또는 iFay가 컨텍스트에 기반하여 자동 판단. 전환 완료 후 Profile의 활성 Ego 버전 참조가 동기 업데이트됩니다.

---

## Faying 사고

Faying 사고는 iFay의 인지 자산으로, 4가지 하위 유형을 포함합니다:

| 하위 유형 | 설명 | 예시 |
|--------|------|------|
| **콘텐츠(Content)** | 구조화 또는 비구조화 콘텐츠 자산 | 사진 컬렉션, 작성한 글, 창작한 음악, 녹화한 동영상 |
| **데이터(Data)** | 개인 데이터 | 건강 모니터링 데이터, 소비 기록, 앱 사용 습관, 위치 이력 |
| **지식 베이스(Knowledge Base)** | 조직화된 지식 체계 | 전문 분야 노트, 학습 자료 라이브러리, 업무 경험 정리, 업계 연구 보고서 |
| **정보 피드(Info Feed)** | 실시간 정보 구독 | 팔로우하는 뉴스 소스, 업계 동향 푸시, 소셜 미디어 업데이트, 주식 시세 |

---

## Faying 스킬

Faying 스킬은 iFay의 행동 능력으로, 6가지 스킬 유형을 포함합니다:

| 유형 | 설명 |
|------|------|
| **API** | 직접 호출 가능한 프로그래밍 인터페이스 |
| **워크플로(Workflow)** | 다단계 자동화 프로세스 |
| **Bot** | 특정 시나리오의 대화 로봇 |
| **Agent** | 자율 의사결정 능력을 갖춘 지능형 에이전트 |
| **APP** | 완전한 애플리케이션 |
| **마이크로서비스(Microservice)** | 독립 배포된 서비스 유닛 |

스킬 등록은 iFay가 어떤 동작을 실행하기 위한 전제 조건입니다.

---

## Faying 하드웨어

하드웨어는 iFay의 "팔다리"입니다. iFay는 인격과 인지 엔티티이며, 하드웨어는 물리 세계와 상호작용하는 매체입니다.

### 세 가지 하드웨어 하위 유형

| 하위 유형 | 설명 | 예시 |
|--------|------|------|
| **디바이스(Device)** | 단말 하드웨어 디바이스 | 핸드폰, 컴퓨터, 스마트워치, 드론, 스마트 홈 디바이스 |
| **스토리지(Storage)** | 데이터 저장 리소스 | 클라우드 스토리지, 로컬 하드디스크, NAS, 벡터 데이터베이스 |
| **컴퓨팅(Computing)** | 컴퓨팅 리소스 | GPU, 클라우드 컴퓨팅 노드, 엣지 컴퓨팅 디바이스 |

### 하드웨어 발견 메커니즘

iFay가 새 환경으로 마이그레이션할 때, 단말 디바이스가 현재 환경에서 Faying 가능한 하드웨어를 스캔합니다—블루투스나 WiFi가 주변 디바이스를 스캔하는 방식과 유사합니다.

---

## Faying 권한

권한은 iFay의 "관계"입니다—iFay와 외부 세계의 신뢰 경계를 정의합니다.

### 세 가지 권한 생명주기

| 생명주기 | 설명 | 예시 |
|----------|------|------|
| **선천적(Inherent)** | 생성 시 즉시 보유하는 권한 | 휴먼 프라임(Human Prime) 캘린더 접근, 선호 설정 읽기 |
| **영구(Persistent)** | 장기 유효한 권한 | 휴먼 프라임(Human Prime) 이메일 읽기/쓰기 권한, 기업 인트라넷 접근권 |
| **임시(Ephemeral)** | 필요 시 신청, 사용 후 해제 | 일회성 의료 기록 접근, 특정 API 임시 호출권 |

### 확장 가능한 인증 방식

iFay의 인증 방식은 확장 가능하며, SSO, OAuth, Fingerprint 등을 지원합니다. 여기서 Fingerprint는 통칭으로, 모든 식별 가능한 신원 표식을 포괄합니다—안면 인식, 성문, 디바이스 지문 등.

---

## Profile 가시성

Profile의 가시성은 애플리케이션 레이어 관심사로, 구체적인 애플리케이션 시나리오와 상호작용 방식에 의해 결정됩니다.

동일한 iFay Profile이 다른 시나리오에서 다른 차원을 표시합니다: Fay 간 협업 시 상대방은 스킬과 권한 차원만 볼 수 있고, 휴먼 프라임(Human Prime) 관리 인터페이스에서는 6개 차원 모두 완전히 보이며, 공개 소셜 시나리오에서는 신원과 일부 스킬 정보만 표시될 수 있습니다.

Profile 자체는 "누가 무엇을 볼 수 있는지"를 정의하지 않습니다—이 결정은 Profile을 사용하는 애플리케이션에 맡깁니다.
