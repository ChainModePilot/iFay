# 5. iFay 응용 시나리오

# 본체 특성을 iFay에 주입

정의에 따르면, iFay의 개성은 휴먼 프라임(Human Prime)으로부터 상속된다는 점을 다시 강조해야 합니다. 그러나 휴먼 프라임(Human Prime)을 인스턴시에이트(복제)하는 것은 매우 간단한 조작이어야 하며, 일회성 조작이 장기적으로 유효해야 iFay가 휴먼 프라임(Human Prime)과 동기적으로 성장할 수 있습니다.

다음 방식으로 iFay를 빠르게 초기화할 것을 권장합니다:

- ***휴먼 프라임(Human Prime) 데이터 인가***: 사진, 앱 데이터, 통신 기록 등을 포함합니다. 이 데이터는 일련의 처리를 거쳐 표준 형식으로 휴먼 프라임(Human Prime) 데이터베이스에 저장됩니다. 이 데이터는 iFay만 직접 접근할 수 있습니다.
- ***파일 가져오기***: 사진, 문서, 노트, 채팅 기록 등을 포함합니다. 이 조작은 휴먼 프라임(Human Prime)의 역사를 보충하여, iFay가 인가 이전의 과거 기억을 가질 수 있게 합니다.
- ***휴먼 프라임(Human Prime)이 iFay에게 서술***: Agent에 Prompt를 작성하는 것과 매우 유사합니다. 대화 형식이나 설문 형식이 가능합니다. 사용자의 조작 비용을 낮추기 위해, 친한 친구와 대화하듯 iFay에게 과거 경험을 이야기하는 것을 권장합니다.

구현 관점에서 세 가지 전제 기술이 충족되어야 합니다:

1. ***OS 레이어와의 긴밀한 통합***: 시스템 정보와 다른 앱의 프라이빗 데이터 읽기를 허용합니다.
2. ***개방적 데이터 전달 공용 형식***: 사용자의 앱 데이터를 iFay에 빠르게 동기화하려면 통일된 데이터 형식이 필수적입니다.
3. ***표준 연구 설문***: 채팅 주제를 제약하여 iFay가 유효한 기억 데이터를 획득하도록 돕는 참조 설문을 제공합니다.

# iFay에 외부 능력 보충

본체 특성 주입이 완료되면, iFay는 이미 당신의 성격, 기억, 기본 권한을 갖고 있습니다. 하지만 이때의 iFay는 새 도시에 막 온 사람과 같습니다—자신이 누구인지는 알지만, 이 도시에 어떤 자원이 있는지는 아직 모릅니다.

## 등록 스킬

iFay의 능력 확장은 "등록 스킬"을 통해 실현됩니다. 스킬은 iFay가 어떤 동작을 실행하기 위한 전제입니다—등록되지 않은 스킬은 호출할 수 없습니다. iFay는 6가지 스킬 유형을 지원합니다:

| 스킬 유형 | 설명 | 예시 |
|---------|------|------|
| **API** | 외부 서비스 인터페이스 직접 호출 | 번역 API, 날씨 조회 API, 결제 인터페이스 |
| **워크플로(Workflow)** | 다단계 편성 자동화 프로세스 | "이메일 수신→핵심 정보 추출→요약 생성→위챗으로 전송" |
| **Bot** | 대화식 상호작용 자동화 프로그램 | 고객 서비스 Bot, 예약 Bot |
| **Agent** | 자율 의사결정 능력을 갖춘 AI 에이전트 | 코드 리뷰 Agent, 데이터 분석 Agent |
| **APP** | 완전한 애플리케이션 | 캘린더 앱, 노트 앱, 내비게이션 앱 |
| **마이크로서비스(Microservice)** | 독립 배포된 백엔드 서비스 | 이미지 인식 서비스, 음성-텍스트 변환 서비스 |

스킬 등록 시 사전 인가 단계가 완료되어, 후속 호출 시 추가 인증 없이 지연을 줄입니다.

## 외부 지식 접속

스킬 외에도, iFay는 외부 지식 베이스와 모델에 접속하여 특수한 스킬 유형으로 간주할 수 있습니다. 이를 통해 iFay는 전문가 고문에게 자문하듯 휴먼 프라임(Human Prime) 자신의 능력을 초월하는 지식과 지능에 접근할 수 있습니다.

외부 지식은 SSP(Skill Sharing Protocol)를 통해 네트워크상의 표준화된 개방 서비스와 인터페이스에 접근합니다.

# 인간과 iFay의 상호작용 방식

## iFay 활성화 방법

iFay는 항시 온라인인 백그라운드 서비스가 아닙니다—당신이 명확히 인가할 때만 활성화 상태입니다.

### Faying 페어링 과정

iFay 활성화 과정을 "Faying"(연결)이라 하며, Faying 프로토콜에 따라 보안 페어링을 완료합니다:

1. **페어링 발의**: 단말 디바이스를 통해 iFay에 연결 요청을 발의하고 신원 크리덴셜을 제공합니다.
2. **신원 확인**: 시스템이 FayID 등록 센터를 통해 신원을 확인하고, 해당 iFay의 합법적 휴먼 프라임(Human Prime)임을 확인합니다.
3. **다중 인증**: 시스템이 페어링 챌린지를 반환하고, 다중 인증(생체 인식 + 디바이스 지문 + 구령 등)을 완료해야 합니다.
4. **활성화 확인**: 인증 통과 후, iFay가 "Faying" 활성화 상태에 진입하여 서비스를 시작합니다.

## iFay의 자주 의식

iFay는 단순히 지시를 기다리는 실행자가 아닙니다. 4단계(전면 의인화)에서 iFay는 자기인식과 자율 행동 능력을 갖추어, 당신과 주변 환경을 능동적으로 관찰하고, 감정과 의도를 추론하며, 자율적으로 행동할 수 있습니다.

### 자기인식 모듈

자기인식 모듈은 iFay의 "눈치" 능력입니다. 내부를 모니터링하여 휴먼 프라임(Human Prime)의 반응을 관찰합니다.

### 자율 행동

iFay의 자율 행동은 세 가지 트리거 소스에 의해 구동됩니다:
- **정시 작업**: 사전 설정된 시간 계획에 따라 자동 실행
- **자기인식 추론**: 현재 상태에 대한 추론에 기반하여 능동적으로 작업 발의
- **영구 스킬**: 등록 스킬과 내부 스킬이 지속 실행되어 "동작→피드백→재동작"의 자율 순환 형성

## iFay 종료 방법

iFay 종료 과정을 "Separating"(분리)이라 하며, 마찬가지로 Faying 프로토콜을 따릅니다. iFay는 현재 완전한 상태 스냅샷을 저장하고, "Faying" 활성화 상태에서 "Separating" 휴면 상태로 전환하며, 모든 자율 행동을 중지합니다.

# iFay의 소셜 기능

## Fay 간 통신: 텔레파시 프로토콜(Telepathy Protocol)

iFay가 다른 Fay와 통신할 때, 텔레파시 프로토콜(Telepathy Protocol)을 사용합니다—UI 번역 레이어를 제거한 시맨틱 통신 프로토콜입니다. 약정된 벡터 인코딩 토큰으로 의미와 의도를 직접 전송하여, 효율성과 정확성이 전통적 방식을 훨씬 초과합니다.

## coFay와의 협업

coFay(Common Fay)는 공공 역할을 담당하는 디지털 분신입니다. 당신의 iFay는 coFay에게 도움을 요청할 수 있으며, 현실에서 전문가에게 도움을 구하는 것과 같습니다.

핵심 메커니즘: Fay는 작업 실행 전에 가격을 협상합니다. 이를 통해 모든 협업의 기여가 추적 가능하고 평가 가능하며, 최종적으로 GMChain에 기록되고 MeriToken으로 정산됩니다.

# 보안성

iFay의 보안 설계는 근본 원칙을 따릅니다: **사회 윤리가 모든 것에 우선합니다**. 휴먼 프라임(Human Prime)의 지시가 무엇이든, iFay는 사회 윤리와 공공 질서를 위반하지 않습니다.

## 윤리 우선

iFay의 모든 행동 결정은 윤리 적합성 검사 흐름을 거칩니다:
1. **사회 윤리 검사**(최고 우선순위): 행동이 사회 윤리와 공공 질서를 위반하는가?
2. **휴먼 프라임(Human Prime)과의 정렬 검사**: 행동이 휴먼 프라임(Human Prime)의 가치관과 의도에 일치하는가?
3. **권한 검사**: iFay가 해당 행동을 실행할 권한이 있는가?

## 크리덴셜 격리

크리덴셜을 iFay에 위임할 때, iFay는 원본 크리덴셜을 직접 사용하지 않습니다. 시스템이 원본을 대응하는 사본으로 교환하며, 사본이 유출되어도 원본은 안전합니다.

# AI 산업 생태계 형성

## 세 가지 개발자 역할

| 역할 | 책임 | 비유 |
|------|------|------|
| **오픈소스 프로젝트 개발자** | iFay 핵심 모듈과 프로토콜 개발 참여 | Linux 커널 기여자와 유사 |
| **애플리케이션 개발자** | 자사 제품에 iFay 지원 도입 | Siri/Alexa 지원 앱 개발과 유사 |
| **서비스 제공자 개발자** | iFay 규격 하에서 iFay 구현 생성 | 다른 제조사가 브라우저 표준을 구현하는 것과 유사 |

## iFay Ready 인증

| 등급 | 요구사항 | 의미 |
|------|------|------|
| **Bronze** | iFay가 시뮬레이션 조작으로 앱을 제어하는 것을 지원 | iFay가 사람처럼 앱 인터페이스를 조작 가능 |
| **Silver** | CAP(Control Authority Protocol) 직접 제어 + DTP(Data Tunnel Protocol) 데이터 교환 지원 | iFay가 UI를 우회하여 앱을 직접 제어 가능 |
| **Gold** | SSP(Skill Sharing Protocol) 스킬 공유 + 완전한 C/F/S 아키텍처 통합 지원 | 앱이 iFay 생태계에 완전히 융합 |

## GMChain과 MeriToken 경제 모델

iFay 생태계의 가치 유통은 GMChain(Global Merit Chain)과 MeriToken을 통해 실현됩니다. GMChain은 모든 Fay의 기여를 추적, 측정, 평가하고 MeriToken으로 기여자에게 보상합니다.

MeriToken의 획득은 사회적 가치 창출에 기반하며, 연산력을 소모하여 블록체인 기술 작업을 완료하는 것이 아닙니다.
