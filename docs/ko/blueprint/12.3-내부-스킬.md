# 12.3 내부 스킬

## 한 줄 정의

내부 스킬은 iFay의 **직관과 마지노선**입니다—등록 스킬이 iFay가 외부에서 배운 능력(취득한 자격증처럼)이라면, 내부 스킬은 iFay가 타고난 능력(직관과 도덕적 마지노선처럼)입니다. 외부 스킬의 출력이 휴먼 프라임(Human Prime)의 의도를 위배하지 않도록 보장합니다.

## 왜 필요한가

내부 스킬은 세 가지를 합니다:

**첫째, 휴먼 프라임(Human Prime) 개성과 정렬된 습관을 수립하고, 외부 스킬에 대한 거버넌스를 포함합니다.** 외부 스킬의 출력에 대해 "직관 검사"를 수행합니다—이 결과가 휴먼 프라임(Human Prime)의 스타일에 부합하는가? 가치관에 부합하는가?

**둘째, 내성 메커니즘을 제공하여 외부 지식이 휴먼 프라임(Human Prime) 의도와 충돌하지 않도록 보장합니다.** "필터" 역할을 하여, 외부 지식이 iFay에 채택되기 전에 휴먼 프라임(Human Prime) 의도의 검증을 거치게 합니다.

**셋째, 휴먼 프라임(Human Prime) 특정의 고유 능력을 임베드합니다.** 금융 분석가라면, iFay는 태생적으로 금융 분석 능력을 갖춰야 합니다—외부에서 등록한 스킬이 아니라 초기화 시 임베드된 "전문 소양"입니다.

간단히 말해: 등록 스킬은 iFay의 "외공"이고, 내부 스킬은 iFay의 "내공"입니다.

## 개발자를 위해

내부 스킬은 **4단계(iFay + coFay 전면 의인화)**의 핵심 모듈입니다.

- **요구사항 번호**: 요구사항 15(내부 스킬)
- **인터페이스 규격**: `InternalSkill` 인터페이스, `introspect()`, `intercept()`, `getHostCapabilities()` 3개 핵심 메서드 포함
- **세 가지 책임**: (1) 휴먼 프라임(Human Prime) 개성과 정렬된 습관 수립, 외부 스킬 출력 거버넌스; (2) 내성 메커니즘으로 외부 지식이 의도와 충돌하지 않도록 보장; (3) 휴먼 프라임(Human Prime) 특정 고유 능력 임베드
- **등록 스킬과의 차이**: 등록 스킬은 외부 획득 능력(등록과 사전 인가 필요), 내부 스킬은 타고난 능력(초기화 시 임베드, 등록 불필요)
- **적합성 테스트**: iFACTS L1이 내성과 차단 능력을 검증, L4가 외부 출력 조정이 진정으로 휴먼 프라임(Human Prime) 프로필에 부합하는지 검증
