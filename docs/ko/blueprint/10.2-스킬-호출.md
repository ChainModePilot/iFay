# 10.2 스킬 호출

## 한 줄 정의

스킬 호출은 iFay의 **입과 전화**입니다—시뮬레이션 조작이 iFay가 손으로 인터페이스를 조작하는 것이라면, 스킬 호출은 iFay가 직접 서비스에 "전화를 거는" 것입니다—인터페이스를 조작할 필요 없이 API를 직접 호출하거나 작업을 트리거합니다.

## 왜 필요한가

문서를 번역해야 한다고 상상해 보세요. 두 가지 방식이 있습니다:

**방식 1**: 브라우저를 열고, 번역 웹사이트에 들어가서, 텍스트를 붙여넣고, 번역을 기다리고, 결과를 복사합니다. 이것이 "시뮬레이션 조작" 방식입니다.

**방식 2**: 번역 회사에 직접 전화를 걸어 "이 문장을 번역해 주세요"라고 말하면, 상대방이 번역 후 직접 결과를 알려줍니다. 이것이 "스킬 호출" 방식입니다—**서비스에 직접 전화를 거는 것**.

스킬 호출은 iFay가 인터페이스를 건너뛰고 목표를 직접 달성하게 합니다. 이미 등록된 각종 서비스—API, 워크플로, Bot, Agent, App, 마이크로서비스—를 효율적으로 활용할 수 있게 합니다.

## 개발자를 위해

- **적합성 테스트**: iFACTS L1이 의도 매칭과 스킬 호출 능력을 검증, L2가 등록 스킬 모듈과의 인터페이스 대접을 검증, L3가 "의도 표현→스킬 호출→기여 기록"의 완전한 링크를 검증
