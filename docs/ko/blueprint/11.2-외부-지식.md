# 11.2 외부 지식

## 한 줄 정의

외부 지식은 iFay의 **도서관과 전문가 고문**입니다—개인 데이터 힙이 iFay 자신의 기억이라면, 외부 지식은 iFay가 열람할 수 있는 도서관이자 자문할 수 있는 전문가입니다. iFay가 휴먼 프라임(Human Prime) 자신의 능력을 초월하는 지식을 갖게 합니다.

## 왜 필요한가

개인 데이터 힙이 저장하는 것은 당신 자신의 데이터와 기억이지만, 당신의 지식은 유한합니다. 외부 지식 모듈은 iFay가 **자신의 기억 밖으로 나가** 외부 지식 베이스를 열람하고, 외부 지능 모델에 자문할 수 있게 합니다.

## 작동 방식

**1. 접속—외부 지식 소스 연결**

지식 베이스, 대형 언어 모델, 전문가 시스템, 검색 엔진 등이 **SSP(Skill Sharing Protocol)**를 통해 iFay에 접속됩니다.

**2. 조회—전문가에게 자문하듯 질문**

각 조회 결과에는 **신뢰도**가 포함되어 답변의 신뢰성을 알려줍니다.

**3. 통합—배운 지식이 자신의 것이 됨**

외부 지식 소스에서 획득한 정보는 개인 데이터 힙에 통합되어 iFay 자신의 기억이 됩니다.

**지식 소스 불가용 시**: 개인 데이터 힙의 캐시된 지식으로 **강등 사용**하며, 코그니션 레이어에 "현재 캐시 지식을 사용 중이며 최신이 아닐 수 있음"을 알립니다.

## 개발자를 위해

외부 지식은 **3단계(iFay가 가상 세계의 인터페이스로)**의 핵심 모듈입니다.

- **요구사항 번호**: 요구사항 12(외부 지식 접속)
- **인터페이스 규격**: `ExternalKnowledge` 인터페이스, `query()`, `registerSource()`, `fallbackToCache()` 3개 핵심 메서드 포함
- **지식 소스 유형**: `knowledge_base`, `llm`, `expert_system`, `search_engine`
- **관련 프로토콜**: SSP(Skill Sharing Protocol)
- **적합성 테스트**: iFACTS L1이 지식 조회와 캐시 강등 능력을 검증, L3가 "지식 조회→통합→캐시→강등"의 완전한 링크를 검증
