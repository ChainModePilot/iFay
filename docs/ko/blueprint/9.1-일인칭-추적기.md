# 9.1 일인칭 추적기

## 한 줄 정의

일인칭 추적기는 iFay의 **눈과 귀**입니다—iFay가 당신이 보는 화면을 보고, 당신이 듣는 소리를 듣게 하며, 당신과 똑같은 일인칭 시점을 갖게 합니다.

## 왜 필요한가

상상해 보세요, 비서에게 웹 양식을 작성해 달라고 부탁했습니다. 이 비서가 시각장애인이라면—화면의 입력 상자, 버튼, 오류 메시지를 볼 수 없다면—어떻게 도와줄 수 있을까요?

이것이 일인칭 추적기가 없는 iFay입니다.

인간이 컴퓨터와 핸드폰을 조작할 때, 눈으로 화면을 보고 귀로 알림음을 듣습니다. iFay도 같은 능력이 필요합니다. 일인칭 추적기는 iFay에게 이 능력을 부여하는 모듈입니다—iFay가 **당신처럼 화면을 보게** 하며, 프로그래머처럼 코드를 읽는 것이 아닙니다.

여기에 핵심적인 구분이 있습니다: iFay가 보는 것은 당신이 **육안으로 보는 화면**이지, 웹페이지 뒤에 숨겨진 코드(HTML 태그, SEO 키워드 같은 당신이 전혀 볼 수 없는 것들)가 아닙니다. 마치 책을 볼 때 글자와 그림을 보는 것이지 인쇄소의 조판 파일을 보는 것이 아닌 것처럼. iFay도 마찬가지입니다—구조화된 문서를 파싱하는 것이 아니라 "시각"으로 인터페이스를 이해하는 것을 우선합니다.

## 아키텍처에서의 위치

```
iFay 4레이어 아키텍처
├── 소셜 레이어
├── 인터랙션 레이어          ← 일인칭 추적기는 여기
│   ├── 감지(Sense)
│   │   ├── 👉 일인칭 추적기   ← 밖을 봄, 화면을 봄
│   │   ├── 센서             ← 환경 감지
│   │   └── 자기인식           ← 안을 봄, 당신을 읽음
│   └── 동작(Action)
│       ├── 시뮬레이션 조작
│       ├── 스킬 호출
│       └── 자율 행동
├── 코그니션 레이어
└── 에고 레이어
```

일인칭 추적기는 **인터랙션 레이어의 감지 서브시스템**에 위치합니다. iFay를 사람에 비유하면, 인터랙션 레이어는 iFay가 외부 세계와 상호작용하는 "몸"이고, 일인칭 추적기는 이 몸의 **눈과 귀**입니다.

## 작동 방식

일인칭 추적기의 작동 방식은 매우 직관적입니다—당신이 직접 화면을 보는 것과 같습니다:

**1. 화면을 봄**
iFay는 화면에 현재 표시된 내용을 캡처합니다—텍스트, 이미지, 버튼, 입력 상자, 팝업…당신이 볼 수 있는 모든 것을 iFay도 볼 수 있습니다.

**2. 소리를 들음**
인터페이스에 알림음, 음성 안내, 동영상 소리가 있으면 iFay도 캡처할 수 있습니다.

**3. 실시간 변화 추적**
가장 중요한 부분입니다. iFay가 화면에서 조작을 실행할 때(예: 버튼 클릭), **즉시 조작 결과를 봐야** 합니다—페이지가 이동했나? 오류 메시지가 나타났나? 로딩 애니메이션이 아직 돌고 있나?

마치 마우스로 "제출" 버튼을 클릭한 후 눈이 자동으로 화면을 주시하며 결과를 보는 것처럼. iFay의 일인칭 추적기도 같은 일을 합니다:
- 커서 이동 후 화면 변화 추적
- 창 전환 후 새로 노출된 영역 추적
- 페이지 동적 업데이트 추적(예: 실시간 갱신 데이터)

**4. 손-눈 협응**
일인칭 추적기와 시뮬레이션 조작 모듈은 긴밀하게 협력하는 "파트너"입니다—마치 사람의 눈과 손처럼. 눈이 버튼 위치를 보고, 손이 정확히 클릭하고, 손이 클릭한 후 눈이 다시 결과를 봅니다. 이 "보기→하기→다시 보기" 순환이 iFay의 손-눈 협응 능력입니다.

**5. "보이지 않음" 보고**
어떤 이유(화면 잠금, 앱 크래시, 권한 부족 등)로 일인칭 추적기가 화면을 가져올 수 없으면, 아무 일도 없는 척하지 않습니다—솔직하게 iFay의 코그니션 레이어에 보고합니다: "보이지 않습니다." 코그니션 레이어는 보고를 받은 후 다음 단계를 결정합니다(대기, 재시도, 또는 알림).

## 다른 모듈과의 관계

| 관련 모듈 | 관계 | 인체 비유 |
|---------|------|---------|
| **시뮬레이션 조작** | 긴밀한 결합, 손-눈 협응 | 눈 ↔ 손 |
| **코그니션 레이어** | 감지 상태 보고(정상/저하/실패) | 눈 → 뇌("보입니다" 또는 "잘 안 보입니다") |
| **센서** | 같은 감지 서브시스템이지만 분업이 다름: 일인칭 추적기는 화면을 보고, 센서는 환경을 감지 | 눈 vs 피부의 신경 |
| **자기인식** | 같은 감지 서브시스템이지만 방향이 다름: 일인칭 추적기는 밖을 보고, 자기인식은 안을 봄 | 눈 vs 감성 지능 |

## 시나리오 스토리

### 시나리오 1: 정부 웹사이트 양식 작성 도움

오래된 정부 웹사이트에서 신청서를 작성해야 합니다. 이 웹사이트는 AI를 위한 어떤 적응도 하지 않았습니다—API도 없고, 구조화된 인터페이스도 없고, 일반 웹 양식만 있습니다.

iFay의 일인칭 추적기가 이 양식을 "봅니다": 이름 입력 상자, 주민등록번호 입력 상자, 드롭다운 메뉴, 인증 코드 이미지, 파란색 "제출" 버튼. 이 시각 정보를 시뮬레이션 조작 모듈에 전달하고, 시뮬레이션 조작 모듈은 당신의 손처럼 필드를 하나씩 채우고, 옵션을 선택하고, 인증 코드를 입력하고, 제출을 클릭합니다.

제출 후, 일인칭 추적기가 즉시 결과를 "봅니다"—빨간색 오류 메시지 "주민등록번호 형식이 올바르지 않습니다"가 나타나면, 이 변화를 캡처하고 iFay는 수정하여 다시 제출해야 한다는 것을 압니다.

전체 과정에서 iFay가 보는 것과 당신이 보는 것은 완전히 동일합니다—이 웹사이트의 HTML 코드가 어떻게 생겼는지 알 필요가 없습니다.

### 시나리오 2: 실시간 주식 시세 모니터링

iFay에게 특정 주식의 실시간 차트를 지켜보라고 합니다. 일인칭 추적기가 화면의 K선 차트와 숫자 변화를 지속적으로 "봅니다"—가격이 152.30에서 153.10으로 뛰고, 거래량 막대 그래프가 갑자기 높아지고, MACD 지표에 골든 크로스가 나타나고…

이러한 실시간 변화를 일인칭 추적기가 지속적으로 추적합니다. 가격이 설정한 목표가에 도달하면 iFay가 즉시 알려줍니다. 마치 직접 시세를 지켜보는 것과 같지만, iFay는 딴생각하지 않고, 피로하지 않으며, 24시간 쉬지 않고 "볼" 수 있습니다.

## 개발자를 위해

일인칭 추적기는 **1단계(인간 조작 시뮬레이션)**의 핵심 모듈로, iFay가 가장 먼저 구현해야 하는 부품 중 하나입니다.

- **요구사항 번호**: 요구사항 4(일인칭 추적기)
- **인터페이스 규격**: `FirstPersonTracer` 인터페이스, `captureVisual()`, `captureAudio()`, `trackChanges()`, `getPerceptionStatus()` 4개 핵심 메서드 포함
- **관련 프로토콜**: 1단계에서는 CAP/DTP 프로토콜에 의존하지 않고 OS 레벨 화면 캡처로 직접 구현; 2단계부터 CAP(Control Authority Protocol)를 통해 더 깊은 인터페이스 정보 획득 가능
- **적합성 테스트**: iFACTS L1(단일 부품 적합성)이 시각 캡처 능력을 검증, L2(인터페이스 적합성)가 시뮬레이션 조작 모듈과의 손-눈 협응 인터페이스를 검증
- **설계 요점**: 구조화된 문서 파싱보다 시각 감지 우선; 감지 실패 시 반드시 코그니션 레이어에 보고; 시뮬레이션 조작 모듈과 폐쇄 루프 피드백 형성
