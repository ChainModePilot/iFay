# 8. 소셜 레이어

소셜 레이어는 iFay의 사회에서의 **존재 방식**을 정의합니다—3가지 근본적인 질문에 답합니다: "나는 누구인가"(아이덴티티), "무엇을 할 권한이 있는가"(사회 권한), "나의 기여는 얼마나 가치 있는가"(사회 기여와 발언권).

소셜 레이어는 iFay 보안 체계의 기반이자, iFay가 사회 관계에 융합하기 위한 전제 조건입니다.

```
iFay 4층 아키텍처
├── 👉 소셜 레이어
│   ├── 8.1 아이덴티티(FayID)          ← 나는 누구인가
│   ├── 8.2 사회 권한                    ← 무엇을 할 권한이 있는가
│   │   ├── 계정/비밀번호
│   │   ├── 인증서(Certificate)
│   │   ├── 인가(Authorization)
│   │   ├── 액세스 토큰
│   │   └── 스마트 컨트랙트
│   └── 8.3 사회 기여와 발언권            ← 나의 기여의 가치
│       ├── 공예체인(GMChain)
│       └── 메리토큰(MeriToken)
├── 인터랙션 레이어
├── 코그니션 레이어
└── 에고 레이어
```

---

## 8.1 아이덴티티(FayID)

### 한 줄 정의

FayID는 iFay의 **주민등록번호**—글로벌 고유, 인간 가독, 기계 가독의 영구 식별자로, 모든 사회적 상호작용에서 iFay가 식별되고 추적되는 기반입니다.

### 왜 필요한가

FayID 설계의 2가지 핵심 고려사항:
1. **통합 식별**: iFay와 coFay는 동일한 FayID 생성 및 관리 메커니즘을 사용합니다.
2. **활성화 바인딩**: FayID는 Faying 프로토콜과 깊이 결합되어, iFay가 휴먼 프라임의 명확한 인가 하에서만 동작하도록 보장합니다.

### 핵심 특성

| 특성 | 설명 |
|------|------|
| **글로벌 고유** | 세계 인구를 초과하는 규모의 식별자 용량 지원 |
| **인간 가독** | 인간이 직관적으로 식별하고 기억 가능 |
| **기계 가독** | 시스템과 Fay 간 효율적 파싱 및 매칭 가능 |
| **영구성** | 한번 할당되면 평생 변하지 않음 |
| **활성화 상태** | Faying 프로토콜과 연동 |

### 개발자 참고

- **독립 프로젝트**: FayID는 [독립 오픈소스 프로젝트](https://github.com/ChainModePilot/FayID)
- **인터페이스 사양**: `FayIDRegistry` 인터페이스, `generate()`, `resolve()`, `getActivationStatus()` 포함
- **적합성 테스트**: iFACTS L1에서 FayID의 고유성과 영구성 검증, L2에서 Faying 프로토콜과의 활성화 상태 연동 검증

## 8.2 사회 권한

### 한 줄 정의

사회 권한은 iFay의 **키체인**—iFay가 당신을 대신하여 서비스를 이용하는 데 필요한 크리덴셜을 안전하게 관리합니다. 이 크리덴셜은 원본이 아닌 안전한 사본입니다.

### 5가지 권한 유형

#### 1. 계정/비밀번호(Account / Password)
가장 일반적인 크리덴셜 유형. 사본 메커니즘을 통해 사용합니다.

#### 2. 인증서(Certificate)
디지털 인증서. 암호화 통신과 신원 인증에 사용. X.509 등 표준 포맷 지원.

#### 3. 인가(Authorization)
OAuth 2.0 등 기존 서드파티 인가 방식과 호환.

#### 4. 액세스 토큰(Access Token)
유효기간이 있는 임시 액세스 크리덴셜. API 호출 등에 사용.

#### 5. 스마트 컨트랙트(Smart Contract)
블록체인 상의 스마트 컨트랙트 크리덴셜. 탈중앙화 서비스와 Fay 간 자동화 협력에 사용.

### 사본 메커니즘과 보안

- **사본 격리**: iFay는 원본 크리덴셜을 직접 보유하지 않고, 권한 제한된 사본만 보유
- **소유자 표시**: 각 크리덴셜의 소유자가 휴먼 프라임인지 iFay 자신인지 명시
- **감사 추적**: 모든 크리덴셜 사용을 기록
- **이상 탐지**: 이상 사용 패턴 탐지 시 자동으로 사본 폐기
- **프라이버시 보호**: 프라이버시 데이터 조회 시 불리언 결과만 반환

### 개발자 참고

- **인터페이스 사양**: `CredentialManager` 인터페이스, `delegate()`, `authenticate()`, `revoke()`, `queryPrivacy()`, `getAuditLog()` 포함
- **5가지 크리덴셜 유형**: `account_password`, `certificate`, `authorization`, `access_token`, `smart_contract`
- **적합성 테스트**: iFACTS L1에서 5가지 크리덴셜 관리 능력과 사본 메커니즘 검증, L4에서 프라이버시 조회의 불리언 반환 제약과 이상 탐지 행동 검증

## 8.3 사회 기여와 발언권(공예체인(GMChain) 과 메리토큰(MeriToken))

### 한 줄 정의

공예체인(GMChain, Global Merit Chain)은 iFay 생태계의 **사회 기여 원장**—모든 Fay와 인간이 사회에 창출한 가치를 기록하고, 메리토큰(MeriToken)으로 이러한 기여를 정량화합니다. 메리토큰은 화폐가 아니라 **발언권과 신용의 척도**입니다.

### 왜 필요한가

메리토큰의 역할:
- **기여 측정**: 당신의 iFay가 사회에 얼마나 많은 가치를 창출했는가
- **신용 구축**: 기여가 많을수록 신용이 높고, 협력 네트워크에서 더 신뢰받음
- **발언권 획득**: 커뮤니티 합의가 필요한 의사결정에서 기여자가 더 큰 발언권을 가짐
- **협력 인센티브**: Fay 간 협력 시 기여가 공정하게 기록되고 보상됨

### 핵심 메커니즘

**1. 기여 기록** — Fay 간 협력 작업, 정보 조립 서비스, API/스킬 서비스, 디바이스/연산력 리소스 등을 기록

**2. 가격 협상** — Fay는 협력 작업 실행 전 μ 단위로 가격을 사전 협상

**3. 메리토큰 원장(MeriToken Ledger)** — 누적 기여 총량, 기여 유형 분포, 신용 점수, 발언권 가중치를 기록

### 기존 암호화폐와의 본질적 차이

| 차원 | 기존 암호화폐 | 메리토큰 |
|------|-------------|---------|
| **획득 방식** | 연산력 소모(마이닝) | 사회적 가치 창출 |
| **가치 앵커** | 시장 수급과 투기 | 사회 기여도 |
| **목적** | 금융 자산 | 신용 도량·발언권·협력 인센티브 |
| **법정 화폐와의 관계** | 교환 가능 | 교환 불가 |

### 장기 비전과 현재 위치

> ⚠️ **중요 설명**: 공예체인/메리토큰은 완전 AI화 사회의 장기 비전 산물이며, 로드맵 5단계의 핵심 컴포넌트입니다. 공예체인은 화폐 주입을 받지 않으며, 메리토큰은 법정 화폐와 교환하지 않습니다.

### 개발자 참고

- **독립 프로젝트**: 공예체인은 [독립 오픈소스 프로젝트](https://github.com/ChainModePilot/Global-Merit-Chain)
- **인터페이스 사양**: `MeritLedger` 인터페이스, `recordContribution()`, `getBalance()`, `getReputation()`, `negotiate()` 포함
- **계량 단위**: μ(MU, Merit Unit)
- **적합성 테스트**: iFACTS L4에서 기여 기록의 변조 불가능성과 가격 협상의 공정성 검증
