# 11. 코그니션 레이어 — 사고

코그니션 레이어의 사고 서브시스템은 iFay의 "뇌의 기억 영역"—iFay의 모든 기억, 지식, 당신에 대한 이해를 관리하여, 모든 행동에 인지 기반을 제공합니다.

---

## 11.1 개인 데이터 힙

### 한 줄 정의

개인 데이터 힙은 iFay의 **기억**—사진, 채팅 기록, 업무 파일, 건강 데이터 등 스마트폰, PC, 클라우드에 흩어진 데이터를 통합 관리하여, 언제든 "떠올릴" 수 있게 합니다.

### 어떻게 작동하는가

**1. 통합** — 통합 읽기/쓰기 인터페이스 제공(데이터는 원래 위치에 유지)
**2. 분류** — 4가지: Content(콘텐츠), Data(데이터), Knowledge Base(지식 베이스), Info Feed(정보 피드)
**3. 영구화** — DTP 프로토콜에 의한 데이터 영구화 저장과 보호
**4. 풍부화** — 원시 데이터에 의미적 태그나 인사이트를 부가

### 다른 모듈과의 관계

| 관련 모듈 | 관계 | 인체 비유 |
|---------|------|---------|
| **센서** | 환경 데이터를 저장 | 감각 신호 → 기억 저장 |
| **외부 지식** | 외부 정보를 통합 | 도서관 지식 → 자신의 기억으로 |
| **정렬 의식** | 데이터에서 프로필을 구축 | 기억에서 "나는 어떤 사람인가"를 총괄 |
| **DTP 프로토콜** | 양방향 데이터 전송 | 기억의 입출력 채널 |

### 개발자 참고

- **인터페이스 사양**: `PersonalDataHeap`, `read()`, `write()`, `enrich()`, `persist()` 포함
- **4가지 데이터 유형**: Content, Data, Knowledge Base, Info Feed
- **적합성 테스트**: iFACTS L1에서 4가지 데이터 읽기/쓰기 능력 검증, L3에서 전체 체인 검증

---

## 11.2 외부 지식

### 한 줄 정의

외부 지식은 iFay의 **도서관과 전문가 고문**—개인 데이터 힙이 iFay 자신의 기억이라면, 외부 지식은 iFay가 참조할 수 있는 도서관과 상담할 수 있는 전문가입니다.

### 어떻게 작동하는가

**1. 접속** — SSP(스킬 공유 프로토콜)를 통해 외부 지식 소스에 접속(지식 베이스, LLM, 전문가 시스템, 검색 엔진)
**2. 조회** — 신뢰도가 포함된 결과를 반환
**3. 통합** — 중요한 지식을 개인 데이터 힙에 통합

**폴백**: 외부 지식 소스 이용 불가 시, 개인 데이터 힙의 캐시 지식으로 자동 강등.

### 다른 모듈과의 관계

| 관련 모듈 | 관계 | 인체 비유 |
|---------|------|---------|
| **개인 데이터 힙** | 지식을 통합 | 도서관 지식 → 노트에 기록 |
| **등록 스킬** | 지식 소스를 스킬로 등록 | 도서관 카드도 스킬 증명 |
| **내부 스킬** | 외부 지식의 충돌을 체크 | 자신의 판단력으로 외부 정보를 평가 |

### 개발자 참고

- **인터페이스 사양**: `ExternalKnowledge`, `query()`, `registerSource()`, `fallbackToCache()` 포함
- **지식 소스 유형**: `knowledge_base`, `llm`, `expert_system`, `search_engine`
- **적합성 테스트**: iFACTS L1에서 지식 조회와 캐시 강등 능력 검증

## 11.3 정렬 의식

### 한 줄 정의

정렬 의식은 iFay의 당신에 대한 **완전한 이해**—당신의 가치관, 선호, 습관, 경계. iFay의 모든 행동의 베이스라인입니다.

### 3가지 업데이트 방식

**1. 데이터 마이닝** — 개인 데이터 힙에서 행동 패턴을 발굴(수동적·지속적)
**2. 실시간 조정** — 자기인식 모듈로부터의 즉시 피드백(능동적·즉시적)
**3. 수동 정의** — 당신이 직접 iFay에 전달(명확·즉시적)

### 프로필 차원

가치 지향, 선호, 습관, 인지 경계, 스킬 경계, 권한 경계, 업무 스타일.

### 다른 모듈과의 관계

| 관련 모듈 | 관계 | 인체 비유 |
|---------|------|---------|
| **개인 데이터 힙** | 데이터에서 프로필을 구축 | 기억에서 자기 인식을 형성 |
| **자기인식** | 실시간으로 프로필을 업데이트 | 관찰 → 이해를 업데이트 |
| **Ego 모델** | 프로필을 개성 제약의 입력으로 제공 | 이해 → 성격을 형성 |
| **내부 스킬** | 행동 제약 기준을 제공 | 이해 → 무엇을 할 수 있는지 결정 |

### 개발자 참고

- **인터페이스 사양**: `AlignedConsciousness`, `getProfile()`, `mineFromDataHeap()`, `adjustFromAwareness()`, `manualUpdate()` 포함
- **프로필 차원**: `values`, `preferences`, `habits`, `cognitiveBoundaries`, `skillBoundaries`, `permissionBoundaries`, `workingStyle`
- **적합성 테스트**: iFACTS L1에서 3가지 프로필 업데이트 방식 검증, L4에서 프로필 업데이트의 행동 제약에 대한 실제 영향 검증
