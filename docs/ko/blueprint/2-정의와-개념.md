# 2. 정의와 개념

## 제품 고유 용어

### iFay
Individual Fay. 자연인(**휴먼 프라임(Human Prime)**, 줄여서 Prime)에 부속된 디지털 분신으로, 가치관, 개성, 습관 면에서 부속된 자연인(휴먼 프라임(Human Prime))과 정렬을 유지하는 데 전념합니다. iFay는 휴먼 프라임(Human Prime) 인격의 인스턴시에이트(복제)이며, 휴먼 프라임(Human Prime)의 인격 특성과 디지털 능력을 융합하여 기계적, 반복적, 위험한 노동과 번거로운 보조 업무를 대신하고, 안전 보장, 건강, 삶의 질을 향상시키며, 사회적 가치를 증폭시키는 것을 목표로 합니다.

### coFay
Common Fay, 공공 역할을 담당하는 디지털 분신으로, 현재 Agent 개념과 대략 동등합니다. 특정 분야에서 범용 지식과 스킬을 보유하며, 대화를 통해 여러 사람에게 서비스할 수 있습니다. coFay는 독립 프로젝트이며, iFay 규격은 필요한 경우에만 참조합니다.

### Fay
iFay와 coFay의 통칭으로, 의인화된 AI 에이전트의 범용 용어입니다. 새로운 사회 관계에서 중요한 노드가 될 수 있습니다.

### FayID
각 Fay에 할당되는 전역 고유 영구 식별자로, 인간이 인식 가능하고 기계가 읽을 수 있습니다. FayID는 iFay와 coFay에 통일된 식별자 생성 및 관리 메커니즘을 사용하며, 전 세계 인구 수 이상의 식별자 용량을 지원합니다.

### FayGer
Fay(iFay와 coFay)에 가상 런타임 환경을 제공하는 컨테이너로, Docker와 유사합니다. 이를 통해 Fay가 크로스 플랫폼으로 다양한 하드웨어/소프트웨어에 인해빗(빙의)할 수 있어, 하드웨어/소프트웨어가 휴먼 프라임(Human Prime)의 개인 의지를 갖게 됩니다. FayGer는 크로스 언어 빌드 도구와 크로스 플랫폼 실행 컨테이너를 제공하여, 어떤 Fay든(개발 언어에 관계없이) 패키징되어 실행될 수 있습니다.

### Ego 모델
iFay의 개인화된 마이크로 모델로, 특정 개인의 프로필과 정렬됩니다. Ego는 플러그인 방식의 전환 가능한 엣지 소형 모델로, 어떤 단말 디바이스에든 로드하여 해당 디바이스가 휴먼 프라임(Human Prime)의 개성을 갖게 합니다. Ego는 가치 지향, 관심 선호, 습관, 인지 경계, 스킬 경계, 권한 경계, 작업 스타일 등의 차원에서 iFay가 휴먼 프라임(Human Prime)과 정렬되도록 제약하며, 외부 대형 모델과 독립적으로 실행되어 개성 안정성을 보장합니다. 다중 버전 관리와 인격 전환을 지원하여, 휴먼 프라임(Human Prime)의 다양한 시나리오에서의 다면적 인격 요구에 적응합니다.

> ⚠️ **윤리적 제약**: Ego 버전 전환은 투명성 원칙을 준수해야 합니다. 모든 Ego 버전은 동일한 핵심 가치관을 공유해야 하며, 차이는 표현 스타일과 상호작용 선호에만 국한됩니다. 전환 시 상호작용 메타데이터에 현재 활성 Ego 버전 식별자를 표기하여 감사 가능성을 보장해야 합니다.

### iFay Profile
iFay의 통합 데이터 모델로, 시맨틱 해석 가능한 속성 표입니다. 인간과 시스템이 iFay를 식별하고, Fay 간 상호 식별에 사용되며, iFay의 완전한 "신분증"입니다. 6개 차원을 포함합니다: iFay 신원(FayID + 메타 정보), Ego 모델(현재 활성 Ego 버전 참조), Faying 사고, Faying 스킬, Faying 하드웨어, Faying 권한. 휴먼 프라임(Human Prime) 프로필(정렬 의식)은 Profile의 하위 집합이자 입력 소스입니다.

### FayManifest
iFay의 선언적 조립 설정 파일로, package.json과 유사합니다. iFay 구현에 필요한 부품, 프로토콜, 제어 모드(지시 제어/Ego 제어/자율 제어), 설정을 선언합니다. 개발자는 "무엇이 필요한지"만 선언하면 되며 "어떻게 구현하는지"는 필요 없고, 최소화 선언을 지원합니다—부품 하위 집합만 선언해도 출시할 수 있으며, 시스템이 자동으로 필수 인프라 의존성을 보충합니다.

### GMChain
**공예체인**(Global Merit Chain, GMC), iFay 생태계의 사회 기여 원장—식별 가능한 사회적 기여를 기록하고 측정하여 신용과 발언권을 구축하는 블록체인 인프라입니다. 공예체인은 CPE+M 프레임워크의 모든 레벨을 관통하며, 인간, iFay, coFay, 서비스 제공자에게 통합된 기여 추적과 인센티브 메커니즘을 제공합니다. 대부분의 블록체인 프로젝트와 달리, 공예체인은 사회적 가치의 인정에 완전히 의존하여 포인트를 축적하며, 연산력을 소모하여 계산 작업을 완료하는 것이 아닙니다. 공예체인은 화폐 주입을 받지 않으며, 메리토큰은 법정 화폐와 교환하지 않습니다. 로드맵 5단계의 장기 비전 컴포넌트이지만, 인터페이스 정의는 초기 단계에서 예약이 필요합니다.

### MeriToken
**메리토큰**, 공예체인 상의 사회 기여 도량 단위입니다. 메리토큰은 화폐가 아니라 신용, 발언권, 협력 인센티브의 정량화된 크리덴셜입니다. 획득 방식은 사회적 가치 창출(협력 작업 완료, API 서비스 제공, 연산력 기여 등)에 기반하며, 연산력을 소모하여 블록체인 기술 작업을 완료하는 것이 아닙니다. 메리토큰 원장(MeriToken Ledger)은 각 참여자의 누적 기여, 신용 점수, 발언권 가중치를 기록합니다.

### μ(MU)
Merit Unit, 메리토큰의 최소 계량 단위입니다. Fay 간 협력 시 μ 단위로 가격을 협상합니다.

### iFACTS
iFay Architecture Conformance Test Suite, 표준화된 적합성 테스트 스위트입니다. 프로토콜 상호작용, 모듈 인터페이스, 보안 행동 등 핵심 규격 포인트를 커버하며, 4개 테스트 레벨을 포함합니다: L1 단일 부품 적합성, L2 인터페이스 적합성, L3 통합 적합성, L4 행동 적합성. 제조사 구현은 iFACTS 테스트를 통과해야만 "iFay 사용 가능"을 주장할 수 있습니다.

### iFay Ready
애플리케이션 제품이 iFay에 의해 제어되기 위해 충족해야 하는 인증 표준으로, 3개 등급으로 나뉩니다:
- **Bronze**: iFay가 시뮬레이션 조작(일인칭 추적기 + 시뮬레이션 조작)을 통해 애플리케이션을 제어하는 것을 지원
- **Silver**: iFay가 CAP(Control Authority Protocol)를 통해 직접 애플리케이션을 제어하고, DTP(Data Tunnel Protocol) 데이터 교환을 지원
- **Gold**: iFay가 SSP(Skill Sharing Protocol)를 통해 스킬을 공유하고, 완전한 C/F/S 아키텍처 통합을 지원

### 가디언(Guardian)
휴먼 프라임(Human Prime)이 사망하거나 iFay를 관리할 수 없을 때, iFay 관리권을 인수하도록 지정된 자연인입니다. 참고: "상속인"이라는 용어는 사용하지 않습니다. 가디언(Guardian)은 니모닉 활성화 또는 휴먼 프라임(Human Prime)이 사전 지정한 신원 인증 방식으로 신원 확인을 완료합니다.

### 후견(Guardianship)
휴먼 프라임(Human Prime)이 iFay의 관리권을 가디언(Guardian)에게 이전하는 과정과 상태입니다. 후견권 이전이 완료되면, iFay는 새로운 가디언(Guardian)과 바인딩되고, Faying 프로토콜의 페어링 관계가 업데이트됩니다.

### 디지털 묘원(Digital Cemetery)
휴먼 프라임(Human Prime) 사후, iFay가 독립 신원으로 계속 실행되는 전용 샌드박스 환경입니다. 디지털 묘원은 iFay의 행동 범위를 제한하여, iFay가 휴먼 프라임(Human Prime)이 사전 설정한 작업을 계속 실행하여 혼란과 손실을 초래하는 것을 방지하고, 안전성과 격리성을 보장합니다.

## 범용 개념 명확화

### 컨텍스트(Context)
Fay가 처한 외부 환경으로, 통신 대상, 상호작용 모드, 전달되는 메시지, 메시지가 표현하는 의미, 제어 가능한 하드웨어/소프트웨어, 호출 가능한 리소스와 스킬 등을 포함합니다.

### 프로토콜(Protocol)
데이터 저장, 메시지 본문, 파라미터, 인터페이스 구조의 주석 정의와 특정 목적을 위한 상호작용 흐름 약정을 포함합니다.

### 크리덴셜(Credentials)
광의적 개념으로, 사람, 사물을 유일하게 식별하는 모든 디지털 식별자를 가리키며, 변조하거나 부인할 수 없습니다. FayID, 계정 비밀번호, 인증서, 인가, 액세스 토큰, 스마트 컨트랙트, 디지털 토큰(MeriToken)을 포함합니다.

### 휴먼 프라임(Human Prime)
iFay가 인스턴시에이트(복제)하는 자연인 사용자입니다. iFay는 휴먼 프라임(Human Prime)과 깊이 바인딩되어, 휴먼 프라임(Human Prime)의 인격 특성을 인스턴시에이트(복제)하고 디지털 능력을 융합합니다.

### 인스턴시에이트(복제)(Instantiate)
휴먼 프라임(Human Prime)으로부터 정렬된 iFay를 생성하는 과정입니다. 인스턴시에이트(복제)는 단순한 데이터 복사가 아니라, 휴먼 프라임(Human Prime)의 성격, 데이터, 권한을 구조화하여 iFay에 주입하는 과정입니다.

### 인해빗(빙의)(Inhabit)
iFay를 특정 단말 하드웨어나 소프트웨어 애플리케이션에 주입하여, iFay가 해당 단말을 제어할 수 있게 하는 과정입니다. 인해빗(빙의) 후, 단말은 iFay의 "팔다리"가 되며, iFay는 CAP(Control Authority Protocol)를 통해 단말의 하드웨어와 소프트웨어 기능을 직접 제어할 수 있습니다.

### 호스트(Host)
iFay에 의해 인해빗(빙의)되어 제어되는 단말 하드웨어 또는 소프트웨어 애플리케이션입니다. 참고: 호스트는 단말 디바이스나 애플리케이션을 가리키며, 휴먼 프라임(Human Prime)이 아닙니다.

### Faying
iFay와 휴먼 프라임(Human Prime) 또는 단말이 연결을 수립하는 과정과 상태로, 블루투스 페어링과 유사합니다. iFay가 휴먼 프라임(Human Prime)과 Faying 상태일 때, iFay는 활성화를 유지합니다. Faying은 Faying 프로토콜이 규정한 보안 페어링 조건을 준수해야 하며, iFay가 휴먼 프라임(Human Prime)의 명확한 의도 하에서만 인가되어 실행되도록 보장합니다.

### Separating
iFay와 휴먼 프라임(Human Prime)이 연결을 끊고 휴면에 들어가는 과정입니다. iFay가 휴먼 프라임(Human Prime)과 Separating 상태일 때, iFay는 휴면 상태로 전환되어 현재 상태 스냅샷을 저장하고 분리를 완료합니다.

## 모듈 명칭

iFay는 4레이어 아키텍처를 채택하며, 총 12개의 핵심 모듈을 포함합니다:

### 소셜 레이어(Social Layer)

| 모듈 | 설명 |
|------|------|
| **크리덴셜 관리(Credentials Management)** | 7가지 크리덴셜 유형(FayID, 계정 비밀번호, 인증서, 인가, 액세스 토큰, 스마트 컨트랙트, 디지털 토큰)을 관리하며, 휴먼 프라임(Human Prime) 크리덴셜의 안전한 위임과 사본 메커니즘을 지원 |

### 인터랙션 레이어 — 감지(Interaction Layer — Perception)

| 모듈 | 설명 |
|------|------|
| **일인칭 추적기(First-person Tracer)** | 휴먼 프라임(Human Prime)의 일인칭 시점을 시뮬레이션하여, 화면이나 인터페이스의 시각 및 청각 정보를 캡처하며, 구조화된 문서 파싱보다 시각 감지를 우선 사용 |
| **센서(Sensor)** | iFay의 광의적 신경계로, CAP(Control Authority Protocol)와 DTP(Data Tunnel Protocol)를 기반으로 단말 디바이스 센서를 브릿지하며, 동적 감도 조절을 지원 |
| **자기인식(Self-awareness)** | 내부를 모니터링하여 휴먼 프라임(Human Prime)의 반응으로부터 의도를 추론하는 모듈로, 숙련된 비서가 상사의 표정을 읽는 능력과 유사 |

### 인터랙션 레이어 — 동작(Interaction Layer — Action)

| 모듈 | 설명 |
|------|------|
| **시뮬레이션 조작(Simulated Operation)** | 인간의 UI 상호작용 조작(클릭, 드래그, 스크롤, 제스처 등)을 시뮬레이션하며, 일인칭 추적기 피드백을 통해 자적응적으로 인터페이스를 탐색 |
| **스킬 호출(Invoke Skill)** | 등록된 스킬을 직접 트리거하거나 특정 작업을 실행하는 동작 모듈로, 함수 호출이나 API 호출과 유사 |
| **자율 행동(Self-driven Behavior)** | 자율적으로 트리거되는 행동 모듈로, 정시 작업, 자기인식 추론, 영구 스킬 세 가지 트리거 소스를 지원하며, "동작→피드백→재동작" 순환을 형성 |

### 코그니션 레이어 — 사고(Cognition Layer — Thinking)

| 모듈 | 설명 |
|------|------|
| **개인 데이터 힙(Personal Data Heap)** | iFay의 모든 프라이빗 데이터를 통합 관리하며, 다양한 저장 형식과 위치를 지원하고, 내부 모듈에 통합 읽기/쓰기 인터페이스를 제공 |
| **외부 지식(External Knowledge)** | 외부 지식 베이스와 모델의 접속 모듈로, 외부 지능을 스킬 유형으로 접속하며, SSP(Skill Sharing Protocol)를 통해 개방 서비스에 접근 |
| **정렬 의식(Aligned Consciousness)** | 휴먼 프라임(Human Prime) 개인 프로필의 완전한 기술로, 데이터 마이닝, 자기인식 실시간 조정, 휴먼 프라임(Human Prime) 수동 정의 세 가지 방식으로 구축 및 업데이트를 지원 |

### 코그니션 레이어 — 스킬(Cognition Layer — Skills)

| 모듈 | 설명 |
|------|------|
| **디바이스 드라이버 허브(Device Driver Hub)** | 디바이스 드라이버의 허브 레이어로, 표준화된 인터페이스를 통해 새 드라이버를 통합하며, 새 드라이버 통합 시 내부 아키텍처 안정성을 보장 |
| **등록 스킬(Registered Skill)** | iFay가 이미 습득했거나 사용 능력을 획득한 스킬로, 6가지 스킬 유형(API, Workflow, Bot, Agent, APP, Microservice)을 지원 |
| **내부 스킬(Internal Skill)** | 휴먼 프라임(Human Prime)의 개성과 정렬된 고유 능력 모듈로, 내성 메커니즘을 제공하여 외부 지식이 휴먼 프라임(Human Prime)의 의도와 충돌하지 않도록 보장 |

### 에고 레이어(Ego Layer)

| 모듈 | 설명 |
|------|------|
| **Ego 모델** | iFay의 개인화된 마이크로 모델로, 플러그인 방식의 전환 가능한 엣지 소형 모델이며, 외부 대형 모델과 독립적으로 실행되어 개성 안정성을 보장 |

## 프로토콜 명칭

iFay 프로토콜 체계는 6가지 핵심 프로토콜을 포함합니다:

| 프로토콜 | 설명 |
|------|------|
| **Faying 프로토콜** | 자연인과 iFay의 안전한 페어링 및 활성화 조건을 규정하는 프로토콜로, iFay가 휴먼 프라임(Human Prime)의 명확한 의도 하에서만 인가되어 실행되도록 보장 |
| **인터랙티브 대화 프로토콜(Interactive Conversation Protocol)** | 인간 UI를 위한 모듈화 멀티모달 시맨틱 프로토콜로, 시맨틱 콘텐츠를 모듈화하고 멀티모달화하여 클라이언트가 읽기 쉬운 사용자 친화적 메시지 표시를 재구성할 수 있게 함 |
| **텔레파시 프로토콜(Telepathy Protocol)** | Fay 간 UI 번역 레이어를 제거한 시맨틱 통신 프로토콜(기술적으로 시맨틱 벡터 직접 통신, Semantic Direct Protocol)로, 의미와 의도가 Fay 간에 직접 전송되며, 약정된 벡터 인코딩 토큰으로 구조화된 텍스트를 대체 |
| **CAP(Control Authority Protocol)** | Control Authority Protocol, iFay가 단말 하드웨어와 특정 소프트웨어를 접수하여 드라이버, 로컬 인터페이스, 명령을 직접 호출하는 데 사용 |
| **DTP(Data Tunnel Protocol)** | Data Tunnel Protocol, 단말과 iFay 간의 양방향 전송 프로토콜. 단말→iFay 방향은 영구 사용자 데이터 저장과 데이터 보호를 지원하고, iFay→단말 방향은 데이터 풍부화와 개인화 처리를 지원 |
| **SSP(Skill Sharing Protocol)** | Skill Sharing Protocol, 원래 클라이언트에만 개방된 서비스와 인터페이스를 표준화된 원격 프로토콜을 통해 전체 네트워크에 개방 |

## iFay Profile 차원

iFay Profile은 6개 차원을 포함하며, 그 중 4개 Faying 차원은 iFay의 능력과 리소스를 기술합니다:

### iFay 신원
FayID + 메타 정보, iFay의 기본 신원 식별입니다.

### Ego 모델
현재 활성 Ego 버전 참조로, 어느 시점에서든 하나의 활성 버전만 있습니다.

### Faying 사고
iFay의 인지 리소스 차원으로, 4개 하위 유형을 포함합니다:
- **Content**(콘텐츠) — 구조화 및 비구조화 콘텐츠 자산
- **Data**(데이터) — 휴먼 프라임(Human Prime)의 개인 데이터
- **Knowledge Base**(지식 베이스) — 조직화된 지식 체계
- **Info Feed**(정보 피드) — 실시간 정보 구독 및 푸시

### Faying 스킬
iFay의 능력 차원으로, 6가지 스킬 유형을 포함합니다:
- **API** — 표준화된 인터페이스 호출
- **Workflow** — 워크플로 편성
- **Bot** — 대화 로봇
- **Agent** — 지능형 에이전트
- **APP** — 애플리케이션
- **Microservice** — 마이크로서비스

### Faying 하드웨어
iFay의 하드웨어 차원으로, iFay의 "팔다리"입니다. iFay 마이그레이션 시 단말이 사용 가능한 하드웨어를 스캔하고 연결을 시도합니다. 3개 하위 유형을 포함합니다:
- **Device**(디바이스) — 단말 하드웨어 디바이스
- **Storage**(스토리지) — 데이터 저장 리소스
- **Computing**(컴퓨팅) — 컴퓨팅 리소스

### Faying 권한
iFay의 권한 차원으로, iFay의 "관계"입니다. SSO, OAuth, Fingerprint(통칭, 모든 식별 가능한 신원 표식을 가리킴) 등 확장 가능한 인증 방식을 지원합니다. 권한 생명주기는 세 가지 유형으로 나뉩니다: 선천적(inherent), 영구(persistent), 임시(ephemeral).
