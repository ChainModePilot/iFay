# 10.2 Invoke Skill

## One-Line Definition

Invoke Skill is iFay's **mouth and telephone** — if Simulated Operation is iFay using its hands to operate interfaces, Invoke Skill is iFay directly "calling" a service — no interface needed, directly calling APIs or triggering tasks.

## Why It's Needed

Imagine you need to translate a document. You have two options:

**Option 1**: Open a browser, go to a translation website, paste the text, wait for translation, copy the result. This is the "Simulated Operation" approach — operating through the interface step by step.

**Option 2**: Directly call the translation company's phone, say "translate this passage for me," and they tell you the result directly. No website to open, no interface to operate. This is the "Invoke Skill" approach — **directly calling the service**.

Why is the second option needed? Because in many cases, operating an interface is an unnecessary intermediary. What you really want isn't "open a translation website" but "translate this passage into English." Invoke Skill lets iFay skip the interface and directly accomplish your goal.

This is like division of labor in human society: you don't need to personally go to the bakery and knead dough (operate the interface) — you just make a phone call saying "deliver a loaf of bread" (invoke a skill). Invoke Skill enables iFay to efficiently leverage all registered services — APIs, workflows, Bots, Agents, Apps, microservices — like an assistant who's great at getting things done over the phone.

## Its Position in the Architecture

```
iFay Four-Layer Architecture
├── Social Layer
├── Interaction Layer          ← Invoke Skill is here
│   ├── Perception (Sense)
│   │   └── ...
│   └── Action
│       ├── Simulated Operation  ← Hands: operates interfaces
│       ├── 👉 Invoke Skill      ← Mouth: directly "calls" services
│       └── Self-driven Behavior ← Will: decides when to act
├── Cognition Layer
│   └── Skills
│       └── Registered Skill     ← Invoke Skill can only call registered skills
└── Ego Layer
```

Invoke Skill sits in the **Interaction Layer's Action subsystem**, alongside Simulated Operation and Self-driven Behavior. Its prerequisite is that the skill being called must already be registered — unregistered skills cannot be invoked.

## How It Works

Invoke Skill's workflow follows a clear chain: **Intent → Match → Call → Record**.

**1. Intent** — iFay receives a task intent (from the Human Prime's instruction, Self-driven Behavior's trigger, or another module's request).

**2. Match** — iFay matches the intent against the registered skill inventory to find the most suitable skill to call.

**3. Call** — iFay invokes the matched skill. Since the skill was pre-authorized during registration, the call executes immediately with zero delay.

**4. Record** — The invocation and its result are recorded. If the iFay ecosystem's GMChain is active, the contribution is also logged for MeriToken settlement.

## Scenario Stories

### Scenario 1: Instant Translation Without Opening Any App

You receive a Japanese email. You tell iFay: "Translate this email for me."

iFay doesn't open a browser or navigate to a translation website. Instead, it directly invokes the registered translation API skill — the one that was pre-authorized during registration. Two seconds later, the translation result is presented to you. No pop-ups asking for passwords, no spinning authentication wheels — everything was prepared during registration.

### Scenario 2: Multi-Skill Orchestration for a Complex Task

You say: "Help me prepare for tomorrow's client meeting."

iFay orchestrates multiple skill invocations:
1. Invokes the calendar API to retrieve tomorrow's meeting details
2. Invokes the email API to find recent correspondence with this client
3. Invokes a document generation workflow to compile a meeting brief
4. Invokes the office services API to book the meeting room and order refreshments

Each invocation is direct and immediate — no interfaces to navigate, no forms to fill. The entire preparation is completed in under a minute.

## For Developers

Invoke Skill belongs to **Phase 2 (Direct Client Takeover)** as a core module.

- **Requirement ID**: Requirement 10 (Invoke Skill)
- **Interface Specification**: `InvokeSkillService` interface with core methods: `invoke()` (invoke skill), `matchIntent()` (match intent to skill), and `getInvocationLog()` (invocation log)
- **Prerequisite**: Skills must be registered via `RegisteredSkillManager` before they can be invoked
- **Related Modules**: Registered Skill (`RegisteredSkillManager`) provides the skill inventory; Credentials Management (`CredentialManager`) provides authentication
- **Conformance Testing**: iFACTS L1 verifies intent matching and skill invocation capability; L2 verifies interface integration with Registered Skill module; L3 verifies the complete "intent expression → skill invocation → contribution recording" chain
