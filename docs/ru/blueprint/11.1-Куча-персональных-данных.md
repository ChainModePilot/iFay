# 11.1 Куча персональных данных

## Определение в одну строку

Куча персональных данных — это **память** iFay — ваши фотографии, логи чатов, рабочие документы, данные о здоровье… разбросанные по телефонам, компьютерам и облачным хранилищам. Куча персональных данных унифицирует их управление, позволяя iFay «вспомнить» любой момент из прошлого в любое время.

## Зачем это нужно

Ваша цифровая жизнь разбросана: фотографии в галерее телефона, рабочие документы на жёстком диске компьютера, логи чатов в мессенджерах, данные о здоровье в приложении умных часов. Куча персональных данных решает это: **данные остаются на месте, но iFay читает и пишет их все через единый интерфейс**.

## Его место в архитектуре

```
Четырёхслойная архитектура iFay
├── Социальный слой
├── Слой взаимодействия
│   ├── Восприятие
│   │   ├── Сенсор              ← Данные сенсоров поступают в Кучу
│   │   └── ...
│   └── Действие
├── Когнитивный слой          ← Куча персональных данных здесь
│   ├── Мышление
│   │   ├── 👉 Куча персональных данных  ← Память: унифицированное управление всеми приватными данными
│   │   ├── Внешние знания               ← Внешние знания тоже хранятся в Куче
│   │   └── Согласованное сознание       ← Анализирует данные из Кучи для профиля Human Prime
│   └── Навыки
└── Слой Ego
```

## Как это работает

**1. Унифицированный** — Один интерфейс для всех данных, независимо от места хранения.

**2. Классифицированный** — Четыре типа данных: Контент (фотографии, видео, статьи), Данные (здоровье, финансы, местоположение), База знаний (профессиональные заметки, учебные материалы), Информационная лента (новости, соцсети).

**3. Постоянный** — Через DTP (Протокол туннелирования данных) обеспечивается постоянное хранение и опека данных.

**4. Обогащённый** — Куча не просто пассивно хранит данные — она может выполнять обогащение и персонализацию (корреляция данных здоровья, семантические теги к фотографиям).

## Связи с другими модулями

| Связанный модуль | Связь | Аналогия с телом |
|-----------------|-------|------------------|
| **Сенсор** | Данные среды от сенсоров хранятся в Куче | Сенсорные сигналы → хранение памяти |
| **Внешние знания** | Информация от Внешних знаний интегрируется в Кучу | Знания из библиотеки → становятся вашей памятью |
| **Согласованное сознание** | Анализирует данные из Кучи для построения профиля Human Prime | Обобщение «какой я человек» из воспоминаний |
| **Автономное поведение** | Извлекает информацию из Кучи для решения о проактивных действиях | Решение «что делать сейчас» на основе памяти |
| **Протокол DTP** | Двунаправленная передача данных между терминалами и Кучей | Каналы ввода и вывода памяти |

## Сценарии

### Сценарий 1: Поиск ресторана, который вы посетили три года назад

Вы помните, что ходили в ресторан три года назад, но забыли название. iFay ищет в Куче: историю местоположений, фотографии, логи чатов — и находит «Ханчжоу Флейвор · Филиал Сиху» через перекрёстную валидацию данных из разных источников.

### Сценарий 2: Обогащённые инсайты из данных о здоровье

iFay коррелирует данные умных часов (пульс, сон, шаги) с данными календаря и привычками использования телефона, генерируя персонализированные инсайты о здоровье и предлагая корректировки режима.

## Для разработчиков

Куча персональных данных относится к **Фазе 2 (Прямой захват клиента)** как основной модуль.

- **ID требования**: Требование 11 (Куча персональных данных)
- **Спецификация интерфейса**: Интерфейс `PersonalDataHeap` с четырьмя основными методами: `read()`, `write()`, `enrich()`, `persist()`
- **Четыре типа данных**: Content, Data, Knowledge Base, Info Feed
- **Поддерживаемые места хранения**: `runtime_memory`, `cloud`, `vector_db`, `local`
- **Связанный протокол**: DTP для двунаправленной передачи данных и опеки данных
- **Тестирование соответствия**: iFACTS L1 верифицирует чтение/запись для всех четырёх типов; L2 верифицирует интерфейсы данных с Сенсором и Внешними знаниями; L3 верифицирует полную цепочку «сбор → хранение → обогащение → извлечение»
