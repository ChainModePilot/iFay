# 10.3 Автономное поведение

## Определение в одну строку

Автономное поведение — это **автономная воля** iFay — Имитация операций — это руки, Вызов навыков — это рот, а Автономное поведение — это iFay, самостоятельно решающий «когда действовать и когда говорить». Оно превращает iFay из пассивного исполнителя в проактивного деятеля.

## Зачем это нужно

По-настоящему хороший помощник **проактивно** делает вещи: автоматически готовит еженедельную сводку каждый понедельник утром, замечает ваше хмурое выражение и проактивно спрашивает о трудностях, всегда помнит проверить логистику при получении уведомления о доставке.

Важный механизм безопасности: если автономное действие iFay оказывается неверным, он **приостанавливается и спрашивает вас**.

## Его место в архитектуре

```
Четырёхслойная архитектура iFay
├── Социальный слой
├── Слой взаимодействия          ← Автономное поведение здесь
│   ├── Восприятие (Sense)
│   │   └── Самосознание    ← Выводит ваше намерение ──┐
│   └── Действие                                        │
│       ├── Имитация операций  ← Руки                   │
│       ├── Вызов навыков      ← Рот                    │
│       └── 👉 Автономное поведение ← Воля ←────────────┘
├── Когнитивный слой
│   └── Навыки
│       ├── Зарегистрированные навыки  ← Источник постоянных навыков
│       └── Внутренние навыки          ← Источник постоянных навыков
└── Слой Ego
```

## Как это работает

Автономное поведение имеет **три типа триггеров**:

**1. Запланированные задачи — Делать, когда придёт время**

Простейший триггер, как будильник. «Генерировать еженедельный отчёт каждый понедельник утром в 9:00.»

**2. Выводы самосознания — Почувствовать потребность, проактивно действовать**

Наиболее «интеллектуальный» триггер. iFay наблюдает за вашим состоянием через модуль Самосознания и проактивно действует.

**3. Постоянные навыки — Привычки, работающие непрерывно в фоне**

Некоторые навыки не одноразовые — они работают непрерывно. Например, «мониторить эту акцию и оповестить, если упадёт ниже 50».

**Цикл Действие → Обратная связь → Повторное действие**

Независимо от типа триггера, Автономное поведение следует основному циклу: действие → наблюдение результатов → корректировка стратегии или приостановка для запроса.

**Предохранительный клапан: Приостановка и подтверждение**

Когда iFay обнаруживает, что результаты автономного поведения не соответствуют вашему намерению, он **приостанавливает все последующие автономные действия** и спрашивает вас.

## Связи с другими модулями

| Связанный модуль | Связь | Аналогия с телом |
|-----------------|-------|------------------|
| **Самосознание** | Выводы самосознания запускают Автономное поведение | Внутренние чувства → побуждение к действию |
| **Имитация операций** | Решает «что делать», Имитация операций обрабатывает «делать руками» | Воля → руки |
| **Вызов навыков** | Решает «что делать», Вызов навыков обрабатывает «делать по телефону» | Воля → рот |
| **Зарегистрированные навыки** | Постоянные навыки — один источник триггеров | Долгосрочные привычки побуждают к действию |
| **Внутренние навыки** | Внутренние навыки — другой источник триггеров | Инстинкт побуждает к действию |
| **Согласованное сознание** | Предоставляет профиль Human Prime, помогая судить «стоит ли это делать» | Понимание вас определяет границы действий |

## Сценарии

### Сценарий 1: Автогенерация еженедельного отчёта (триггер запланированной задачи)

Понедельник утро 8:55, вы ещё в метро. Запланированная задача iFay срабатывает: извлекает данные за неделю, генерирует отчёт, проверяет содержание. К 9:00 отчёт готов на вашем рабочем столе с предложениями.

### Сценарий 2: Проактивное пополнение продуктов (триггер самосознания)

Среда вечером, iFay обнаруживает через данные умного холодильника: осталась одна упаковка молока, 3 яйца, отсек для овощей почти пуст. Комбинируя ваши пищевые привычки и историю покупок, iFay проактивно формирует список покупок. Перед заказом приостанавливается — сумма превышает обычный бюджет — и спрашивает подтверждение.

## Для разработчиков

Автономное поведение относится к **Фазе 4 (iFay + coFay Полная персонификация)** как основной модуль.

- **ID требования**: Требование 14 (Автономное поведение)
- **Спецификация интерфейса**: Интерфейс `SelfDrivenBehavior` с четырьмя основными методами: `scheduleTask()`, `handleInference()`, `pauseAndConfirm()`, `getLoopStatus()`
- **Три источника триггеров**: `scheduled`, `self_awareness`, `registered_skill` / `internal_skill`
- **Основной механизм**: Непрерывный цикл действие → обратная связь → повторное действие; приостановка и запрос подтверждения при несоответствии результатов намерению Human Prime
- **Тестирование соответствия**: iFACTS L1 верифицирует отзывчивость на три источника триггеров; L2 верифицирует интерфейс с модулем Самосознания; L4 верифицирует ограничения безопасности автономного поведения (механизм приостановки и подтверждения)
