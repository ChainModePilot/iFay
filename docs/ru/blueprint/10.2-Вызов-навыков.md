# 10.2 Вызов навыков

## Определение в одну строку

Вызов навыков — это **рот и телефон** iFay — если Имитация операций — это iFay, использующий руки для управления интерфейсами, то Вызов навыков — это iFay, напрямую «звонящий» сервису — без интерфейса, прямой вызов API или запуск задач.

## Зачем это нужно

Во многих случаях управление интерфейсом — ненужный посредник. Что вам действительно нужно — не «открыть сайт перевода», а «перевести этот текст на английский». Вызов навыков позволяет iFay пропустить интерфейс и напрямую достичь вашей цели.

## Его место в архитектуре

```
Четырёхслойная архитектура iFay
├── Социальный слой
├── Слой взаимодействия          ← Вызов навыков здесь
│   ├── Восприятие (Sense)
│   └── Действие
│       ├── Имитация операций  ← Руки: управляет интерфейсами
│       ├── 👉 Вызов навыков   ← Рот: напрямую «звонит» сервисам
│       └── Автономное поведение ← Воля: решает, когда действовать
├── Когнитивный слой
│   └── Навыки
│       └── Зарегистрированные навыки  ← Вызов навыков может вызывать только зарегистрированные навыки
└── Слой Ego
```

Предварительное условие: вызываемый навык должен быть уже зарегистрирован — незарегистрированные навыки не могут быть вызваны.

## Как это работает

Рабочий процесс: **Намерение → Сопоставление → Вызов → Запись**.

1. **Намерение** — iFay получает намерение задачи
2. **Сопоставление** — iFay сопоставляет намерение с инвентарём зарегистрированных навыков
3. **Вызов** — iFay вызывает сопоставленный навык. Поскольку навык был предварительно авторизован при регистрации, вызов выполняется мгновенно с нулевой задержкой
4. **Запись** — Вызов и его результат записываются. Если GMChain активен, вклад также регистрируется для расчёта MeriToken

## Связи с другими модулями

| Связанный модуль | Связь | Аналогия с телом |
|-----------------|-------|------------------|
| **Зарегистрированные навыки** | Вызов навыков может вызывать только зарегистрированные навыки | Нужна лицензия для практики |
| **Имитация операций** | Оба в подсистеме Действия, но разные подходы | Телефонный звонок vs. личный визит |
| **Автономное поведение** | Решает «когда звонить», Вызов навыков обрабатывает фактический вызов | Воля решает → рот говорит |
| **Управление учётными данными** | Нуждается в учётных данных для аутентификации | Нужно показать документы |
| **GMChain** | Вызовы навыков могут быть записаны в GMChain как вклад | Рабочие записи для оценки производительности |

## Сценарии

### Сценарий 1: Мгновенный перевод без открытия приложений

Вы получаете письмо на японском. iFay напрямую вызывает зарегистрированный навык API перевода — предварительно авторизованный при регистрации. Две секунды спустя результат перевода представлен вам.

### Сценарий 2: Оркестрация нескольких навыков для сложной задачи

Вы говорите: «Помоги мне подготовиться к завтрашней встрече с клиентом.» iFay оркестрирует множество вызовов навыков: API календаря, API почты, рабочий процесс генерации документов, API офисных сервисов — всё за минуту.

## Для разработчиков

Вызов навыков относится к **Фазе 2 (Прямой захват клиента)** как основной модуль.

- **ID требования**: Требование 10 (Вызов навыков)
- **Спецификация интерфейса**: Интерфейс `InvokeSkillService` с основными методами: `invoke()`, `matchIntent()`, `getInvocationLog()`
- **Предварительное условие**: Навыки должны быть зарегистрированы через `RegisteredSkillManager`
- **Тестирование соответствия**: iFACTS L1 верифицирует сопоставление намерений и вызов навыков; L2 верифицирует интеграцию с модулем Зарегистрированных навыков; L3 верифицирует полную цепочку «выражение намерения → вызов навыка → запись вклада»
