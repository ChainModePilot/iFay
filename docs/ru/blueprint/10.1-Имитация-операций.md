# 10.1 Имитация операций

## Определение в одну строку

Имитация операций — это **руки** iFay — он может кликать, перетаскивать, прокручивать и печатать как человек, управляя любым существующим программным интерфейсом без необходимости каких-либо ИИ-специфичных модификаций ПО.

## Зачем это нужно

ПО, которое вы используете — банковские сайты, страховые системы, государственные платформы — все спроектированы для людей, и ни одно не предоставляет «вход только для ИИ». Имитация операций позволяет iFay помочь вам прямо сейчас, не дожидаясь, пока каждое ПО в мире будет перепроектировано для ИИ.

Но Имитация операций фундаментально отличается от традиционных «скриптов автоматизации» (RPA). RPA следует предварительно написанному скрипту шаг за шагом. Имитация операций iFay **управляется восприятием**: он «видит» экран через Трекер от первого лица, понимает текущее состояние интерфейса, затем решает, что делать дальше.

## Его место в архитектуре

```
Четырёхслойная архитектура iFay
├── Социальный слой
├── Слой взаимодействия          ← Имитация операций здесь
│   ├── Восприятие (Sense)
│   │   ├── Трекер от первого лица   ← Глаза: видит, что на экране
│   │   └── ...
│   └── Действие
│       ├── 👉 Имитация операций  ← Руки: клик, перетаскивание, ввод
│       ├── Вызов навыков         ← Рот: напрямую «звонит» сервисам
│       └── Автономное поведение  ← Воля: решает, когда действовать
├── Когнитивный слой
└── Слой Ego
```

## Как это работает

Основной рабочий процесс: **Воспринять, Действовать, Обратная связь**.

**1. Воспринять** — Сначала «увидеть» текущий интерфейс через Трекер от первого лица.

**2. Действовать** — Поддерживает все операции: клик, перетаскивание, прокрутка, ввод текста, жесты от края, мультитач-жесты.

**3. Обратная связь** — После каждой операции повторно «смотрит» на интерфейс для подтверждения успеха.

Этот цикл «действовать → наблюдать → корректировать» — фундаментальное отличие от скриптов RPA. При встрече с незнакомым интерфейсом iFay **исследует** как человек.

## Связи с другими модулями

| Связанный модуль | Связь | Аналогия с телом |
|-----------------|-------|------------------|
| **Трекер от первого лица** | «Глаза» Имитации операций — воспринимает состояние интерфейса | Руки ↔ Глаза (координация «рука-глаз») |
| **Вызов навыков** | Оба в подсистеме Действия, но разные подходы | Работа руками vs. телефонный звонок |
| **Автономное поведение** | Решает «когда действовать», Имитация операций обрабатывает «как действовать» | Воля решает → руки выполняют |
| **Модель Ego** | Ego ограничивает стиль поведения (скорость операций, уровень осторожности) | Личность влияет на способ действий |
| **Управление учётными данными** | Нуждается в учётных данных при входе | Руки берут ключи, чтобы открыть двери |

## Сценарии

### Сценарий 1: Заполнение сложной формы страхового возмещения

Вы говорите iFay: «Помоги мне заполнить заявление на страховое возмещение по автомобилю». iFay «видит» страницу входа, входит с делегированными учётными данными, находит форму, заполняет поля, загружает фотографии, адаптивно выбирает опции из обновлённых выпадающих меню и отправляет — всё через визуальное восприятие, а не предварительно написанный скрипт.

### Сценарий 2: Заказ еды в обновлённом приложении

iFay открывает приложение доставки еды, но интерфейс изменился после обновления. Вместо ошибки (как RPA-скрипт) iFay сканирует новый макет, находит перемещённую иконку поиска, адаптивно исследует интерфейс и завершает заказ.

## Для разработчиков

Имитация операций относится к **Фазе 1 (Имитация действий человека)** как основной модуль.

- **ID требования**: Требование 5 (Имитация операций)
- **Спецификация интерфейса**: Интерфейс `SimulatedOperation` с тремя основными методами: `execute()`, `explore()`, `getPostActionState()`
- **Поддерживаемые типы операций**: `click`, `drag`, `scroll`, `gesture`, `type`
- **Основной дизайн**: Управляется восприятием, не скриптами; замкнутый цикл «действовать → воспринять → скорректировать»; поддерживает адаптивное исследование незнакомых интерфейсов
- **Тестирование соответствия**: iFACTS L1 верифицирует способность выполнения каждого типа операций; L2 верифицирует координацию «рука-глаз» с Трекером от первого лица
- **Связь с iFay Ready**: Сертификация Bronze требует поддержки управления приложением через Имитацию операций
