# 5. Сценарии применения iFay

# Внедрение черт Prime в iFay

По определению стоит повторить, что личность iFay наследуется от его Human Prime (человеческого оригинала). Однако инстанциация (репликация) из Prime должна быть очень простой операцией, и одноразовая операция должна оставаться эффективной долгосрочно, чтобы iFay мог расти синхронно с Prime.

Мы рекомендуем быстро инициализировать iFay следующими методами:

- ***Авторизация данных Prime***: Включая фотографии, данные приложений, записи коммуникаций и т.д. Эти данные обрабатываются через серию шагов и хранятся в стандартном формате в базе данных Prime. Эти данные могут быть доступны напрямую только iFay. Это помогает iFay поддерживать долгосрочную синхронизацию с Human Prime. (Мы приветствуем производителей ОС и оборудования для совместной разработки стандартов типа интерпретатора, позволяющих iFay получать данные под системным контролем. Мы рассмотрим это в другом проекте под названием [Fayward](https://github.com/ChainModePilot/Fayward).)
- ***Импорт файлов***: Эти файлы включают фотографии, документы, заметки, логи чатов и т.д. Эта операция дополняет историю Prime, обеспечивая iFay прошлыми воспоминаниями до авторизации.
- ***Prime рассказывает iFay***: Это очень похоже на написание промпта для Agent. Может быть в форме разговора или опросника. Для снижения операционных затрат пользователя мы рекомендуем поговорить с iFay о вашем прошлом опыте, как с близким другом.

С точки зрения реализации должны быть выполнены три предварительных технологических условия:

1. ***Тесная интеграция с уровнем ОС***: Позволяющая чтение системной информации и приватных данных других приложений. Ссылаясь на управление авторизацией iOS и Android, сторонним приложениям может быть сложно достичь такого уровня доверия. Поэтому производителям ОС и оборудования необходимо совместно предоставить централизованные методы управления для приложений данных.
2. ***Открытый публичный формат пересылки данных***: В настоящее время операции пересылки между приложениями в основном выполняются в форме файлов. Например, обмен фотографиями и документами с другим приложением. Однако системе сложно поделиться персонализированными структурированными данными с другим приложением. Поэтому, если мы хотим быстро синхронизировать данные пользовательских приложений с iFay, необходим единый формат данных.
3. ***Стандартный исследовательский опросник***: Чат — это высоко дивергентная форма взаимодействия, и помогает ли содержание чата iFay действительно понять Human Prime, зависит от обсуждаемых тем. Чтобы помочь iFay сфокусировать темы чата и получить эффективные данные памяти, мы предоставим справочные опросники для ограничения области тем чата. Этот опросник может быть напрямую добавлен как промпт к LLM.

# Дополнение iFay внешними возможностями

После завершения внедрения черт Prime iFay уже обладает вашей личностью, воспоминаниями и базовыми разрешениями. Но на этом этапе iFay подобен человеку, только что прибывшему в новый город — знает, кто он, но ещё не знает, какие ресурсы город может предложить. Дополнение iFay внешними возможностями означает помощь в установлении связей с внешним миром.

## Регистрация навыков

Расширение возможностей iFay достигается через «Зарегистрированные навыки». Навыки — это предварительное условие для выполнения iFay любого действия — незарегистрированные навыки не могут быть вызваны. iFay поддерживает шесть типов навыков:

| Тип навыка | Описание | Примеры |
|-----------|----------|---------|
| **API** | Прямые вызовы интерфейсов внешних сервисов | API перевода, API запроса погоды, платёжный интерфейс |
| **Workflow** | Многошаговые автоматизированные процессы оркестрации | «Получить письмо → извлечь ключевую информацию → сгенерировать сводку → отправить в мессенджер» |
| **Bot** | Диалоговые автоматизированные программы | Бот службы поддержки, бот записи на приём |
| **Agent** | ИИ-агенты с автономным принятием решений | Agent код-ревью, Agent анализа данных |
| **APP** | Полноценные приложения | Приложение-календарь, приложение заметок, навигационное приложение |
| **Microservice** | Независимо развёрнутые бэкенд-сервисы | Сервис распознавания изображений, сервис преобразования речи в текст |

При регистрации навыка выполняется шаг предварительной авторизации, обеспечивающий отсутствие необходимости дополнительной аутентификации при последующих вызовах, снижая задержку. Например, когда вы регистрируете навык API перевода для iFay, iFay завершает привязку API Key и верификацию разрешений на этапе регистрации, так что каждый последующий запрос на перевод может быть выполнен мгновенно.

Когда iFay офлайн, ожидающие действия кэшируются и автоматически выполняются асинхронно при восстановлении связи — как написание писем в самолёте, которые автоматически отправляются при посадке.

## Доступ к внешним знаниям

Помимо навыков, iFay также может обращаться к внешним базам знаний и моделям, рассматривая их как особый тип навыка. Это позволяет iFay получать доступ к знаниям и интеллекту за пределами собственных возможностей Human Prime, как консультация с экспертом-советником.

Например, вы — программист, но ваш iFay может обращаться к медицинской базе знаний. Когда вы плохо себя чувствуете, iFay может объединить ваши персональные данные о здоровье с медицинскими знаниями для предоставления предварительных рекомендаций по здоровью — он не заменяет врача, а действует как знающий друг, помогающий вам сделать первоначальную оценку.

Внешние знания доступны через Протокол обмена навыками (SSP) для обращения к стандартизированным открытым сервисам и интерфейсам в сети. Когда внешние источники знаний недоступны, iFay деградирует до использования кэшированных знаний из Кучи персональных данных и уведомляет Когнитивный слой о текущем состоянии деградации.

# Методы взаимодействия человека с iFay

## Как активировать iFay

iFay — это не постоянно работающий фоновый сервис — он находится в активированном состоянии только когда вы явно его авторизуете. Этот дизайн гарантирует, что iFay не будет действовать автономно без вашего ведома.

### Процесс сопряжения Faying

Процесс активации iFay называется «Faying» (подключение), следуя протоколу Faying для безопасного сопряжения. Весь процесс аналогичен сопряжению Bluetooth, но с более высоким уровнем безопасности:

1. **Инициация сопряжения**: Вы отправляете запрос на подключение к iFay через терминальное устройство (телефон, компьютер и т.д.), предоставляя учётные данные идентичности.
2. **Верификация личности**: Система верифицирует вашу личность через Центр регистрации FayID, подтверждая, что вы являетесь законным Human Prime этого iFay.
3. **Многофакторная аутентификация**: Система возвращает вызов сопряжения, и вам нужно пройти многофакторную аутентификацию (например, биометрия + отпечаток устройства + парольная фраза).
4. **Подтверждение активации**: После прохождения аутентификации iFay переходит в активированное состояние «Faying» и начинает обслуживать вас.

Если кто-то попытается активировать вашего iFay без вашего явного намерения, система отклонит запрос на активацию и зарегистрирует событие — как если бы кто-то пытался разблокировать ваш телефон вашим отпечатком пальца, но безуспешно.

### Сценарий: Первая настройка и ежедневное переподключение

**Первая настройка**: Лили только что создала своего собственного iFay. Она открывает клиент iFay на телефоне, и система проводит её через назначение FayID, инициализацию Модели Ego и создание Профиля. Затем Лили рассказывает свои личностные черты iFay через разговор, авторизует фотографии и данные коммуникаций на телефоне и делегирует учётные данные для входа в часто используемые приложения. Весь процесс ощущается как глубокая послеобеденная беседа с новым другом.

**Ежедневное переподключение**: На следующее утро Лили берёт телефон, и клиент iFay обнаруживает её лицо и отпечаток устройства, автоматически завершая сопряжение Faying. iFay восстанавливается из гибернации в активированное состояние, загружает снимок состояния, сохранённый прошлой ночью, и бесшовно продолжает — как если бы ваш помощник ждал рядом с вами, пока вы проснётесь.

## Автономное осознание iFay

iFay — это не просто исполнитель, ожидающий команд. В Фазе 4 (Полная персонификация) iFay обладает возможностями самосознания и автономного поведения, способен проактивно наблюдать за вами и окружающей средой, выводить ваши чувства и намерения и предпринимать автономные действия.

### Модуль Самосознания

Модуль Самосознания — это способность iFay «читать обстановку». Он отслеживает реакции Human Prime, подобно опытному помощнику, читающему выражение лица работодателя. Когда модуль Самосознания выводит новое намерение от вас, он передаёт вывод модулю Автономного поведения и Когнитивному слою, запуская соответствующие действия.

### Автономное поведение

Автономное поведение iFay обеспечивается тремя источниками триггеров:

- **Запланированные задачи**: Автоматически выполняются по предустановленным временным планам, например, организация сводки вашего расписания каждое утро в 8:00.
- **Выводы самосознания**: Проактивная инициация задач на основе выводов о вашем текущем состоянии, например, проактивное напоминание об отдыхе при обнаружении повышенного уровня стресса.
- **Постоянные навыки**: Зарегистрированные навыки и внутренние навыки, работающие непрерывно, формирующие автономный цикл «действие → обратная связь → повторное действие».

Этот цикл «действие → обратная связь → повторное действие» — ядро автономной работы iFay. После выполнения действия iFay получает обратную связь через модуль Самосознания (ваши реакции, изменения среды), затем решает следующий шаг соответственно. Если результат выполнения не соответствует вашему намерению, iFay приостановит последующие автономные действия и запросит ваше подтверждение.

### Сценарий: iFay проактивно напоминает о встрече

В 14:00 вы сосредоточены на написании кода. Модуль Самосознания iFay обнаруживает через данные сенсоров, что ваш пульс немного повысился и скорость набора увеличилась — эти сигналы предполагают, что вы можете быть в напряжённом состоянии. Тем временем сканирование запланированных задач iFay обнаруживает важную презентацию для клиента в 15:00 в вашем календаре.

iFay делает комплексное суждение: вы спешите закончить работу и, вероятно, забыли о предстоящей встрече. Поэтому iFay через Протокол интерактивного диалога мягко напоминает вам способом, не нарушающим ваш поток: «У вас презентация для клиента в 15:00. Слайды готовы. Хотите, чтобы я заказал кофе для переговорной?»

Вы отвечаете «Конечно», и iFay немедленно вызывает зарегистрированные навыки офисных сервисов для завершения заказа кофе и отправляет материалы встречи на большой экран переговорной — весь процесс потребовал от вас лишь одно слово.

## Как выключить iFay

Процесс выключения iFay называется «Separating» (отключение), также следуя протоколу Faying.

### Процесс разделения

1. **Инициация разделения**: Вы отправляете запрос на разделение iFay (может быть голосовая команда, жест или кнопка клиента).
2. **Снимок состояния**: iFay сохраняет полный снимок текущего состояния, включая текущие задачи, контекстную информацию и временные данные.
3. **Переход в гибернацию**: iFay переключается из активированного состояния «Faying» в состояние гибернации «Separating», останавливая все автономные действия.
4. **Подтверждение разделения**: Система подтверждает завершение разделения.

В гибернации iFay не будет выполнять никаких автономных действий, но снимок состояния полностью сохраняется и может быть бесшовно восстановлен при следующей активации.

### Сценарий: Перед сном iFay переходит в гибернацию

В 23:00 вы готовитесь ко сну. Вы говорите телефону: «Спокойной ночи». iFay распознаёт это как сигнал разделения и начинает процесс разделения:

- Сохраняет весь контекст сегодняшних разговоров и прогресс по незавершённым задачам
- Отмечает элементы, о которых нужно напомнить вам завтра утром, как запланированные задачи
- Проверяет, есть ли срочные дела, требующие обработки во время вашего сна (если да, спрашивает вас перед разделением)
- Завершает снимок состояния и переходит в режим гибернации

На следующее утро вы берёте телефон, iFay реактивируется через протокол Faying, загружает вчерашний снимок состояния и сообщает вам: «Доброе утро. Я закончил черновик отчёта, о котором вы говорили вчера. Также ваш дневной рейс зарегистрирован — место 12A, у окна.»

# Социальные функции iFay

iFay — не остров. В Фазе 4 iFay обладает способностью общаться и сотрудничать с другими Fay (включая iFay других людей и публичные coFay).

## Межфайная коммуникация: Протокол Телепатии

Когда вашему iFay нужно общаться с другими Fay, они используют Протокол Телепатии — протокол семантической коммуникации, удаляющий слой трансляции UI. Традиционная межприложенческая коммуникация требует кодирования информации в текст, передачи, затем декодирования, с потенциальной потерей информации на каждом шаге. Протокол Телепатии использует согласованные векторно-кодированные токены для прямой передачи смысла и намерения, с эффективностью и точностью, значительно превышающими традиционные методы.

## Сотрудничество с coFay

coFay (Common Fay) — это цифровой аватар, выполняющий публичные роли, примерно эквивалентный текущей концепции Agent. Ваш iFay может обращаться за помощью к coFay, как вы обращаетесь к профессионалам в реальной жизни.

Ключевой механизм: Fay договариваются о ценах перед выполнением задач. Это обеспечивает отслеживаемость и оцениваемость вклада каждого сотрудничества, в конечном итоге записываемого в GMChain и рассчитываемого в MeriToken.

## Сценарий: Ваш iFay договаривается о бронировании с туристическим coFay

Вы говорите iFay: «Забронируй авиабилеты и отель в Токио на следующую неделю, бюджет до 8000 юаней.»

Ваш iFay немедленно связывается с профессиональным туристическим coFay через Протокол Телепатии. Вот их «разговор» (полностью прозрачный для вас — вы увидите только конечный результат):

1. **Передача намерения**: Ваш iFay кодирует ваши требования (направление, время, бюджет, предпочтения) как семантические векторы и отправляет их туристическому coFay.
2. **Переговоры о цене**: Туристический coFay выставляет 15μ (единиц MeriToken) за полный сервис бронирования. Ваш iFay, основываясь на предпочтениях расходов в Модели Ego, оценивает цену как разумную и принимает.
3. **Совместное выполнение**: Туристический coFay вызывает интерфейсы навыков SSP авиакомпаний и отелей для поиска оптимальных вариантов.
4. **Возврат результатов**: Туристический coFay возвращает три альтернативных плана вашему iFay.
5. **Персонализированная фильтрация**: Ваш iFay фильтрует лучший вариант на основе ваших предпочтений (место у окна, тихий отель, близко к метро).
6. **Представление вам**: iFay представляет рекомендованный план через Протокол интерактивного диалога в формате модульной карточки: «Нашёл отличную комбинацию — прямой рейс ANA, место 12A у окна, отель в 3 минутах ходьбы от станции Синдзюку, итого 7200 юаней. Подтвердить бронирование?»

Вы говорите «Бронируй», iFay завершает бронирование, GMChain записывает вклад этого сотрудничества, и туристический coFay зарабатывает 15μ.

# Безопасность

Дизайн безопасности iFay следует одному фундаментальному принципу: **Социальная этика приоритетнее всего**. Независимо от инструкций Human Prime, iFay не будет нарушать социальную этику и общественный порядок. На этой основе iFay защищает интересы Human Prime через множество уровней механизмов безопасности.

## Этика прежде всего

Каждое поведенческое решение iFay проходит через процесс проверки этического соответствия:

1. **Проверка социальной этики** (высший приоритет): Нарушает ли поведение социальную этику и общественный порядок? Если да, отказать в выполнении и объяснить причину Human Prime.
2. **Проверка согласованности с Prime**: Соответствует ли поведение ценностям и намерениям Human Prime? Если нет, в зависимости от серьёзности, либо приостановить и запросить подтверждение, либо позволить Модели Ego скорректировать перед выполнением.
3. **Проверка разрешений**: Имеет ли iFay разрешение на выполнение этого действия? Если нет, запросить авторизацию.

## Изоляция учётных данных

Когда вы делегируете учётные данные iFay, iFay не использует напрямую ваши оригинальные учётные данные. Система обменивает оригинальные учётные данные на соответствующие копии, и iFay использует копии для входа и аутентификации. Это означает, что даже если копия скомпрометирована, ваши оригинальные учётные данные остаются в безопасности. При обнаружении компрометации или аномального использования копии система немедленно отзывает копию и уведомляет вас.

## Защита конфиденциальности

Когда iFay авторизован для запроса приватных данных, он возвращает только булевые результаты (истина или ложь) без раскрытия конкретных приватных данных. Например, когда сервису нужно проверить, старше ли вы 18 лет, iFay ответит только «да» или «нет», не раскрывая вашу конкретную дату рождения.

## Стабильность Ego

Модель Ego работает независимо как встроенная микромодель, не подверженная обновлениям внешних больших моделей. Это предотвращает внезапные изменения личности iFay из-за обновлений внешних моделей или преднамеренного вмешательства — ваш iFay всегда будет тем «ним», которого вы знаете.

## Аудиторский след

Все критические операции записываются в защищённые от подделки аудиторские журналы, включая использование учётных данных, вызовы навыков, переходы состояний и изменения разрешений. Это обеспечивает отслеживаемость и аудируемость каждого действия.

## Сценарий: iFay отказывает в неэтичном запросе

Ваш коллега одалживает ваш компьютер и пытается использовать вашего iFay для просмотра внутренних файлов компании-конкурента.

Процесс проверки этического соответствия iFay немедленно активируется:

1. **Проверка социальной этики**: Доступ к внутренним файлам других может составлять корпоративный шпионаж, нарушая социальную этику.
2. **Верификация личности**: Протокол Faying обнаруживает, что отпечаток устройства и поведенческие паттерны текущего оператора не совпадают с Human Prime.
3. **Отказ в выполнении**: iFay отклоняет запрос и записывает аудиторский журнал.
4. **Уведомление Prime**: iFay отправляет вам уведомление: «Кто-то попытался получить доступ к конфиденциальной информации через ваше устройство. Запрос отклонён. Подробности в аудиторском журнале.»

На протяжении всего процесса iFay не раскрыл никакой информации о том, какими учётными данными или разрешениями вы обладаете — он просто ответил «запрос отклонён».

# Формирование экосистемы ИИ-индустрии

iFay — это не просто продукт, а открытая экосистема. Она предоставляет чёткие пути участия для трёх типов разработчиков и стимулирует процветание экосистемы через стандарты сертификации и экономические модели.

## Три роли разработчиков

| Роль | Ответственность | Аналогия |
|------|----------------|----------|
| **Разработчики open-source проекта** | Участвуют в разработке основных модулей и протоколов iFay, продвигая систему iFay | Аналогично контрибьюторам ядра Linux |
| **Разработчики приложений** | Внедряют поддержку iFay в свои продукты, позволяя продуктам управляться iFay, чтобы пользователи могли делегировать iFay использование продуктов | Аналогично разработке приложений, совместимых с Siri/Alexa |
| **Разработчики сервис-провайдеров** | Создают реализации iFay в рамках спецификации iFay, позволяя пользователям выбирать iFay от разных поставщиков | Аналогично разным поставщикам, реализующим стандарты браузеров |

## Сертификация iFay Ready

Для управления прикладными продуктами со стороны iFay они должны пройти сертификацию iFay Ready. Сертификация разделена на три уровня:

| Уровень | Требования | Значение |
|---------|-----------|----------|
| **Bronze** | Поддержка управления приложением через имитацию операций | iFay может управлять интерфейсом вашего приложения как человек |
| **Silver** | Поддержка прямого управления через протокол CAP + обмен данными через протокол DTP | iFay может обходить UI для прямого управления вашим приложением |
| **Gold** | Поддержка обмена навыками через протокол SSP + полная интеграция архитектуры C/F/S | Ваше приложение полностью интегрировано в экосистему iFay |

Сертификация интегрируется с тестовым фреймворком iFACTS — уровень Silver использует тесты L2 Соответствия интерфейсов для верификации, уровень Gold использует тесты L2 + L3 Интеграционного соответствия для верификации.

## FayManifest стимулирует рост экосистемы

Декларативная сборка FayManifest драматически снижает барьер разработки для iFay. Разработчикам не нужно понимать всю сложность системы — достаточно объявить необходимое подмножество компонентов в JSON-файле, и система автоматически дополнит инфраструктурные зависимости. Это означает:

- iFay, которому нужно только управлять дронами, объявляет лишь Хаб драйверов устройств, Сенсор, протокол CAP и протокол DTP
- iFay, сфокусированный на обработке документов, объявляет лишь Вызов навыков, Зарегистрированные навыки и связанные API
- Система автоматически дополняет FayID, среду выполнения FayGer, систему разрешений и другую необходимую инфраструктуру

## Экономическая модель GMChain и MeriToken

Поток ценности в экосистеме iFay реализуется через GMChain (Global Merit Chain) и MeriToken. GMChain отслеживает, измеряет и оценивает все вклады Fay, вознаграждая участников MeriToken. Типы вклада включают:

- Услуги по сборке информации
- Предоставление API
- Предоставление устройств
- Предоставление среды выполнения
- Другие идентифицируемые ценностные вводы

Приобретение MeriToken основано на создании социальной ценности, а не на потреблении вычислительной мощности для технической работы блокчейна. GMChain поддерживает направленную эмиссию, обеспеченную фиатной валютой, облигациями, сертификатами активов, золотом и другим обеспечением, взаимодействуя с реальными методами признания ценности.

## Сценарий: Создание iFay для управления дроном за выходные

Команда разработчиков стартапа по производству дронов хочет интегрировать возможности iFay в свой продукт. В пятницу днём тимлид открывает документацию FayManifest и пишет файл-декларацию:

```json
{
  "name": "drone-controller-ifay",
  "version": "1.0.0",
  "description": "Реализация iFay для управления дроном",
  "vendor": "SkyPilot Inc",
  "modules": [
    { "id": "device_driver_hub" },
    { "id": "sensor", "config": { "types": ["gps", "imu", "camera"] } },
    { "id": "invoke_skill" },
    { "id": "registered_skill" }
  ],
  "protocols": [
    { "id": "cap_protocol" },
    { "id": "dtp_protocol", "config": { "realtime": true } }
  ],
  "controlMode": "ego",
  "drivers": [
    { "name": "Flight Controller", "type": "device", "driverPackage": "@drone-drivers/fc-generic" }
  ],
  "ego": {
    "modelSource": "@ego-models/drone-pilot-v1",
    "scenarioTags": ["aerial_photography", "inspection"]
  }
}
```

Среда выполнения FayGer автоматически разбирает этот Manifest, дополняет FayID, систему разрешений, безопасность и этическое соответствие и другие инфраструктурные зависимости, собирая полный экземпляр iFay для управления дроном.

В субботу команда завершает адаптацию и тестирование драйвера контроллера полёта. В воскресенье они верифицируют соответствие каждого модуля спецификации через тесты iFACTS L1 Соответствия отдельного компонента.

К утру понедельника этот iFay для управления дроном уже может принимать голосовые команды пользователя, автономно планировать маршруты полёта, транслировать аэросъёмку в реальном времени и автоматически возвращаться домой при обнаружении низкого заряда батареи — весь цикл разработки занял менее выходных.
