# 4. Kommunikationskanäle

Die iFay-Community bietet mehrere Kommunikationskanäle, um den Zusammenarbeitsbedürfnissen in verschiedenen Szenarien gerecht zu werden.

## GitHub Discussions

GitHub Discussions ist die primäre asynchrone Diskussionsplattform der iFay-Community und eignet sich für:

- Öffentliche Diskussion von SEPs (Specification Enhancement Proposals)
- Vertiefte Erörterung technischer Fragen
- Funktionsanfragen und Designdiskussionen
- Tägliche Kommunikation innerhalb der Arbeitsgruppen

Wir empfehlen, wichtige technische Diskussionen auf GitHub Discussions zu führen, um sicherzustellen, dass Diskussionsinhalte nachvollziehbar und durchsuchbar sind.

## GitHub Issues

GitHub Issues wird für die konkrete Aufgabenverfolgung und Problemverwaltung verwendet:

- Fehlerberichte
- Konkrete Entwicklungsaufgaben
- Vorschläge zur Dokumentationsverbesserung
- iFACTS-Testfallprobleme

Bitte geben Sie beim Einreichen eines Issues detaillierte Reproduktionsschritte oder Kontextinformationen an, damit die Maintainer das Problem schnell lokalisieren und beheben können.

## Discord

Discord ist die Echtzeit-Kommunikationsplattform der iFay-Community. Jede Arbeitsgruppe hat ihren eigenen dedizierten Kanal:

- `#general`: Allgemeine Diskussion
- `#wg-fayid`: FayID-Arbeitsgruppe
- `#wg-ego`: Ego-Arbeitsgruppe
- `#wg-protocols`: Protokoll-Arbeitsgruppe
- `#wg-fayger`: FayGer-Arbeitsgruppe
- `#wg-ifacts`: iFACTS-Arbeitsgruppe
- `#wg-docs`: Dokumentations-Arbeitsgruppe
- `#wg-community`: Community-Arbeitsgruppe

Discord eignet sich für schnellen Austausch und informelle Diskussionen. Bitte beachten Sie jedoch, dass wichtige technische Entscheidungen auf GitHub dokumentiert werden sollten, um Transparenz und Nachvollziehbarkeit zu gewährleisten.

## Regelmäßige Meetings

- **Monatliches Core-Maintainer-Meeting**: Findet einmal im Monat statt, wird öffentlich durchgeführt und steht allen Community-Mitgliedern zur Beobachtung offen. Tagesordnungen und Protokolle werden auf GitHub veröffentlicht.
- **Arbeitsgruppen-Meetings**: Jede Arbeitsgruppe kann ihre eigene Meeting-Häufigkeit und ihr Format festlegen, wobei mindestens ein Synchronisationsmeeting pro Monat empfohlen wird.

## Mailingliste

Die Mailingliste wird für wichtige Ankündigungen und Benachrichtigungen verwendet:

- Benachrichtigungen über neue Versionsveröffentlichungen
- Benachrichtigungen über SEP-Statusänderungen
- Ankündigungen wichtiger Governance-Entscheidungen
- Benachrichtigungen über Community-Veranstaltungen

Durch das Abonnieren der Mailingliste stellen Sie sicher, dass Sie keine wichtigen Informationen verpassen.

## Verhaltenskodex

Alle Kommunikationskanäle folgen dem iFay-Verhaltenskodex. Wir sind bestrebt, eine freundliche, inklusive und professionelle Community-Umgebung zu fördern. Bitte wahren Sie in allen Interaktionen Respekt und Konstruktivität.

## Sprache

Englisch ist die primäre Arbeitssprache der iFay-Community, aber wir begrüßen die Teilnahme in anderen Sprachen. Wenn Sie sich mit Deutsch, Chinesisch oder einer anderen Sprache wohler fühlen, verwenden Sie diese bitte ohne Bedenken — Community-Mitglieder werden bei der Übersetzung helfen, um sicherzustellen, dass Ihre Ideen korrekt vermittelt werden.

Unser Ziel ist es sicherzustellen, dass Sprache kein Hindernis für die Teilnahme an der iFay-Community darstellt.
