# 4. 沟通渠道

iFay 社区提供多种沟通渠道，以满足不同场景下的协作需求。

## GitHub Discussions

GitHub Discussions 是 iFay 社区的主要异步讨论平台，适用于：

- SEP（规范增强提案）的公开讨论
- 技术问题的深入探讨
- 功能请求和设计方案讨论
- 各工作组的日常交流

我们鼓励将重要的技术讨论放在 GitHub Discussions 上进行，以确保讨论内容可追溯、可搜索。

## GitHub Issues

GitHub Issues 用于具体的任务跟踪和问题管理：

- Bug 报告
- 具体的开发任务
- 文档改进建议
- iFACTS 测试用例问题

提交 Issue 时，请尽量提供详细的复现步骤或上下文信息，以便维护者快速定位和处理。

## Discord

Discord 是 iFay 社区的实时沟通平台。各工作组拥有独立的频道：

- `#general`：通用讨论
- `#wg-fayid`：FayID 工作组
- `#wg-ego`：Ego 工作组
- `#wg-protocols`：协议工作组
- `#wg-fayger`：FayGer 工作组
- `#wg-ifacts`：iFACTS 工作组
- `#wg-docs`：文档工作组
- `#wg-community`：社区工作组

Discord 适合快速交流和非正式讨论。但请注意，重要的技术决策应在 GitHub 上记录，以确保透明和可追溯。

## 定期会议

- **核心维护者月度会议**：每月举行一次，公开进行，所有社区成员均可旁听。会议议程和纪要会在 GitHub 上发布。
- **工作组会议**：各工作组可自行安排会议频率和形式，建议至少每月一次同步进展。

## 邮件列表

邮件列表用于发布重要公告和通知：

- 新版本发布通知
- SEP 状态变更通知
- 重大治理决策公告
- 社区活动通知

订阅邮件列表可确保你不会错过重要信息。

## 行为准则

所有沟通渠道均遵循 iFay 行为准则。我们致力于营造一个友好、包容、专业的社区环境。请在所有互动中保持尊重和建设性。

## 语言

英语是 iFay 社区的主要工作语言，但我们欢迎使用其他语言参与讨论。如果你更习惯使用中文、德语或其他语言，请放心使用——社区成员会协助翻译，确保你的想法被准确传达。

我们的目标是让语言不成为参与 iFay 社区的障碍。
