# 2. 定义与概念

## 产品专有名词

### iFay
Individual Fay。附属于自然人（称为**人类原型，Human Prime，简称 Prime**）的数字化身，致力于在价值观、个性和习惯方面与其所附属的自然人（人类原型）保持对齐。iFay 是人类原型人格的复刻（Instantiate），融合了人类原型的人格特征与数字能力，旨在代替人类原型执行机械性、重复性、危险性劳动和繁琐辅助工作，提升人类原型的安全保障、健康和生活质量，并放大人类原型的社会价值。

### coFay
Common Fay，承担公共角色的数字化身，大致等同于当前 Agent 的概念。它在某一领域拥有通用知识和技能，可以通过对话为多人服务。coFay 是独立项目，iFay 规范仅在必要时引用。

### Fay
iFay 和 coFay 的统称，拟人化 AI 代理的通用术语。它可能是新型社会关系中的重要节点。

### FayID
分配给每个 Fay 的全局唯一持久标识符，人类可识别且机器可读。FayID 对 iFay 和 coFay 使用统一的标识符生成和管理机制，支持超过全球人口数量级的标识符容量。

### FayGer
为 Fay（iFay 和 coFay）提供虚拟运行时环境的容器，类似 Docker。这确保了 Fay 可以跨平台附身（Inhabit）于不同的硬软件，使硬软件具备人类原型的个人意志。FayGer 提供跨语言构建工具和跨平台运行容器，使任何 Fay（无论开发语言）都能被打包并执行。

### Ego 模型
iFay 的个性化微型模型，与特定个人的画像对齐。Ego 是可插拔、可切换的边缘小模型，可加载到任何终端设备上使该设备拥有人类原型个性。Ego 约束 iFay 在价值取向、兴趣偏好、习惯、认知边界、技能边界、权限边界、工作风格等维度与人类原型对齐，独立于外部大型模型运行，确保个性稳定性。支持多版本管理与人格切换，以适应人类原型在不同场景下的多面人格需求。

> ⚠️ **伦理约束**：Ego 版本切换必须遵循透明性原则。所有 Ego 版本必须共享同一套核心价值观，差异仅限于表达风格和交互偏好。切换时应在交互元数据中标注当前活跃的 Ego 版本标识，确保可审计。

### iFay Profile
iFay 的统一数据模型，语义可解释的属性表。用于人类和系统识别 iFay，以及 Fay 间相互识别，是 iFay 的完整"身份证"。包含六个维度：iFay 身份（FayID + 元信息）、Ego 模型（当前活跃 Ego 版本引用）、Faying 思维、Faying 技能、Faying 硬件、Faying 权限。人类原型画像（对齐意识）是 Profile 的子集和输入源。

### FayManifest
iFay 的声明式组装配置文件，类似 package.json。声明 iFay 实现所需的部件、协议、控制模式（指令控制/Ego 控制/自主控制）和配置。开发者只需声明"需要什么"而非"如何实现"，支持最小化声明——只声明一个部件子集即可上线，系统自动补充必需的基础设施依赖。

### GMChain
**公誉链**（Global Merit Chain，GMC），iFay 生态的社会贡献账本——一种通过记录和衡量可识别社会贡献来建立信誉和话语权的区块链基础设施。GMChain 贯穿 CPE+M 框架的所有层级，为人类、iFay、coFay 和服务提供商提供统一的贡献追踪和激励机制。与大多数区块链项目不同，GMChain 完全依赖社会价值的认定来积累积分，而非通过消耗算力完成计算任务。GMChain 不接受货币注资，MeriToken 不与法定货币兑换。属于路线图第五阶段的远期愿景组件，但接口定义在早期阶段即需预留。

### MeriToken
**美誉值**，GMChain 上的社会贡献度量单位。MeriToken 不是货币，而是信誉、话语权和协作激励的量化凭证。获取方式基于创造社会价值（如完成协作任务、提供 API 服务、贡献算力等），而非通过消耗算力完成区块链技术工作。MeriToken Ledger（美誉值账本）记录每个参与者的累计贡献、信誉评分和话语权权重。

### μ（MU）
Merit Unit，MeriToken 的最小计量单位。Fay 间协作时以 μ 为单位协商价格。

### iFACTS
iFay Architecture Conformance Test Suite，标准化合规性测试套件。覆盖协议交互、模块接口、安全行为等关键规范点，包含四个测试层级：L1 单部件合规、L2 接口合规、L3 集成合规、L4 行为合规。厂商实现必须通过 iFACTS 测试才能声称其产品"iFay 可用"。

### iFay Ready
应用产品被 iFay 操控所需满足的认证标准，分为三个等级：
- **Bronze**：支持 iFay 通过模拟操作（第一人称追踪器 + 模拟操作）操控应用
- **Silver**：支持 iFay 通过 CAP 协议直接控制应用，支持 DTP 协议数据交换
- **Gold**：支持 iFay 通过 SSP 协议共享技能，支持完整的 C/F/S 架构集成

### 监护人（Guardian）
人类原型过世或无法管理 iFay 时，被指定接管 iFay 管理权的自然人。注意：不使用"继承人"一词。监护人通过助记词激活或人类原型预先指定的身份认证方式完成身份验证。

### 监护（Guardianship）
人类原型将 iFay 的管理权转移给监护人的过程和状态。监护权转移完成后，iFay 与新的监护人绑定，并更新 Faying 协议的配对关系。

### 数字墓园（Digital Cemetery）
人类原型过世后，iFay 以独立身份继续运行的专属沙箱环境。数字墓园限制 iFay 的行为范围，防止 iFay 延续执行人类原型预设任务而造成混乱和损失，确保安全性和隔离性。

## 通用概念澄清

### 上下文（Context）
Fay 所处的外部环境，包括通信对象、交互模式、传达的消息、消息表达的含义、可控的硬软件、可调用的资源和技能等。

### 协议（Protocol）
包括数据存储、消息体、参数和接口结构的注释定义，以及针对特定目的的交互流程约定。

### 凭证（Credentials）
广义概念，指用于唯一标识人、事、物的任何数字标识符，不可篡改或否认。包括 FayID、账号密码、证书、授权、访问令牌、智能合约和数字代币（MeriToken）。

### 人类原型（Human Prime）
iFay 所复刻的自然人用户。iFay 与人类原型深度绑定，复刻人类原型的人格特征并融合数字能力。

### 复刻（Instantiate）
从人类原型生成与其对齐的 iFay 的过程。复刻不是简单的数据复制，而是将人类原型的性格、数据和权限结构化注入 iFay 的过程。

### 附身（Inhabit）
将 iFay 注入到某个终端硬件或软件应用上，以便 iFay 可以控制该终端的过程。附身后，终端成为 iFay 的"肢体"，iFay 可以通过 CAP 协议直接控制终端的硬件和软件功能。

### 宿主（Host）
被 iFay 附身（Inhabit）并控制的终端硬件或软件应用。注意：宿主指的是终端设备或应用，而非人类原型。

### Faying
iFay 与人类原型或终端建立连接的过程和状态，类似蓝牙配对。当 iFay 与人类原型处于 Faying 状态时，iFay 保持激活。Faying 需遵循 Faying 协议规定的安全配对条件，确保 iFay 仅在人类原型明确意图下被授权运行。

### Separating
iFay 与人类原型断开连接进入休眠的过程。当 iFay 与人类原型处于 Separating 状态时，iFay 切换为休眠状态，保存当前状态快照并完成分离。

## 模块名称

iFay 采用四层架构，共包含 12 个核心模块：

### 社交层（Social Layer）

| 模块 | 说明 |
|------|------|
| **凭证管理（Credentials Management）** | 管理七种凭证类型（FayID、账号密码、证书、授权、访问令牌、智能合约、数字代币），支持人类原型凭证的安全委托与副本机制 |

### 交互层 — 感知（Interaction Layer — Perception）

| 模块 | 说明 |
|------|------|
| **第一人称追踪器（First-person Tracer）** | 模拟人类原型第一人称视角，捕获屏幕或界面上的视觉和听觉信息，优先使用视觉感知而非解析结构化文档 |
| **传感器（Sensor）** | iFay 的广义神经系统，基于 CAP 和 DTP 协议桥接终端设备传感器，支持动态调节灵敏度 |
| **自我感知（Self-awareness）** | 向内监测人类原型反应以推断意图的模块，类似熟练助手解读老板面部表情的能力 |

### 交互层 — 动作（Interaction Layer — Action）

| 模块 | 说明 |
|------|------|
| **模拟操作（Simulated Operation）** | 模拟人类 UI 交互操作（点击、拖拽、滚动、手势等），通过第一人称追踪器反馈自适应探索界面 |
| **技能调用（Invoke Skill）** | 直接触发已注册技能或执行特定任务的动作模块，类似函数调用或 API 调用 |
| **自驱行为（Self-driven Behavior）** | 自主触发的行为模块，支持定时任务、自我感知推断和持久技能三种触发源，形成"动作→反馈→再动作"循环 |

### 认知层 — 思维（Cognition Layer — Thinking）

| 模块 | 说明 |
|------|------|
| **个人数据堆（Personal Data Heap）** | 统一管理 iFay 所有私有数据，支持多种存储格式和位置，向内部模块提供统一读写接口 |
| **外部知识（External Knowledge）** | 外部知识库和模型的接入模块，将外部智能作为技能类型接入，通过 SSP 协议访问开放服务 |
| **对齐意识（Aligned Consciousness）** | 人类原型个人画像的完整描述，支持从数据挖掘、自我感知实时调整和人类原型手动定义三种方式建立和更新 |

### 认知层 — 技能（Cognition Layer — Skills）

| 模块 | 说明 |
|------|------|
| **设备驱动中枢（Device Driver Hub）** | 设备驱动的中枢层，通过标准化接口集成新驱动，确保新驱动集成时内部架构稳定 |
| **注册技能（Registered Skill）** | iFay 已掌握或获得使用能力的技能，支持六种技能类型（API、Workflow、Bot、Agent、APP、Microservice） |
| **内部技能（Internal Skill）** | 与人类原型个性对齐的固有能力模块，提供内省机制，确保外部知识不与人类原型意图冲突 |

### 自我层（Ego Layer）

| 模块 | 说明 |
|------|------|
| **Ego 模型** | iFay 的个性化微型模型，可插拔、可切换的边缘小模型，独立于外部大型模型运行，确保个性稳定性 |

## 协议名称

iFay 协议体系包含六种核心协议：

| 协议 | 说明 |
|------|------|
| **Faying 协议** | 规定自然人与 iFay 安全配对及激活条件的协议，确保 iFay 仅在人类原型明确意图下被授权运行 |
| **交互对话协议（Interactive Conversation Protocol）** | 面向人类 UI 的模块化多模态语义协议，将语义内容模块化和多模态化，使客户端能重构易读的用户友好消息展示 |
| **心灵感应协议（Telepathy Protocol）** | Fay 间去除 UI 翻译层的语义通信协议（技术上即语义向量直接通信，Semantic Direct Protocol），允许意义和意图在 Fay 之间直接传输，使用约定的向量编码令牌替代结构化文本 |
| **控制权限协议（CAP）** | Control Authority Protocol，用于 iFay 接管终端硬件和特定软件，直接调用驱动程序、本地接口和命令 |
| **数据隧道协议（DTP）** | Data Tunnel Protocol，终端与 iFay 间的双向传输协议。终端→iFay 方向支持持久化用户数据存储和数据监护；iFay→终端方向支持数据丰富化和个性化处理 |
| **技能共享协议（SSP）** | Skill Sharing Protocol，将原本仅对客户端开放的服务和接口通过标准化远程协议向全网开放 |

## iFay Profile 维度

iFay Profile 包含六个维度，其中四个 Faying 维度描述 iFay 的能力和资源：

### iFay 身份
FayID + 元信息，iFay 的基础身份标识。

### Ego 模型
当前活跃 Ego 版本引用，任一时刻仅有一个活跃版本。

### Faying 思维
iFay 的认知资源维度，包含四个子类型：
- **Content**（内容）— 结构化和非结构化的内容资产
- **Data**（数据）— 人类原型的个人数据
- **Knowledge Base**（知识库）— 经过组织的知识体系
- **Info Feed**（信息流）— 实时信息订阅与推送

### Faying 技能
iFay 的能力维度，包含六种技能类型：
- **API** — 标准化接口调用
- **Workflow** — 工作流编排
- **Bot** — 对话机器人
- **Agent** — 智能代理
- **APP** — 应用程序
- **Microservice** — 微服务

### Faying 硬件
iFay 的硬件维度，是 iFay 的"肢体"。iFay 迁移时终端扫描并尝试连接可用硬件。包含三个子类型：
- **Device**（设备）— 终端硬件设备
- **Storage**（存储）— 数据存储资源
- **Computing**（算力）— 计算资源

### Faying 权限
iFay 的权限维度，是 iFay 的"关系"。支持 SSO、OAuth、Fingerprint（统称，指一切可识别身份的标识）等可扩展鉴权方式。权限生命周期分为三种类型：与生俱来（inherent）、持久（persistent）、临时（ephemeral）。
