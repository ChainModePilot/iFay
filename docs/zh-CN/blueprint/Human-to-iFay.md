# 从人类到 iFay

iFay 的个性不是凭空生成的——它继承自宿主。每一个 iFay 都是其宿主的人格延伸，而这种延伸需要一个结构化的注入过程。

本页描述宿主如何从三个维度将自身特征注入 iFay：**性格**塑造 Ego 模型、**数据**填充个人数据堆、**权限**建立凭证委托。三个维度缺一不可——性格让 iFay 知道"我是谁"，数据让 iFay 拥有"我的过去"，权限让 iFay 能够"替我行动"。

---

## 宿主性格（→ Ego 模型）

宿主的性格是 Ego 模型的训练和设定输入。Ego 模型是 iFay 的内嵌微型模型，它约束 iFay 在价值取向、兴趣偏好、习惯、认知边界、技能边界、权限边界和工作风格等维度与宿主保持对齐。

### 三种注入方式

| 方式 | 说明 |
|------|------|
| **对话叙述** | 宿主与 iFay 进行引导式对话，iFay 通过标准研究问卷约束对话主题范围，确保采集到有效的性格数据。这不是闲聊——每一轮对话都围绕特定的性格维度展开。 |
| **问卷填写** | 宿主直接填写结构化的性格问卷，系统将答案转化为 Ego 模型的训练参数。 |
| **直接配置** | 高级用户可以直接编辑 Ego 模型的约束参数，精确控制 iFay 的行为边界。 |

### 多版本 Ego

人是多面的。你在工作中严谨专注，在朋友面前轻松幽默，在家人身边温柔耐心。Ego 模型支持多个版本，每个版本对应宿主人格的一个侧面。任一时刻只有一个 Ego 版本处于活跃状态，支持手动切换和基于上下文的自动切换。

### 场景

> Isabel 刚刚激活了她的 iFay。系统引导她进入一段对话——不是随意的聊天，而是围绕标准研究问卷设计的引导式交流。
>
> "当你收到一封措辞模糊的工作邮件时，你倾向于直接回复澄清，还是先私下找对方沟通？"
>
> Isabel 选择了后者，并补充说她更喜欢在非正式场合解决潜在的误解。iFay 记录下这个偏好，将其编码为 Ego 模型中"沟通风格"维度的一个参数。
>
> 二十分钟后，Isabel 的 iFay 已经初步理解了她的沟通偏好、决策风格和风险态度。这些特征将成为 iFay 替她处理日常事务时的行为准则。

## 宿主数据（→ 个人数据堆）

数据是 iFay 的"记忆"。没有数据的 iFay 就像一个失忆的人——知道自己是谁，却不记得自己经历过什么。个人数据堆是 iFay 统一管理所有私有数据的组件，宿主数据是它的初始化输入。

### 四种数据类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **内容（Content）** | 结构化或非结构化的内容资产 | 照片、视频、文章、创作作品 |
| **数据（Data）** | 个人数据 | 应用使用记录、健康数据、消费记录 |
| **知识库（Knowledge Base）** | 组织化的知识体系 | 专业笔记、学习资料、工作文档 |
| **信息流（Info Feed）** | 实时信息订阅 | 新闻源、行业动态、社交媒体关注 |

### 三种注入方式

1. **授权宿主数据**：授权 iFay 访问宿主的照片、应用数据、通信记录等。系统与操作系统层紧密集成，支持在系统监督下读取系统信息和其他应用的私有数据。
2. **导入文件**：导入文档、笔记、聊天记录等长期记忆文件。
3. **指定信息源**：指定书籍、信息源和知识库，让 iFay 持续从这些来源获取信息。

所有数据以标准格式存储到个人数据堆中，仅可被 iFay 直接访问。这是一次性初始化操作，但支持长期增长——iFay 会随着宿主的生活持续积累新的数据。

### 场景

> 陈明决定把过去十年的数字生活交给他的 iFay。
>
> 他首先授权 iFay 访问手机相册——超过 30,000 张照片和视频，记录了从大学毕业到孩子出生的每一个重要时刻。然后他导入了微信聊天记录、工作邮件存档和 Notion 笔记库。
>
> 当 iFay 处理完这些数据后，它不仅知道陈明去年去了哪里旅行，还理解了他的社交圈、工作节奏和生活习惯。当陈明的同事问起上次团建的餐厅名字时，iFay 能在几秒内从照片的地理标签和聊天记录中找到答案——就像陈明自己回忆起来一样。

## 宿主权限（→ 凭证委托）

权限决定了 iFay 能"替你做什么"。没有权限的 iFay 就像一个没有钥匙的管家——知道该做什么，却打不开任何一扇门。

### 三种权限来源

iFay 的权限有三种来源，它们之间没有包含关系：

| 来源 | 说明 | 示例 |
|------|------|------|
| **宿主直接授权** | 宿主将已有凭证委托给 iFay 使用 | 委托 Netflix 账号、银行查询权限 |
| **宿主权限换取** | 由宿主的权限交换而来的衍生权限 | 宿主的企业账号换取的 API 访问权 |
| **iFay 自主获得** | iFay 通过自身行为获得的独立权限 | 通过 GMChain 贡献获得的 API 访问权、在 Fay 社交网络中建立的信任凭证 |

### 无包含关系

一个关键设计：宿主权限与 iFay 权限之间**没有包含关系**。iFay 可以拥有宿主没有的权限（比如 iFay 通过贡献获得的 API 访问权），宿主也可以拥有未委托给 iFay 的权限（比如宿主的私人银行账户）。

### 七种凭证类型

iFay 支持管理以下七种凭证类型：FayID、账号密码、证书、授权、访问令牌、智能合约、数字代币（MeriToken）。宿主委托凭证时，原始凭证会被交换为对应的副本凭证，iFay 使用副本进行登录和认证，确保原始凭证的安全。

### 场景

> 李薇把她的 Netflix 账号委托给了 iFay——这是**宿主直接授权**。iFay 收到的不是李薇的原始密码，而是一个对应的副本凭证。现在 iFay 可以根据李薇的观影偏好自动管理她的播放列表。
>
> 与此同时，李薇的 iFay 一直在通过 GMChain 为其他 Fay 提供信息组装服务，积累了可观的 MeriToken。凭借这些贡献，iFay 获得了一个高级天气数据 API 的访问权——这是**iFay 自主获得**的权限，李薇本人并不拥有这个 API 的访问权。
>
> 但李薇的私人投资账户，她选择不委托给 iFay。这完全没问题——宿主和 iFay 的权限边界是独立的。
