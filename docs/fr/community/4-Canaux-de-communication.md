# 4. Canaux de communication

La communauté iFay fournit plusieurs canaux de communication pour répondre aux besoins de collaboration dans différents scénarios.

## GitHub Discussions

GitHub Discussions est la principale plateforme de discussion asynchrone de la communauté iFay, adaptée pour :

- La discussion publique des SEPs (Specification Enhancement Proposals)
- L'exploration approfondie de questions techniques
- Les demandes de fonctionnalités et les discussions de conception
- La communication quotidienne au sein des groupes de travail

Nous encourageons les discussions techniques importantes à se dérouler sur GitHub Discussions pour garantir que le contenu des discussions soit traçable et consultable.

## GitHub Issues

GitHub Issues est utilisé pour le suivi de tâches spécifiques et la gestion des problèmes :

- Rapports de bogues
- Tâches de développement spécifiques
- Suggestions d'amélioration de la documentation
- Problèmes liés aux cas de test iFACTS

Lors de la soumission d'un Issue, veuillez fournir des étapes de reproduction détaillées ou des informations contextuelles pour aider les mainteneurs à localiser et résoudre rapidement le problème.

## Discord

Discord est la plateforme de communication en temps réel de la communauté iFay. Chaque groupe de travail dispose de son propre canal dédié :

- `#general` : Discussion générale
- `#wg-fayid` : Groupe de travail FayID
- `#wg-ego` : Groupe de travail Ego
- `#wg-protocols` : Groupe de travail Protocoles
- `#wg-fayger` : Groupe de travail FayGer
- `#wg-ifacts` : Groupe de travail iFACTS
- `#wg-docs` : Groupe de travail Documentation
- `#wg-community` : Groupe de travail Communauté

Discord est adapté aux échanges rapides et aux discussions informelles. Cependant, veuillez noter que les décisions techniques importantes doivent être enregistrées sur GitHub pour garantir la transparence et la traçabilité.

## Réunions régulières

- **Réunion mensuelle des Core Maintainers** : Tenue une fois par mois, conduite publiquement et ouverte à l'observation de tous les membres de la communauté. Les ordres du jour et les comptes rendus des réunions sont publiés sur GitHub.
- **Réunions des groupes de travail** : Chaque groupe de travail peut organiser sa propre fréquence et son format de réunion, avec une recommandation d'au moins une réunion de synchronisation par mois.

## Liste de diffusion

La liste de diffusion est utilisée pour les annonces et notifications importantes :

- Notifications de publication de nouvelles versions
- Notifications de changement de statut des SEP
- Annonces de décisions de gouvernance majeures
- Notifications d'événements communautaires

S'abonner à la liste de diffusion vous garantit de ne pas manquer d'informations importantes.

## Code de conduite

Tous les canaux de communication suivent le Code de Conduite d'iFay. Nous nous engageons à favoriser un environnement communautaire amical, inclusif et professionnel. Veuillez maintenir le respect et la constructivité dans toutes les interactions.

## Langue

L'anglais est la langue de travail principale de la communauté iFay, mais nous accueillons la participation dans d'autres langues. Si vous êtes plus à l'aise en français, en chinois, en allemand ou dans une autre langue, n'hésitez pas à l'utiliser — les membres de la communauté vous aideront avec la traduction pour garantir que vos idées soient transmises avec précision.

Notre objectif est de garantir que la langue ne soit pas un obstacle à la participation à la communauté iFay.
