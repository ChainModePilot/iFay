# 11.1 Tas de données personnelles

## Définition en une ligne

Le Tas de données personnelles est la **mémoire** d'iFay — vos photos, journaux de chat, documents de travail, données de santé… dispersés entre téléphones, ordinateurs et drives cloud. Le Tas de données personnelles unifie leur gestion, permettant à iFay de « se rappeler » n'importe quel moment du passé à tout instant.

## Pourquoi c'est nécessaire

Pensez à comment fonctionne votre propre mémoire. Votre cerveau ne stocke pas les « souvenirs visuels » à un endroit, les « souvenirs auditifs » à un autre, et les « souvenirs émotionnels » à un troisième — quand vous vous rappelez un voyage, images, sons et sentiments reviennent tous ensemble. Votre cerveau **gère toutes ces informations de manière unifiée**, vous permettant de naturellement « vous souvenir » de n'importe quelle expérience.

Mais votre vie numérique n'est rien de tel. Vos photos sont dans la galerie de votre téléphone, les documents de travail sur le disque dur de votre ordinateur, les journaux de chat dans les applications de messagerie, les données de santé dans l'app de votre montre connectée, et les drives cloud contiennent encore plus de fichiers… Ces données sont dispersées dans une douzaine d'endroits différents, dans différents formats, sans connexion entre elles.

Si iFay doit devenir votre avatar numérique, il doit pouvoir « se souvenir » de tout vous concernant — mais il ne peut pas exiger que vous déplaciez toutes vos données au même endroit. La solution du Tas de données personnelles est : **les données restent où elles sont, mais iFay les lit et les écrit toutes via une interface unifiée**. Tout comme votre cerveau ne se soucie pas de quels neurones stockent un souvenir particulier — il a juste besoin de pouvoir le « rappeler ».

Le Tas de données personnelles est le système de mémoire d'iFay. Il donne à iFay une « mémoire de vie » complète, consultable et en croissance continue.

## Sa position dans l'architecture

```
Architecture à quatre couches d'iFay
├── Couche Sociale
├── Couche d'Interaction
│   ├── Sens
│   │   ├── Capteur              ← Les données du Capteur affluent dans le Tas de données
│   │   └── ...
│   └── Action
│       └── ...
├── Couche de Cognition          ← Le Tas de données personnelles est ici
│   ├── Pensée
│   │   ├── 👉 Tas de données personnelles  ← Mémoire : gestion unifiée de toutes les données privées
│   │   ├── Connaissances externes          ← Bibliothèque : les connaissances externes aussi stockées dans le Tas
│   │   └── Conscience alignée              ← Explore le profil du Human Prime depuis le Tas
│   └── Compétences
│       └── ...
└── Couche Ego
```

Le Tas de données personnelles se situe dans le **sous-système de Pensée de la Couche de Cognition** et sert d'« entrepôt central » pour toutes les données d'iFay. Sa position détermine son rôle : presque chaque autre module interagit avec lui.

- Les données collectées par les **Capteurs** de la Couche d'Interaction sont finalement stockées ici
- Les informations acquises par les **Connaissances externes** sont aussi intégrées ici
- La **Conscience alignée** explore les données d'ici pour construire le profil du Human Prime
- Le **Comportement autonome** récupère des informations d'ici pour décider s'il faut agir proactivement

C'est comme le centre de mémoire du cerveau — toutes les informations collectées par les sens convergent finalement ici, et tous les modules qui ont besoin de « se rappeler » quelque chose récupèrent l'information d'ici.

## Comment ça fonctionne

Le fonctionnement du Tas de données personnelles peut être résumé en quatre mots-clés : **Unifié, Classifié, Persistant, Enrichi**.

**1. Unifié — Une interface pour toutes les données**

Vos photos pourraient être sur iCloud, les documents sur Google Drive, les journaux de chat sur votre téléphone local, et les données de santé dans le cloud de votre montre connectée. Le Tas de données personnelles ne déplace pas toutes ces données au même endroit — au lieu de cela, il fournit une **interface de lecture/écriture unifiée**. Les autres modules d'iFay n'ont qu'à dire « je veux trouver des photos prises à Hangzhou il y a trois ans », et le Tas de données personnelles va automatiquement à l'emplacement de stockage correspondant pour les trouver.

**2. Classifié — Quatre types de données**

Le Tas de données personnelles gère quatre types de données :

- **Contenu** : Choses que vous créez — photos, vidéos, articles, notes, code
- **Données** : Informations structurées — données de santé, registres financiers, historique de localisation, données de capteurs
- **Base de connaissances** : Connaissances organisées — vos notes professionnelles, matériaux d'étude, résumés d'expérience de travail
- **Flux d'informations** : Informations continuellement mises à jour — abonnements d'actualités, flux de réseaux sociaux, notifications e-mail

Ces quatre types couvrent virtuellement toutes les formes de données de votre vie numérique.

**3. Persistant — Garde sécurisée des données**

Via **DTP (Protocole de tunnel de données)**, le Tas de données personnelles gère le stockage persistant et la garde des données. « Garde » signifie plus que juste le stockage — cela signifie aussi la protection : garantir que les données ne sont pas perdues, ne sont pas accédées sans autorisation, et peuvent être entièrement récupérées quand nécessaire.

**4. Enrichi — Rendre les données brutes plus précieuses**

Le Tas de données personnelles ne stocke pas passivement les données — il peut aussi effectuer un **enrichissement et une personnalisation** des données. Par exemple :
- Vos données de santé sont un tas de chiffres (rythme cardiaque 72, tension 120/80, pas 8000). Le Tas de données personnelles peut les corréler pour générer des insights comme « votre santé globale aujourd'hui est bonne, et vous avez fait 20% d'exercice de plus qu'hier »
- Vos photos ne sont pas juste des fichiers image. Le Tas de données personnelles peut ajouter des tags sémantiques — « photo de famille au Lac de l'Ouest, Hangzhou, été 2022 »

## Relations avec les autres modules

| Module associé | Relation | Analogie corporelle |
|----------------|----------|---------------------|
| **Capteur** | Les données environnementales collectées par les capteurs sont stockées dans le Tas de données personnelles | Signaux sensoriels → stockage en mémoire |
| **Connaissances externes** | Les informations acquises par les Connaissances externes sont intégrées dans le Tas de données personnelles | Connaissances apprises à la bibliothèque → deviennent votre propre mémoire |
| **Conscience alignée** | La Conscience alignée explore les données du Tas de données personnelles pour construire le profil du Human Prime | Résumer « quel type de personne je suis » à partir des souvenirs |
| **Comportement autonome** | Le Comportement autonome récupère des informations du Tas de données personnelles pour décider si une action proactive est nécessaire | Décider « que devrais-je faire maintenant » basé sur la mémoire |
| **Protocole DTP** | Les données sont transmises bidirectionnellement entre les terminaux et le Tas de données personnelles via DTP | Canaux d'entrée et de sortie pour la mémoire |

## Histoires de scénarios

### Scénario 1 : Trouver un restaurant visité il y a trois ans

Le week-end, vous voulez emmener un ami au restaurant. Vous vous souvenez y être allé il y a trois ans — la nourriture était excellente — mais vous avez oublié le nom et l'adresse. Tout ce dont vous vous souvenez c'est « c'était probablement à Hangzhou, on a mangé de la cuisine de Hangzhou, et je crois que j'y suis allé avec des collègues ».

Vous dites à iFay : « Aide-moi à trouver ce restaurant de cuisine de Hangzhou où je suis allé il y a trois ans. »

iFay commence à chercher dans le Tas de données personnelles :
1. Vérifie d'abord l'**historique de localisation** (type Données) : votre trace d'activité à Hangzhou il y a trois ans, filtrant les lieux où vous vous êtes attardé dans les zones de restauration
2. Puis vérifie les **photos** (type Contenu) : photos que vous avez prises pendant cette période, trouvant plusieurs photos de nourriture d'un restaurant
3. Puis vérifie les **journaux de chat** (type Contenu) : découvre que vous avez envoyé un message dans le groupe de collègues disant « la cuisine de Hangzhou aujourd'hui était incroyable », avec la localisation du restaurant attachée
4. Enfin croise les validations : données de localisation, photos et journaux de chat pointent tous vers le même restaurant — « Saveur de Hangzhou · Succursale du Lac de l'Ouest »

iFay vous dit : « Trouvé — c'est 'Saveur de Hangzhou · Succursale du Lac de l'Ouest', au XX Route, District du Lac de l'Ouest. Tu y es allé avec des collègues le 15 juin il y a trois ans. Tu as pris trois photos de nourriture et l'as recommandé dans le groupe. Tu veux que je fasse une réservation ? »

### Scénario 2 : Insights enrichis à partir de données de santé

Vous portez une montre connectée chaque jour qui enregistre continuellement votre rythme cardiaque, sommeil, pas, oxygène sanguin, et plus. Ces données brutes sont stockées dans le Tas de données personnelles (type Données).

Un jour, après avoir effectué un traitement d'enrichissement sur ces données, iFay vous donne un insight :

« Au cours des deux dernières semaines, votre temps de sommeil moyen est passé de 7,5 heures à 6,2 heures, et votre ratio de sommeil profond est passé de 22% à 15%. Pendant ce temps, votre rythme cardiaque au repos est passé de 68 à 74. Je remarque que pendant cette période votre calendrier a beaucoup de réunions en soirée (données de planning de travail de la Base de connaissances), et vous avez fréquemment posé votre téléphone après 1h du matin (données d'habitudes d'utilisation). »

« Suggestion : vous pourriez avoir besoin d'ajuster votre routine. Voulez-vous que je replanifie toutes les réunions après 22h au lendemain ? »

## Pour les développeurs

Le Tas de données personnelles appartient à la **Phase 2 (Prise de contrôle directe du client)** comme module fondamental.

- **ID d'exigence** : Exigence 11 (Tas de données personnelles)
- **Spécification d'interface** : Interface `PersonalDataHeap` avec quatre méthodes fondamentales : `read()` (lecture unifiée), `write()` (écriture unifiée), `enrich()` (enrichissement de données), et `persist()` (stockage persistant)
- **Quatre types de données** : Contenu, Données, Base de connaissances, Flux d'informations
- **Emplacements de stockage supportés** : Mémoire d'exécution (`runtime_memory`), stockage cloud (`cloud`), base de données vectorielle (`vector_db`), stockage local (`local`)
- **Protocole associé** : DTP (Protocole de tunnel de données) pour la transmission bidirectionnelle de données et la garde des données entre les terminaux et le Tas de données
- **Modules associés** : Capteur (`SensorModule`) fournit les données environnementales, Connaissances externes (`ExternalKnowledge`) fournit les informations externes, Conscience alignée (`AlignedConsciousness`) explore le profil du Human Prime depuis le Tas
- **Tests de conformité** : iFACTS L1 valide les capacités de lecture/écriture pour les quatre types de données, L2 valide les interfaces de données avec les modules Capteur et Connaissances externes, L3 valide la chaîne complète « collecte de données → stockage → enrichissement → récupération »
- **Notes de conception** : L'interface de lecture/écriture unifiée abstrait les différences de stockage sous-jacentes ; supporte les requêtes sémantiques (`semanticQuery`) ; enrichissement et traitement de personnalisation des données
