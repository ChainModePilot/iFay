# 12.2 Compétences enregistrées

✅ « Compétences enregistrées » fait référence à une compétence qu'iFay a déjà maîtrisée ou dont il a acquis la capacité d'utilisation.

❌ Cela ne signifie pas s'inscrire ou créer un compte dans un système externe.

## Définition en une ligne

Les Compétences enregistrées sont la **certification de compétences** d'iFay — tout comme les humains ont besoin d'un permis de conduire pour conduire ou d'une licence médicale pour exercer la médecine, iFay doit « enregistrer » une compétence avant de pouvoir l'utiliser. L'enregistrement n'est pas juste un archivage — c'est une pré-autorisation, garantissant qu'aucune authentification supplémentaire n'est nécessaire au moment de l'exécution.

## Pourquoi c'est nécessaire

Les Compétences enregistrées d'iFay fonctionnent comme l'obtention d'une certification :

- **Confirmer la capacité** : Quel type est cette compétence ? Que peut-elle faire ?
- **Compléter la pré-autorisation** : Obtenir la permission d'utilisation à l'avance, pour que l'exécution réelle n'ait pas besoin de repasser par le processus d'authentification
- **Enregistrer au dossier** : Le Profil d'iFay enregistre toutes les compétences enregistrées, comme lister toutes les certifications sur votre CV

iFay supporte six types de compétences : **API, Workflow, Bot, Agent, APP, Microservice**. Quel que soit le type, l'enregistrement est le prérequis pour l'utilisation — pas d'enregistrement, pas de droits d'exécution.

## Comment ça fonctionne

Le fonctionnement des Compétences enregistrées peut être résumé en quatre mots-clés : **Enregistrer, Pré-autoriser, Mettre en cache, Rafraîchir**.

**1. Enregistrer — « Intégration » de la compétence**

Quand iFay a besoin d'utiliser une nouvelle compétence, l'enregistrement doit être complété d'abord. Le processus d'enregistrement enregistre les informations clés de la compétence : nom et type, point d'accès, méthode d'authentification requise, et description de capacité.

**2. Pré-autoriser — Obtenir le « pass » à l'avance**

La valeur fondamentale de l'enregistrement réside dans la pré-autorisation. Pendant l'enregistrement, iFay complète toutes les étapes d'authentification à l'avance — obtention de jetons d'accès, établissement de connexions sécurisées, vérification des permissions. Ainsi, quand une tâche doit réellement être exécutée, iFay peut l'appeler directement sans délai.

**3. Mettre en cache — Fonctionne hors ligne aussi**

iFay n'est pas toujours en ligne. Quand iFay est hors ligne, le module de Compétences enregistrées met en cache les actions en attente. Une fois le réseau rétabli, ces actions en cache sont exécutées de manière asynchrone.

**4. Rafraîchir — Garder l'autorisation valide**

La pré-autorisation n'est pas permanente. Les jetons d'accès expirent, les permissions peuvent changer. Le module de Compétences enregistrées rafraîchit périodiquement le statut d'autorisation, garantissant que les permissions de chaque compétence sont valides quand elles sont appelées.

## Relations avec les autres modules

| Module associé | Relation | Analogie corporelle |
|----------------|----------|---------------------|
| **Invocation de compétences** | L'Invocation ne peut appeler que des compétences enregistrées ; l'enregistrement est le prérequis de l'exécution | Doit avoir une certification pour travailler |
| **Gestion des identifiants** | La pré-autorisation des Compétences enregistrées dépend des identifiants d'authentification fournis par la Gestion des identifiants | Les certifications nécessitent une preuve d'identité pour postuler |
| **Compétences internes** | Les Compétences enregistrées sont des capacités apprises de l'extérieur ; les Compétences internes sont des capacités innées ; elles se complètent | Certifications obtenues vs intuition naturelle |
| **Comportement autonome** | Le Comportement autonome appelle aussi des compétences enregistrées lors de l'exécution autonome de tâches | L'action spontanée nécessite aussi la capacité correspondante |
| **Profil iFay** | Toutes les compétences enregistrées sont enregistrées dans la dimension compétences du Profil | Liste des certifications sur votre CV |

## Histoires de scénarios

### Scénario 1 : Enregistrer une API de traduction — réponse instantanée grâce à la pré-autorisation

Vous communiquez fréquemment avec des clients à l'étranger, donc vous faites enregistrer à iFay une compétence API de traduction professionnelle.

Pendant l'enregistrement, iFay complète les étapes suivantes :
- Enregistre le type de l'API de traduction (compétence de type API), l'adresse du point d'accès et la description de capacité
- Utilise vos identifiants délégués pour compléter l'authentification API et obtenir un jeton d'accès
- Stocke de manière sécurisée le jeton et marque le statut de pré-autorisation comme « complété »

Une semaine plus tard, vous recevez un e-mail en japonais. Vous dites à iFay : « Aide-moi à traduire cet e-mail. »

Parce que la compétence de traduction était déjà enregistrée et pré-autorisée, iFay appelle **immédiatement** l'API de traduction, vous présentant le résultat en deux secondes. Pas de popup demandant votre mot de passe, pas de roue qui tourne en attendant l'authentification — tout a été préparé lors de l'enregistrement.

### Scénario 2 : Mise en cache hors ligne dans un avion

Vous êtes dans un vol pour New York, téléphone en mode avion. Pendant le vol, vous pensez à plusieurs choses qu'iFay doit gérer :

- « Envoie un e-mail au Directeur Zhang pour confirmer l'heure de la réunion de mercredi prochain »
- « Réserve le restaurant japonais le mieux noté près de mon hôtel à New York »
- « Vérifie la météo de demain à New York »

iFay reçoit ces trois demandes. Toutes les trois compétences sont déjà enregistrées, mais il n'y a pas de réseau maintenant.

Le module de Compétences enregistrées met en cache ces trois actions en attente et vous dit : « Compris. Ces trois choses nécessitent un accès Internet pour être complétées. Je m'en occuperai immédiatement après l'atterrissage. »

L'avion atterrit, vous allumez votre téléphone, et le réseau se rétablit. Le module de Compétences enregistrées détecte la connexion restaurée et exécute immédiatement les trois actions en cache de manière asynchrone : e-mail envoyé, restaurant réservé, informations météo poussées vers vous.

## Pour les développeurs

Les Compétences enregistrées appartiennent à la **Phase 2 (Prise de contrôle directe du client)** comme module fondamental.

- **ID d'exigence** : Exigence 9 (Gestion des Compétences enregistrées)
- **Spécification d'interface** : Interface `RegisteredSkillManager` avec cinq méthodes fondamentales : `register()` (enregistrer compétence), `query()` (interroger compétence), `refreshAuthorization()` (rafraîchir autorisation), `cacheOfflineAction()` (mettre en cache action hors ligne), et `flushCachedActions()` (exécuter actions en cache)
- **Six types de compétences** : `api`, `workflow`, `bot`, `agent`, `app`, `microservice`
- **États de pré-autorisation** : `pre_authorized`, `pending`, `expired`
- **Modules associés** : Invocation de compétences (`InvokeSkillService`) consomme les compétences enregistrées, Gestion des identifiants (`CredentialManager`) fournit le support d'authentification, Compétences internes (`InternalSkill`) fournit des capacités complémentaires
- **Notes de conception** : L'enregistrement est le prérequis de toute action ; la pré-autorisation est complétée lors de l'enregistrement garantissant une exécution sans délai ; la mise en cache hors ligne supporte l'exécution asynchrone ; le statut d'autorisation est périodiquement rafraîchi
- **Tests de conformité** : iFACTS L1 valide l'enregistrement et la pré-autorisation pour les six types de compétences, L2 valide l'intégration d'interface avec l'Invocation de compétences et la Gestion des identifiants, L3 valide le flux complet de mise en cache hors ligne et d'exécution de récupération
