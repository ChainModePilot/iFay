# 4. コミュニケーションチャネル

iFayコミュニティは、さまざまなシナリオでのコラボレーションニーズに対応するため、複数のコミュニケーションチャネルを提供しています。

## GitHub Discussions

GitHub Discussionsは、iFayコミュニティの主要な非同期ディスカッションプラットフォームであり、以下に適しています：

- SEP（仕様拡張提案）の公開議論
- 技術的な質問の詳細な検討
- 機能リクエストと設計に関する議論
- ワーキンググループ内の日常的なコミュニケーション

重要な技術的議論はGitHub Discussions上で行うことを推奨します。これにより、議論の内容が追跡可能かつ検索可能になります。

## GitHub Issues

GitHub Issuesは、具体的なタスク追跡と課題管理に使用されます：

- バグ報告
- 具体的な開発タスク
- ドキュメント改善の提案
- iFACTSテストケースの問題

Issueを提出する際は、メンテナーが問題を迅速に特定し対処できるよう、詳細な再現手順やコンテキスト情報を提供してください。

## Discord

Discordは、iFayコミュニティのリアルタイムコミュニケーションプラットフォームです。各ワーキンググループには専用のチャネルがあります：

- `#general`：一般的な議論
- `#wg-fayid`：FayIDワーキンググループ
- `#wg-ego`：Egoワーキンググループ
- `#wg-protocols`：プロトコルワーキンググループ
- `#wg-fayger`：FayGerワーキンググループ
- `#wg-ifacts`：iFACTSワーキンググループ
- `#wg-docs`：ドキュメントワーキンググループ
- `#wg-community`：コミュニティワーキンググループ

Discordは迅速なやり取りや非公式な議論に適しています。ただし、重要な技術的決定は透明性と追跡可能性を確保するためにGitHub上で記録する必要があることにご注意ください。

## 定期ミーティング

- **コアメンテナー月例ミーティング**：月に1回開催され、公開で行われ、すべてのコミュニティメンバーが傍聴できます。ミーティングのアジェンダと議事録はGitHub上で公開されます。
- **ワーキンググループミーティング**：各ワーキンググループは独自のミーティング頻度と形式を設定できますが、少なくとも月に1回の同期ミーティングを推奨します。

## メーリングリスト

メーリングリストは、重要なお知らせと通知に使用されます：

- 新バージョンリリースの通知
- SEPステータス変更の通知
- 重要なガバナンス決定の発表
- コミュニティイベントの通知

メーリングリストに登録することで、重要な情報を見逃すことがなくなります。

## 行動規範

すべてのコミュニケーションチャネルはiFay行動規範に従います。私たちは、友好的で、包括的で、プロフェッショナルなコミュニティ環境の醸成に取り組んでいます。すべてのやり取りにおいて、敬意と建設性を保ってください。

## 言語

英語はiFayコミュニティの主要な作業言語ですが、他の言語での参加も歓迎します。日本語、中国語、ドイツ語、その他の言語の方が使いやすい場合は、遠慮なくご使用ください。コミュニティメンバーが翻訳を支援し、あなたのアイデアが正確に伝わるようにします。

私たちの目標は、言語がiFayコミュニティへの参加の障壁にならないようにすることです。
