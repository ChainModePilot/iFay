# 9.2 センサー

## 一言定義

センサーはiFayの**神経系統**——iFayが周囲環境のあらゆる変化を感知できるようにする。温度、位置から心拍、光線まで、あなたの体中に張り巡らされた神経末端のように。

## なぜ必要か

一人称トレーサーがiFayの目と耳なら、センサーはiFayの**全身の神経ネットワーク**である。

あなたの体を考えてみよう：目と耳だけで世界を知覚しているわけではない。皮膚は温度と接触を感じ、内耳はバランスと加速度を感知し、体は空腹、疲労、寒さを教えてくれる。

スマートフォンやスマートウォッチも同様：GPSは位置を、加速度計は歩行か走行かを、光センサーは室内か屋外かを、心拍センサーは心拍の速さを知っている。

iFayのセンサーモジュールは、これらすべてのデバイス上のセンサーを**統一的に接続**し、iFayに完全な神経系統を持たせる。しかもこの神経系統は拡張可能——将来どんな新型センサーが登場しても接続できる。

## アーキテクチャ内の位置

```
iFay 4層アーキテクチャ
├── ソーシャルレイヤー
├── インタラクションレイヤー    ← センサーはここ
│   ├── 知覚（Sense）
│   │   ├── 一人称トレーサー   ← 外を見る、画面を見る
│   │   ├── 👉 センサー        ← 環境を知覚（温度、位置、運動……）
│   │   └── セルフアウェアネス  ← 内を見る、あなたを読み取る
│   └── アクション（Action）
│       ├── シミュレーテッドオペレーション
│       ├── スキル呼び出し
│       └── 自律行動
├── コグニションレイヤー
│   ├── 思考
│   │   ├── パーソナルデータヒープ  ← センサーデータは最終的にここに保存
│   │   └── ...
│   └── スキル
│       ├── デバイスドライバーハブ  ← センサーのハードウェアインターフェースを管理
│       └── ...
└── エゴレイヤー
```

センサーは**インタラクションレイヤーの知覚サブシステム**に位置する。ただしセンサーには特別な点がある：それ自体は**感度の調節**のみを担当し（いつデータを多く収集し、いつ少なく収集するかを決定）、実際のハードウェア接続とデータ保存はそれぞれコグニションレイヤーの**デバイスドライバーハブ**と**パーソナルデータヒープ**が担当する。

## 動作の仕組み

**1. ブリッジ——デバイス上のセンサーを接続**

センサーモジュールは「翻訳官」のように、すべての異なるデバイス・異なるタイプのセンサーデータを、iFayが理解できるフォーマットに統一翻訳する。**CAP（Control Authority Protocol）**と**DTP（Data Tunnel Protocol）**に基づいてこのブリッジを実現する。

**2. 調節——動的感度**

iFayのセンサーは常に最高精度ですべてのセンサーデータを収集する必要はない。例えば：
- オフィスで静かに作業中、GPSは毎秒位置を更新する必要はない
- しかし車のナビゲーション中は、GPSは高頻度更新が必要
- 睡眠中、心拍センサーはサンプリング頻度を下げられる
- しかし心拍異常を検出したら、即座にサンプリング頻度を上げる

**3. 拡張——将来のセンサーも接続可能**

センサーモジュールの設計はオープンである——新しいセンサータイプが登場した時、デバイスドライバーハブを通じて新しいドライバーを登録するだけで、センサーモジュールがその感度を管理し、パーソナルデータヒープがそのデータを保存できる。

## 他モジュールとの関係

| 関連モジュール | 関係 | 人体の類比 |
|---------|------|---------|
| **デバイスドライバーハブ** | センサーの実際のハードウェアインターフェースを管理 | 神経末端 → 神経伝導路 |
| **パーソナルデータヒープ** | センサーが収集したデータを保存 | 感覚信号 → 記憶保存 |
| **一人称トレーサー** | 同じ知覚サブシステムだが分担が異なる | 目 vs 全身の神経 |
| **セルフアウェアネス** | センサーが環境データを提供し、セルフアウェアネスがそれを利用してヒューマンプライム（Human Prime）の状態を推論 | 神経系統が感覚を提供 → 脳が感情を解読 |
| **CAP / DTP プロトコル** | センサーはこの2つのプロトコルに基づいてデータブリッジを実現 | 神経信号の伝送プロトコル |

## シナリオストーリー

### シナリオ1：スマートヘルスリマインド

午後3時、あなたはPC前に3時間座っている。iFayがスマートウォッチのセンサーデータから発見する：
- **加速度計**：過去3時間ほぼ大きな動きなし（ずっと座っている）
- **心拍センサー**：心拍が通常の72bpmから85bpmに上昇
- **皮膚温度センサー**：手首の温度がやや上昇

センサーモジュールがこれらのデータを集約しコグニションレイヤーに伝達。iFayが総合判断後、穏やかにリマインドする：「3時間座り続けています。心拍がやや高めです。立ち上がって歩きませんか？」

### シナリオ2：無人機の自律飛行

iFayが航空撮影任務の無人機にデプロイされている。複数のセンサーがある：GPS、IMU、カメラ、超音波/LiDAR、気圧計、風速センサー。

センサーモジュールがすべてのデータストリームを統一接続する。安定飛行中はIMUの感度を適度に下げられるが、突然の風速変化を検出すると、センサーモジュールは即座にIMUと気圧計の感度を上げ、iFayがより精密に無人機の姿勢を制御できるようにする。

## 開発者向け

センサーモジュールは**フェーズ2（クライアントの直接接管）**のコアモジュール。

- **要件番号**：要件7（センサーモジュール）
- **インターフェース仕様**：`SensorModule`インターフェース、`registerSource()`、`adjustSensitivity()`、`getDataStream()`、`getActiveStatus()`の4つのコアメソッド
- **関連プロトコル**：CAP（Control Authority Protocol）でセンサーハードウェアを接管、DTP（Data Tunnel Protocol）で双方向データ転送
- **コンプライアンステスト**：iFACTS L1で感度調節能力を検証、L2でデバイスドライバーハブとパーソナルデータヒープとのインターフェース対接を検証
