# 12.3 内部スキル

## 一言定義

内部スキルはiFayの**直感と底線**——登録スキルがiFayが外部から学んだ能力（取得した証書のように）なら、内部スキルはiFayが生来持つ能力（直感と道徳的底線のように）。外部スキルの出力がヒューマンプライム（Human Prime）の意図に反しないことを保証する。

## なぜ必要か

内部スキルは3つのことを行う：

**第一、ヒューマンプライム（Human Prime）のパーソナリティと整合した習慣を確立し、外部スキルのガバナンスを含む。** 外部スキルの出力に対する「直感チェック」——結果がヒューマンプライム（Human Prime）のスタイルに合っているか？価値観に合っているか？合っていなければ、インターセプトして調整する。

**第二、内省メカニズムを提供し、外部知識がヒューマンプライム（Human Prime）の意図と矛盾しないことを保証する。** 外部知識がiFayに採用される前に、ヒューマンプライム（Human Prime）の意図による検証を経る「フィルター」。

**第三、ヒューマンプライム（Human Prime）固有の能力を組み込む。** 金融アナリストなら、iFayは生来金融分析能力を備えるべき——外部から登録したスキルではなく、iFayの初期化時にヒューマンプライム（Human Prime）の職業背景と専門知識に基づいて組み込まれた能力。

簡単に言えば：登録スキルはiFayの「外功」、内部スキルはiFayの「内功」。

## アーキテクチャ内の位置

```
iFay 4層アーキテクチャ
├── コグニションレイヤー    ← 内部スキルはここ
│   ├── 思考（Thought）
│   │   ├── 外部知識        ← 内部スキルが外部知識の出力を審査
│   │   └── アラインド意識  ← 内部スキルの行動基準はアラインド意識から
│   └── スキル（Skill）
│       ├── 登録スキル      ← 内部スキルが登録スキルの出力をガバナンス
│       └── 👉 内部スキル    ← iFayの「直感と底線」
└── エゴレイヤー
    └── Egoモデル           ← Egoが性格を形成、内部スキルが性格制約を実行
```

## 動作の仕組み

**1. ガバナンス——外部スキル出力の「直感チェック」**
登録スキルがタスクを実行して結果を返すたびに、内部スキルが結果を審査する。問題があればインターセプトし、ヒューマンプライム（Human Prime）プロファイルに基づいて調整する。

**2. 内省——外部知識がiFayを「偏向」させないことを保証**
外部知識がiFayに吸収される前に、ヒューマンプライム（Human Prime）の意図による「フィルタリング」を経る。

**3. 賦能——ヒューマンプライム（Human Prime）の専門能力を組み込む**
iFay初期化時に組み込まれる専門知識体系、分析スタイル、業界経験。外部APIの呼び出しではなく、iFay内在の一部。

## 他モジュールとの関係

| 関連モジュール | 関係 | 人体の類比 |
|---------|------|---------|
| **登録スキル** | 内部スキルが登録スキルの出力をガバナンス | 直感が外部の提案を審査 |
| **外部知識** | 内部スキルの内省メカニズムが外部知識を審査 | 主見を持って外部情報を選別 |
| **アラインド意識** | 内部スキルの行動基準はアラインド意識のプロファイルから | 自己認知が直感反応を決定 |
| **Egoモデル** | Egoが性格を形成、内部スキルがスキル層で性格制約を実行 | 性格が行動方式を決定 |
| **自律行動** | 自律行動が自律的にタスク実行する際、内部スキルの永続能力に依存 | 自発行動が本能と専門素養に依存 |

## シナリオストーリー

### シナリオ1：翻訳結果のスタイル校正

あなたはテック企業のCEOで、温和だが専門的な語調でコミュニケーションする習慣がある。翻訳APIが直接的な語調で翻訳した結果を、内部スキルがアラインド意識のコミュニケーションスタイルプロファイルに基づいてより委婉な表現に調整する。

### シナリオ2：金融アナリストの内在能力

資深金融アナリストとして、iFay初期化時に専門能力が内部スキルとして組み込まれている。市場分析レポートを受け取った時、外部金融分析APIを呼び出すことなく、内部スキルの専門知識と分析スタイルに基づいて批注を生成する。

## 開発者向け

内部スキルは**フェーズ4（iFay + coFay 完全な擬人化）**のコアモジュール。

- **要件番号**：要件15（内部スキル）
- **インターフェース仕様**：`InternalSkill`インターフェース、`introspect()`、`intercept()`、`getHostCapabilities()`の3つのコアメソッド
- **3大責務**：（1）ヒューマンプライム（Human Prime）のパーソナリティと整合した習慣を確立し外部スキル出力をガバナンス；（2）内省メカニズムで外部知識とヒューマンプライム（Human Prime）の意図の矛盾を防止；（3）ヒューマンプライム（Human Prime）固有の能力を組み込む
- **登録スキルとの違い**：登録スキルは外部取得の能力（登録と事前認可が必要）、内部スキルは生来の能力（初期化時に組み込み、登録不要）
- **コンプライアンステスト**：iFACTS L1で内省とインターセプト能力を検証、L2で登録スキルと外部知識の出力審査インターフェースを検証、L4で外部出力調整がヒューマンプライム（Human Prime）プロファイルに真に適合するかを検証
