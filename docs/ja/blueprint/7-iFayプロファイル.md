# 7. iFayプロファイル

iFay Profileは iFayの完全な「身分証」——セマンティックに解釈可能な6次元属性表である。人間がiFayを識別できるだけでなく、システムが解析でき、Fay同士が互いを認識できるようにする。

ヒューマンプライム（Human Prime）プロファイル（アラインド意識 / Aligned Consciousness）はiFay Profileのサブセットかつ入力ソースである。ヒューマンプライム（Human Prime）プロファイルが記述するのは「ヒューマンプライム（Human Prime）がどのような人か」であり、iFay Profileが記述するのは「このiFayがどのような存在か」である——ヒューマンプライム（Human Prime）の人格特性を含むが、iFay自身のスキル、ハードウェア、権限などの独立属性も含む。

ヒューマンプライム（Human Prime）プロファイルは[アラインド意識](./11.3-アラインド意識)モジュールが維持する。スキル次元は[登録スキル](./12.2-登録スキル)が管理し、ハードウェア次元は[デバイスドライバーハブ](./12.1-デバイスドライバーハブ)が管理し、権限次元は[クレデンシャル管理](./8.1-クレデンシャル管理)が管理する。

## iFayアイデンティティ

iFayアイデンティティはFayIDとメタ情報で構成される。FayIDはグローバルに一意で、人間可読、機械可読な永続識別子であり、iFayがすべてのインタラクションシナリオで識別・追跡される基盤である。

iFayアイデンティティはスムーズなアイデンティティ移行をサポートする——iFayが個人アシスタントから自律的な社会的役割に進化する際、FayIDを変更する必要はなく、アイデンティティは自然に継続する。

## Egoモデル

ProfileのEgo次元は、現在アクティブなEgoバージョンの参照を記録する。

Egoモデルはプラグ可能で切り替え可能なエッジ小型モデルであり、任意の端末デバイスにロードすることでそのデバイスにヒューマンプライム（Human Prime）の個性を持たせることができる。iFayは複数のEgoバージョンを持つことをサポートし、ヒューマンプライム（Human Prime）の多面的な人格に適応する——ただし任意の時点で1つのバージョンのみがアクティブ状態にあり、「人格分裂」を防ぐ。

切り替え方式は2つ：ヒューマンプライム（Human Prime）による手動トリガー、またはiFayがコンテキストに基づいて自動判断。切り替え完了後、Profile内のアクティブEgoバージョン参照が同期更新される。

### シナリオ

> 張磊はプロダクトマネージャーで、自分のiFayに2つのEgoバージョンを設定した：「プロフェッショナルモード」と「リラックスモード」。
>
> 平日の午前、iFayは自動的に「プロフェッショナルモード」に切り替わる——メール返信は厳格な措辞で、会議スケジュールは効率を優先し、要件ドキュメント処理はロジックの完全性を重視する。
>
> 週末の夜、iFayは張磊がグルメ推薦を閲覧していることを検出し、自動的に「リラックスモード」に切り替わる——レストラン推薦は雰囲気と味の好みを重視し、語調もカジュアルになる。
>
> 2つのバージョン、同じ張磊。

## Faying思考

Faying思考はiFayの認知資産であり、4つのサブタイプを含む：

| サブタイプ | 説明 | 例 |
|--------|------|------|
| **コンテンツ（Content）** | 構造化または非構造化のコンテンツ資産 | ヒューマンプライム（Human Prime）の写真集、執筆した記事、創作した音楽、録画した動画 |
| **データ（Data）** | 個人データ | 健康モニタリングデータ、消費記録、アプリ使用習慣、位置履歴 |
| **ナレッジベース（Knowledge Base）** | 組織化された知識体系 | 専門分野ノート、学習資料ライブラリ、業務経験まとめ、業界研究レポート |
| **情報フロー（Info Feed）** | リアルタイム情報購読 | フォローしているニュースソース、業界動向プッシュ、ソーシャルメディア更新、株式相場 |

思考次元のデータはパーソナルデータヒープの初期化と継続的な蓄積から来ており、iFayが世界を理解し判断を下す基盤である。

## Fayingスキル

FayingスキルはiFayの行動能力であり、6種類のスキルタイプを含む：

| タイプ | 説明 |
|------|------|
| **API** | 直接呼び出し可能なプログラミングインターフェース（天気照会API、翻訳APIなど） |
| **ワークフロー（Workflow）** | 複数ステップの自動化フロー（「毎週月曜に週報を生成しチームに送信」など） |
| **Bot** | 特定シナリオの対話ロボット（カスタマーサービスBot、予約Botなど） |
| **Agent** | 自律的意思決定能力を持つインテリジェントエージェント（投資分析Agentなど） |
| **APP** | 完全なアプリケーション（カレンダー管理APP、健康追跡APPなど） |
| **マイクロサービス（Microservice）** | 独立デプロイされたサービスユニット（画像認識サービス、音声テキスト変換サービスなど） |

スキル登録はiFayがあらゆるアクションを実行するための前提条件である。登録済みスキルは事前認可が完了し、実行時に追加の認証が不要となる。

## Fayingハードウェア

ハードウェアはiFayの「肢体」である。iFayは人格と認知の実体であり、ハードウェアは物理世界とインタラクションするための媒体である。

### 3つのハードウェアサブタイプ

| サブタイプ | 説明 | 例 |
|--------|------|------|
| **デバイス（Device）** | 端末ハードウェアデバイス | スマートフォン、PC、スマートウォッチ、無人機、スマートホームデバイス |
| **ストレージ（Storage）** | データストレージリソース | クラウドストレージ、ローカルハードディスク、NAS、ベクトルデータベース |
| **コンピューティング（Computing）** | 計算リソース | GPU、クラウドコンピューティングノード、エッジコンピューティングデバイス |

### ハードウェア発見メカニズム

iFayが新しい環境に移行する際、端末デバイスが現在の環境でFaying可能なハードウェアをスキャンする——BluetoothやWiFiが周囲のデバイスをスキャンする方式に類似する。iFay自体はハードウェアスキャンを直接実行せず、スキャン結果に基づいて能力マッチングを行う。

ここに重要な区別がある：

- **iFayがどのハードウェアを操作できるか**は、その認知能力（登録スキル、内部スキル）に依存する
- **iFayがどのハードウェアに接続できるか**は、現在の環境で実際に利用可能なデバイスに依存する

### シナリオ

> バドミントンができる人が体育館に入ることを想像してほしい。
>
> 「バドミントンができる」は彼の**スキル**——これは彼自身の能力であり、どこに行っても持っている。しかし実際にプレーできるかは、体育館に**ラケット**があるかどうかに依存する——これは環境が提供するハードウェアである。
>
> iFayも同様である。無人機制御のスキルを持っていても（飛行制御APIを登録済み）、現在の環境に接続可能な無人機がなければ、そのスキルは発揮できない。逆に、環境に3Dプリンターがあっても、iFayが関連する操作スキルを登録していなければ、そのデバイスを「見る」ことはできても使用できない。

## Faying権限

権限はiFayの「関係」——iFayと外部世界の信頼境界を定義する。

### 3つの権限ライフサイクル

| ライフサイクル | 説明 | 例 |
|----------|------|------|
| **生来（Inherent）** | 作成時に既に持つ権限 | ヒューマンプライム（Human Prime）のカレンダーへのアクセス、ヒューマンプライム（Human Prime）の嗜好設定の読み取り |
| **永続（Persistent）** | 長期有効な権限 | ヒューマンプライム（Human Prime）のメールボックスの読み書き権限、企業イントラネットアクセス権 |
| **一時（Ephemeral）** | 必要時に申請し、使用後に解放 | 医療記録への一回限りのアクセス、特定APIの一時的な呼び出し権 |

### 拡張可能な認証方式

iFayの認証方式は拡張可能で、SSO、OAuth、Fingerprintなどの方法をサポートする。ここでのFingerprintは総称であり、身分を識別できるすべての標識を包含する——顔認識、声紋、デバイスフィンガープリントなど。異なるフィンガープリントタイプは異なる確信度を持つ：顔認識の確信度はデバイスフィンガープリントより高い可能性があり、多要素の組み合わせの確信度は単一要素より高い。

### シナリオ

> 王芳のiFayは3つの異なるライフサイクルの権限を持つ：
>
> **生来**：iFay作成時に王芳のカレンダーへのアクセス権限を自動取得した——これはすべてのiFayの基本能力であり、追加申請は不要。
>
> **永続権限**：王芳がiFayに業務メールボックスの長期管理を認可した。iFayはメールの閲覧、返信の起草、ラベル管理ができ、この権限は王芳が主動的に取り消すまで有効。
>
> **一時権限**：王芳が去年の健康診断レポートを確認する必要がある。iFayに病院システムから記録を取得させる。iFayは一回限りの医療データアクセス権限を一時申請し、レポート取得後に権限は自動解放される——次回必要な時は再申請が必要。

## Profileの可視性

Profileの可視性はアプリケーション層の関心事であり、具体的なアプリケーションシナリオとインタラクション方式によって決定される。

同じiFay Profileでも、異なるシナリオで異なる次元を表示する：Fay間協力時は相手がスキルと権限次元のみを見る必要があるかもしれない；ヒューマンプライム（Human Prime）管理インターフェースではすべての6次元が完全に可視；公開ソーシャルシナリオではアイデンティティと一部のスキル情報のみを表示するかもしれない。

Profile自体は「誰が何を見られるか」を定義しない——この決定はProfileを使用するアプリケーションに委ねられる。
