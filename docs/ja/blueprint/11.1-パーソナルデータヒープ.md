# 11.1 パーソナルデータヒープ

## 一言定義

パーソナルデータヒープはiFayの**記憶**——写真、チャット履歴、業務ドキュメント、健康データ……スマートフォン、PC、クラウドに散在するデータを統一管理し、iFayがいつでも過去を「思い出せる」ようにする。

## なぜ必要か

あなたのデジタルライフのデータは十数か所に散在し、異なるフォーマットで、互いに無関係である。iFayがあなたのデジタル化身になるには、すべてを「記憶」できなければならない——しかしすべてのデータを一か所に移すことは要求できない。パーソナルデータヒープの解決策は：**データは元の場所に残し、iFayが統一的な方法で読み書きする**。

## アーキテクチャ内の位置

```
iFay 4層アーキテクチャ
├── ソーシャルレイヤー
├── インタラクションレイヤー
├── コグニションレイヤー    ← パーソナルデータヒープはここ
│   ├── 思考（Thought）
│   │   ├── 👉 パーソナルデータヒープ  ← 記憶：すべてのプライベートデータを統一管理
│   │   ├── 外部知識                  ← 図書館：外部知識もデータヒープに保存
│   │   └── アラインド意識            ← データヒープからプロファイルを発掘
│   └── スキル（Skill）
└── エゴレイヤー
```

## 動作の仕組み

**1. 統一——1つのインターフェースですべてのデータを管理**

写真がiCloudに、ドキュメントがGoogle Driveに、チャット履歴がローカルに。パーソナルデータヒープはデータを一か所に移すのではなく、**統一的な読み書きインターフェース**を提供する。

**2. 分類——4種類のデータタイプ**

- **コンテンツ（Content）**：あなたが創造したもの——写真、動画、記事、ノート、コード
- **データ（Data）**：構造化情報——健康データ、財務記録、位置履歴、センサーデータ
- **ナレッジベース（Knowledge Base）**：整理された知識——専門ノート、学習資料、業務経験まとめ
- **情報フロー（Info Feed）**：継続更新される情報——ニュース購読、ソーシャルメディア動態、メール通知

**3. 永続——データの安全な保管**

DTP（Data Tunnel Protocol）を通じて、パーソナルデータヒープはデータの永続化保存と監護を担当する。

**4. エンリッチメント——生データをより価値あるものに**

パーソナルデータヒープは受動的にデータを保存するだけでなく、**エンリッチメントとパーソナライズ処理**も行う。

## 他モジュールとの関係

| 関連モジュール | 関係 | 人体の類比 |
|---------|------|---------|
| **センサー** | センサーが収集した環境データを保存 | 感覚信号 → 記憶保存 |
| **外部知識** | 外部知識が取得した情報を統合 | 図書館で学んだ知識 → 自分の記憶に |
| **アラインド意識** | アラインド意識がデータヒープからデータを発掘しプロファイルを構築 | 記憶から「自分はどんな人か」をまとめる |
| **自律行動** | 自律行動がデータヒープから情報を取得し能動的行動の要否を判断 | 記憶に基づいて「何をすべきか」を判断 |
| **DTPプロトコル** | データがDTPプロトコルで端末とデータヒープ間を双方向転送 | 記憶の入出力チャネル |

## シナリオストーリー

### シナリオ1：3年前に行ったレストランを見つける

iFayがパーソナルデータヒープで検索：位置履歴、写真、チャット履歴を交差検証し、レストランを特定する。データソースを横断した統一検索の価値を示す。

### シナリオ2：健康データのエンリッチメント洞察

スマートウォッチの生データ（心拍、睡眠、歩数）を日程データや使用習慣と関連付け、パーソナライズされた健康洞察を生成する。

## 開発者向け

パーソナルデータヒープは**フェーズ2（クライアントの直接接管）**のコアモジュール。

- **要件番号**：要件11（パーソナルデータヒープ）
- **インターフェース仕様**：`PersonalDataHeap`インターフェース、`read()`、`write()`、`enrich()`、`persist()`の4つのコアメソッド
- **4種類のデータタイプ**：Content、Data、Knowledge Base、Info Feed
- **サポートする保存場所**：`runtime_memory`、`cloud`、`vector_db`、`local`
- **関連プロトコル**：DTP（Data Tunnel Protocol）
- **コンプライアンステスト**：iFACTS L1で4種類のデータタイプの読み書き能力を検証、L2でセンサーと外部知識モジュールのデータインターフェースを検証、L3で「データ収集→保存→エンリッチメント→検索」の完全なリンクを検証
