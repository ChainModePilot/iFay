# 10.1 シミュレーテッドオペレーション

## 一言定義

シミュレーテッドオペレーションはiFayの**両手**——人間のようにクリック、ドラッグ、スクロール、タイピングができ、既存のあらゆるソフトウェアインターフェースを操作できる。ソフトウェア側でAI向けの改造は一切不要。

## なぜ必要か

シミュレーテッドオペレーションと従来の「自動化スクリプト」（RPAなど）には本質的な違いがある。RPAは台本を暗記するだけの俳優——事前に書かれたスクリプト通りに実行し、インターフェースが少し変わると止まってしまう。iFayのシミュレーテッドオペレーションは**知覚駆動型**：一人称トレーサーで画面を「見て」、現在のインターフェースの状態を理解し、次に何をすべきか決定する。

## アーキテクチャ内の位置

```
iFay 4層アーキテクチャ
├── ソーシャルレイヤー
├── インタラクションレイヤー    ← シミュレーテッドオペレーションはここ
│   ├── 知覚（Sense）
│   │   ├── 一人称トレーサー   ← 目：画面に何があるか見る
│   │   ├── センサー           ← 神経系統：環境を知覚
│   │   └── セルフアウェアネス  ← 内心：あなたの意図を読み取る
│   └── アクション（Action）
│       ├── 👉 シミュレーテッドオペレーション ← 両手：クリック、ドラッグ、タイピング
│       ├── スキル呼び出し     ← 口：サービスに直接「電話」
│       └── 自律行動           ← 自主意志：いつ行動するか決定
├── コグニションレイヤー
└── エゴレイヤー
```

## 動作の仕組み

**1. 知覚——まず見て、それから手を動かす**

操作前に、iFayは一人称トレーサーで現在のインターフェースを「見る」。HTMLコードを解析するのではなく、人間のように**画面に何が表示されているか**を見る。

**2. 操作——人間のようにインタラクション**

クリック、ドラッグ、スクロール、タイピング、エッジジェスチャー、マルチフィンガージェスチャーなど、人間がインターフェース上でできるすべての操作をサポート。

**3. フィードバック——操作後に結果を見る**

各操作後、一人称トレーサーで再度インターフェースを「見て」、操作が成功したか確認する。この「操作→観察→調整」のサイクルがRPAスクリプトとの根本的な違い。

**未知のインターフェースに遭遇したら？**

人間のように**探索**する：ボタンをクリックして何が起きるか見る、アイコンにホバーしてツールチップがあるか確認する、ページをスクロールして下に何があるか見る。

## 他モジュールとの関係

| 関連モジュール | 関係 | 人体の類比 |
|---------|------|---------|
| **一人称トレーサー** | シミュレーテッドオペレーションの「目」 | 手 ↔ 目（手と目の協調） |
| **スキル呼び出し** | 同じアクションサブシステムだが方式が異なる | 手で操作 vs 電話で用事を済ませる |
| **自律行動** | 自律行動が「いつ手を動かすか」を決定、シミュレーテッドオペレーションが「どう手を動かすか」を担当 | 意志が行動を決定 → 手が実行 |
| **Egoモデル** | Egoモデルがシミュレーテッドオペレーションの行動スタイルを制約 | 性格が仕事の仕方に影響 |
| **クレデンシャル管理** | ログイン時にクレデンシャル管理のアカウント・パスワードを使用 | 手で鍵を持ってドアを開ける |

## シナリオストーリー

### シナリオ1：複雑な保険請求フォームの記入

保険会社の古いウェブサイトで請求申請を記入する必要がある。30以上のフィールドが5ページに分散し、各種ドロップダウンメニューと日付選択器がある。

iFayが作業開始：一人称トレーサーでログインページを「見て」、委託されたクレデンシャルでログイン、「請求申請」入口を見つけてクリック、フォームを順次記入、写真をアップロード、最終確認ページで内容を照合して「送信」をクリック。

プロセス全体で、iFayは事前に書かれたスクリプトを実行しているのではなく、**画面を見て、内容を理解し、判断を下している**。

### シナリオ2：アプリ更新への適応

フードデリバリーアプリを開くが、インターフェースが前回と違う——アプリが更新されたばかりだ。RPAスクリプトならここでエラー停止するが、iFayは違う：新しいレイアウトをスキャンし、検索アイコンの新しい位置を見つけ、柔軟に操作戦略を調整する。

## 開発者向け

シミュレーテッドオペレーションは**フェーズ1（人間操作のシミュレーション）**のコアモジュール。

- **要件番号**：要件5（シミュレーテッドオペレーション）
- **インターフェース仕様**：`SimulatedOperation`インターフェース、`execute()`、`explore()`、`getPostActionState()`の3つのコアメソッド
- **サポートする操作タイプ**：`click`、`drag`、`scroll`、`gesture`、`type`
- **コンプライアンステスト**：iFACTS L1で各操作タイプの実行能力を検証、L2で一人称トレーサーとの手と目の協調インターフェースを検証
- **iFay Ready関連**：Bronze級認証はアプリがiFayのシミュレーテッドオペレーションによる操作をサポートすることを要求
