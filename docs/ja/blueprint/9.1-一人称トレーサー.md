# 9.1 一人称トレーサー

## 一言定義

一人称トレーサーはiFayの**目と耳**——あなたが見ている画面を見て、あなたが聞いている音を聞き、あなたとまったく同じ一人称視点を持つことを可能にする。

## なぜ必要か

想像してみよう、助手にウェブフォームの記入を頼んだとする。もしこの助手が目が見えなかったら——画面上の入力欄、ボタン、エラーメッセージが見えなかったら——どうやって手伝えるだろうか？

これが一人称トレーサーのないiFayである。

人間がPCやスマートフォンを操作する時、目で画面を見て、耳で通知音を聞く。iFayにも同じ能力が必要だ。一人称トレーサーはiFayにこの能力を与えるモジュールである——iFayに**あなたのように画面を見る**ことを可能にし、プログラマーのようにコードを読むのではない。

ここに重要な区別がある：iFayが見るのはあなたが**肉眼で見る画面**であり、ウェブページの背後に隠されたコード（HTMLタグ、SEOキーワードなどあなたには見えないもの）ではない。本を読む時、あなたが見るのは文字と画像であり、印刷所の組版ファイルではない。iFayも同様——構造化ドキュメントを解析するのではなく、「視覚」でインターフェースを理解することを優先する。

## アーキテクチャ内の位置

```
iFay 4層アーキテクチャ
├── ソーシャルレイヤー
├── インタラクションレイヤー    ← 一人称トレーサーはここ
│   ├── 知覚（Sense）
│   │   ├── 👉 一人称トレーサー   ← 外を見る、画面を見る
│   │   ├── センサー             ← 環境を知覚
│   │   └── セルフアウェアネス    ← 内を見る、あなたを読み取る
│   └── アクション（Action）
│       ├── シミュレーテッドオペレーション
│       ├── スキル呼び出し
│       └── 自律行動
├── コグニションレイヤー
└── エゴレイヤー
```

一人称トレーサーは**インタラクションレイヤーの知覚サブシステム**に位置する。

## 動作の仕組み

**1. 画面を見る**
iFayは画面に現在表示されているコンテンツをキャプチャする——テキスト、画像、ボタン、入力欄、ポップアップ……あなたが見えるすべてのものをiFayも見ることができる。

**2. 音を聞く**
インターフェースに通知音、音声再生、動画の音声がある場合、iFayもキャプチャできる。

**3. リアルタイムで変化を追跡**
最も重要な部分。iFayが画面上で操作を実行する時（ボタンのクリックなど）、**即座に操作結果を見る**必要がある——ページ遷移したか？エラーメッセージが出たか？ローディングアニメーションがまだ回っているか？

**4. 手と目の協調**
一人称トレーサーとシミュレーテッドオペレーションモジュールは緊密に連携する「パートナー」——人間の目と手のように。目がボタンの位置を見て、手が正確にクリックし、手がクリックした後、目が結果を見る。この「見る→する→また見る」のサイクルがiFayの手と目の協調能力である。

**5. 「見えない」を報告**
何らかの理由（画面ロック、アプリクラッシュ、権限不足など）で一人称トレーサーが画面を取得できない場合、何も起きていないふりはしない——正直にiFayのコグニションレイヤーに報告する：「見えません。」コグニションレイヤーは報告を受けて次のステップを決定する。

## 他モジュールとの関係

| 関連モジュール | 関係 | 人体の類比 |
|---------|------|---------|
| **シミュレーテッドオペレーション** | 緊密結合、手と目の協調 | 目 ↔ 手 |
| **コグニションレイヤー** | 知覚状態を報告（正常/デグレード/失敗） | 目 → 脳 |
| **センサー** | 同じ知覚サブシステムだが分担が異なる | 目 vs 皮膚の神経 |
| **セルフアウェアネス** | 同じ知覚サブシステムだが方向が異なる：一人称トレーサーは外を見る、セルフアウェアネスは内を見る | 目 vs EQ |

## シナリオストーリー

### シナリオ1：行政サイトのフォーム記入を手伝う

古い行政サイトで申請書を記入する必要がある。このサイトはAI向けの適合は一切されていない——APIもなく、構造化インターフェースもなく、普通のウェブフォームがあるだけ。

iFayの一人称トレーサーがこのフォームを「見る」：氏名入力欄、身分証番号入力欄、ドロップダウンメニュー、認証コード画像、青い「送信」ボタン。これらの視覚情報をシミュレーテッドオペレーションモジュールに伝達し、シミュレーテッドオペレーションモジュールがあなたの手のように、フィールドを一つずつ記入し、オプションを選択し、認証コードを入力し、送信をクリックする。

送信後、一人称トレーサーが即座に結果を「見る」——赤いエラーメッセージ「身分証番号の形式が正しくありません」が表示されたら、この変化をキャプチャし、iFayは修正して再送信する必要があることを知る。

### シナリオ2：株式相場のリアルタイム監視

iFayに株式のリアルタイムチャートを監視させる。一人称トレーサーが画面上のK線チャートと数値変化を持続的に「見ている」——価格が152.30から153.10に跳ね上がり、出来高の棒グラフが突然高くなり、MACD指標にゴールデンクロスが出現……

これらのリアルタイム変化を一人称トレーサーが持続的に追跡する。価格があなたが設定した目標価格に達した時、iFayは即座に通知する。

## 開発者向け

一人称トレーサーは**フェーズ1（人間操作のシミュレーション）**のコアモジュール。

- **要件番号**：要件4（一人称トレーサー）
- **インターフェース仕様**：`FirstPersonTracer`インターフェース、`captureVisual()`、`captureAudio()`、`trackChanges()`、`getPerceptionStatus()`の4つのコアメソッド
- **コンプライアンステスト**：iFACTS L1で視覚キャプチャ能力を検証、L2でシミュレーテッドオペレーションモジュールとの手と目の協調インターフェースを検証
- **設計要点**：構造化ドキュメント解析より視覚知覚を優先；知覚失敗時は必ずコグニションレイヤーに報告；シミュレーテッドオペレーションモジュールとクローズドループフィードバックを形成
