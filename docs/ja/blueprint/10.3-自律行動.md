# 10.3 自律行動

## 一言定義

自律行動はiFayの**自主意志**——シミュレーテッドオペレーションは手、スキル呼び出しは口、自律行動はiFay自身が「いつ手を動かすか、いつ口を開くか」を決定する。iFayを受動的な実行者から能動的な行動者に変える。

## なぜ必要か

真に優れた助手は**能動的に**行動する：
- 毎週月曜朝に自動で先週の業務サマリーを整理する（**定時タスク**）
- あなたが眉をひそめてため息をついているのを見て、困っていることがないか能動的に尋ねる（**察言観色後の行動**）
- 「宅配通知が来たら物流を確認して」と以前言ったことを覚えていて、毎回自動で確認する（**持続的に実行する習慣**）

重要な安全メカニズム：iFayが自主的に行ったことの結果が正しくない場合、**一時停止してあなたに確認する**。

## アーキテクチャ内の位置

```
iFay 4層アーキテクチャ
├── ソーシャルレイヤー
├── インタラクションレイヤー    ← 自律行動はここ
│   ├── 知覚（Sense）
│   │   └── セルフアウェアネス  ← あなたの意図を推論 ──┐
│   └── アクション（Action）                          │
│       ├── シミュレーテッドオペレーション              │
│       ├── スキル呼び出し                            │
│       └── 👉 自律行動      ← 自主意志 ←────────────┘
├── コグニションレイヤー
│   └── スキル
│       ├── 登録スキル       ← 永続スキルソースの一つ
│       └── 内部スキル       ← 永続スキルソースの二つ
└── エゴレイヤー
```

## 動作の仕組み

自律行動には**3つのトリガー方式**がある：

**1. 定時タスク——時間が来たら実行**
最もシンプルなトリガー方式。「毎週月曜朝9時に週報を生成して」と伝えれば、iFayは毎週月曜朝に定時実行する。

**2. セルフアウェアネス推論——あなたのニーズを察知して能動的に行動**
最も「インテリジェント」なトリガー方式。セルフアウェアネスモジュールであなたの状態と環境を観察し、何が必要かを推論して能動的に行動する。

**3. 永続スキル——バックグラウンドで継続実行する習慣**
一度きりではなく継続的に実行されるスキル。例えば「この株が50元を割ったら教えて」。このタスクは条件がトリガーされるまでバックグラウンドで実行され続ける。

**アクション→フィードバック→再アクションのサイクル**

1. **アクション**：iFayが自主的に決定したタスクを実行
2. **フィードバック**：実行結果を観察し、期待通りか判断
3. **再アクション**：期待通りなら次のステップへ；そうでなければ戦略を調整するか一時停止して確認

**安全弁：一時停止して確認**

iFayが自律行動の結果とあなたの意図が一致しないことを発見した場合、**後続のすべての自律アクションを一時停止**し確認する。このメカニズムがiFayの自律性の暴走を防ぐ。

## 他モジュールとの関係

| 関連モジュール | 関係 | 人体の類比 |
|---------|------|---------|
| **セルフアウェアネス** | ヒューマンプライム（Human Prime）の意図を推論し自律行動をトリガー | 内心の感受 → 行動を駆動 |
| **シミュレーテッドオペレーション** | 自律行動が「何をするか」を決定、シミュレーテッドオペレーションが「手で実行」 | 意志 → 手 |
| **スキル呼び出し** | 自律行動が「何をするか」を決定、スキル呼び出しが「電話で実行」 | 意志 → 口 |
| **登録スキル** | 永続スキルは自律行動のトリガーソースの一つ | 長期的習慣が行動を駆動 |
| **内部スキル** | 内部スキルは自律行動のもう一つのトリガーソース | 本能が行動を駆動 |
| **アラインド意識** | アラインド意識がプロファイルを提供し、自律行動が「すべきかどうか」を判断 | あなたへの理解が行動境界を決定 |

## シナリオストーリー

### シナリオ1：毎週月曜に自動で週報を生成（定時タスクトリガー）

月曜朝8:55、あなたはまだ地下鉄の中。iFayの定時タスクがトリガーされる：パーソナルデータヒープから先週の業務データを抽出し、ドキュメント生成APIを呼び出して週報を整理し、9:00にデスクトップに配置する。

### シナリオ2：能動的に食材を補充（セルフアウェアネストリガー）

水曜夜、iFayがスマート冷蔵庫のセンサーデータから発見：牛乳が1本だけ、卵が3個、野菜エリアがほぼ空。食習慣と購買記録を組み合わせて判断し、買い物リストを生成して注文する。ただし金額が通常予算を超えたため、一時停止して確認する——これが安全メカニズムの動作。

## 開発者向け

自律行動は**フェーズ4（iFay + coFay 完全な擬人化）**のコアモジュール。

- **要件番号**：要件14（自律行動）
- **インターフェース仕様**：`SelfDrivenBehavior`インターフェース、`scheduleTask()`、`handleInference()`、`pauseAndConfirm()`、`getLoopStatus()`の4つのコアメソッド
- **3つのトリガーソース**：`scheduled`（定時タスク）、`self_awareness`（セルフアウェアネス推論）、`registered_skill` / `internal_skill`（永続スキル）
- **コンプライアンステスト**：iFACTS L1で3つのトリガーソースの応答能力を検証、L2でセルフアウェアネスモジュールとのインターフェース対接を検証、L4で自律行動の安全制約（一時停止確認メカニズム）を検証
