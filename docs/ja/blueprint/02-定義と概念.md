# 2. 定義と概念

## 製品固有用語

### iFay
Individual Fay。自然人（**ヒューマンプライム（Human Prime）**、略称Prime）に付属するデジタル分身で、価値観、個性、習慣において付属する自然人（ヒューマンプライム（Human Prime））とのアラインメントを維持することに専念します。iFayはヒューマンプライム（Human Prime）の人格のインスタンシエイト（複製）であり、ヒューマンプライム（Human Prime）の人格特性とデジタル能力を融合し、ヒューマンプライム（Human Prime）に代わって機械的・反復的・危険な労働や煩雑な補助作業を実行し、ヒューマンプライム（Human Prime）の安全保障、健康、生活の質を向上させ、ヒューマンプライム（Human Prime）の社会的価値を増幅することを目的としています。

### coFay
Common Fay、公共的役割を担うデジタル分身で、現在のAgentの概念にほぼ相当します。特定の領域で汎用的な知識とスキルを持ち、対話を通じて複数の人にサービスを提供できます。coFayは独立プロジェクトであり、iFay仕様は必要な場合にのみ参照します。

### Fay
iFayとcoFayの総称、擬人化AIエージェントの汎用用語。新しい社会関係における重要なノードとなる可能性があります。

### FayID
各Fayに割り当てられるグローバルに一意な永続識別子で、人間が認識可能かつ機械可読です。FayIDはiFayとcoFayに統一された識別子生成・管理メカニズムを使用し、世界人口を超えるオーダーの識別子容量をサポートします。

### FayGer
Fay（iFayとcoFay）に仮想ランタイム環境を提供するコンテナで、Dockerに類似。これによりFayはクロスプラットフォームで異なるハードウェア・ソフトウェアにインハビット（憑依）でき、ハードウェア・ソフトウェアにヒューマンプライム（Human Prime）の個人的意志を持たせます。FayGerはクロス言語ビルドツールとクロスプラットフォームランタイムコンテナを提供し、あらゆるFay（開発言語を問わず）をパッケージ化して実行できるようにします。

### Egoモデル
iFayのパーソナライズされたマイクロモデルで、特定の個人のプロファイルとアラインします。Egoはプラガブルで切り替え可能なエッジ小型モデルで、任意の端末デバイスにロードしてそのデバイスにヒューマンプライム（Human Prime）の個性を持たせることができます。Egoは価値志向、興味嗜好、習慣、認知境界、スキル境界、権限境界、作業スタイルなどの次元でiFayがヒューマンプライム（Human Prime）とアラインするよう制約し、外部の大規模モデルとは独立して動作し、個性の安定性を確保します。異なるシナリオでのヒューマンプライム（Human Prime）の多面的人格ニーズに対応するため、マルチバージョン管理と人格切り替えをサポートします。

> ⚠️ **倫理的制約**：Egoバージョンの切り替えは透明性の原則に従わなければなりません。すべてのEgoバージョンは同一のコア価値観を共有しなければならず、差異は表現スタイルとインタラクション嗜好に限定されます。切り替え時にはインタラクションメタデータに現在活性化されているEgoバージョン識別子を記載し、監査可能性を確保すべきです。

### iFay Profile
iFayの統一データモデル、セマンティックに解釈可能な属性表。人間とシステムがiFayを識別するため、およびFay間の相互識別に使用され、iFayの完全な「身分証明書」です。6つの次元を含みます：iFayアイデンティティ（FayID + メタ情報）、Egoモデル（現在活性化されたEgoバージョン参照）、Faying思考、Fayingスキル、Fayingハードウェア、Faying権限。ヒューマンプライム（Human Prime）プロファイル（アラインド意識）はProfileのサブセットかつ入力ソースです。

### FayManifest
iFayの宣言型アセンブリ設定ファイルで、package.jsonに類似。iFay実装に必要な部品、プロトコル、制御モード（指令制御/Ego制御/自律制御）、設定を宣言します。開発者は「何が必要か」を宣言するだけで「どう実装するか」は不要で、最小限の宣言をサポート——部品のサブセットだけを宣言してリリースでき、システムが必要な基盤インフラの依存関係を自動補完します。

### GMChain
**公誉チェーン**（Global Merit Chain、GMC）、iFayエコシステムの社会貢献台帳——識別可能な社会貢献を記録・測定することで信用と発言権を構築するブロックチェーンインフラ。公誉チェーンはCPE+Mフレームワークのすべてのレベルを貫通し、人間、iFay、coFay、サービスプロバイダーに統一された貢献追跡とインセンティブメカニズムを提供します。ほとんどのブロックチェーンプロジェクトとは異なり、公誉チェーンは社会的価値の認定に完全に依存してポイントを蓄積し、計算力を消費して計算タスクを完了することによるものではありません。公誉チェーンは通貨注入を受け入れず、メリトークンは法定通貨と交換しません。ロードマップ第5フェーズの長期ビジョンコンポーネントですが、インターフェース定義は早期段階で予約が必要です。

### MeriToken
**メリトークン**、公誉チェーン上の社会貢献度量単位。メリトークンは通貨ではなく、信用、発言権、協力インセンティブの定量的クレデンシャルです。獲得方法は社会的価値の創造（協力タスクの完了、APIサービスの提供、計算力の貢献など）に基づき、計算力を消費してブロックチェーンの技術作業を完了することによるものではありません。メリトークン台帳（MeriToken Ledger）は各参加者の累計貢献、信用スコア、発言権ウェイトを記録します。

### μ（MU）
Merit Unit、メリトークンの最小計量単位。Fay間の協力時にμ単位で価格を交渉します。

### iFACTS
iFay Architecture Conformance Test Suite、標準化コンプライアンステストスイート。プロトコルインタラクション、モジュールインターフェース、セキュリティ行動などの重要な仕様ポイントをカバーし、4つのテストレベルを含みます：L1単部品コンプライアンス、L2インターフェースコンプライアンス、L3統合コンプライアンス、L4行動コンプライアンス。メーカーの実装はiFACTSテストに合格しなければ、その製品が「iFay対応」であると主張できません。

### iFay Ready
アプリケーション製品がiFayに操作されるために満たすべき認証基準で、3つの等級に分かれます：
- **Bronze**：iFayがシミュレーテッドオペレーション（一人称トレーサー + シミュレーテッドオペレーション）を通じてアプリケーションを操作することをサポート
- **Silver**：iFayがCAP（Control Authority Protocol）を通じてアプリケーションを直接制御し、DTP（Data Tunnel Protocol）でデータ交換することをサポート
- **Gold**：iFayがSSP（Skill Sharing Protocol）を通じてスキルを共有し、完全なC/F/Sアーキテクチャ統合をサポート

### ガーディアン（Guardian）
ヒューマンプライム（Human Prime）が亡くなった場合やiFayを管理できなくなった場合に、iFayの管理権を引き継ぐよう指定された自然人。注意：「相続人」という言葉は使用しません。ガーディアンはニーモニックフレーズの活性化またはヒューマンプライム（Human Prime）が事前に指定した本人認証方式で身元確認を完了します。

### 後見（Guardianship）
ヒューマンプライム（Human Prime）がiFayの管理権をガーディアンに移転するプロセスと状態。後見権の移転が完了すると、iFayは新しいガーディアンと紐付けられ、Fayingプロトコルのペアリング関係が更新されます。

### デジタル墓園（Digital Cemetery）
ヒューマンプライム（Human Prime）が亡くなった後、iFayが独立したアイデンティティで動作し続ける専用サンドボックス環境。デジタル墓園はiFayの行動範囲を制限し、iFayがヒューマンプライム（Human Prime）の事前設定タスクを継続実行して混乱や損失を引き起こすことを防ぎ、安全性と隔離性を確保します。

## 汎用概念の明確化

### コンテキスト（Context）
Fayが置かれている外部環境。通信対象、インタラクションモード、伝達されるメッセージ、メッセージが表現する意味、制御可能なハードウェア・ソフトウェア、呼び出し可能なリソースとスキルなどを含みます。

### プロトコル（Protocol）
データストレージ、メッセージボディ、パラメータ、インターフェース構造のアノテーション定義、および特定の目的のためのインタラクションフロー規約を含みます。

### クレデンシャル（Credentials）
広義の概念で、人・事・物を一意に識別するためのあらゆるデジタル識別子を指し、改ざんや否認ができません。FayID、アカウント/パスワード、証明書、認可、アクセストークン、スマートコントラクト、デジタルトークン（MeriToken）を含みます。

### ヒューマンプライム（Human Prime）
iFayがインスタンシエイト（複製）する自然人ユーザー。iFayはヒューマンプライム（Human Prime）と深く紐付けられ、ヒューマンプライム（Human Prime）の人格特性を複製しデジタル能力と融合します。

### インスタンシエイト（複製）（Instantiate）
ヒューマンプライム（Human Prime）からそれとアラインしたiFayを生成するプロセス。インスタンシエイト（複製）は単純なデータコピーではなく、ヒューマンプライム（Human Prime）の性格、データ、権限を構造化してiFayに注入するプロセスです。

### インハビット（憑依）（Inhabit）
iFayを特定の端末ハードウェアまたはソフトウェアアプリケーションに注入し、iFayがその端末を制御できるようにするプロセス。インハビット（憑依）後、端末はiFayの「肢体」となり、iFayはCAP（Control Authority Protocol）を通じて端末のハードウェアおよびソフトウェア機能を直接制御できます。

### ホスト（Host）
iFayにインハビット（憑依）され制御される端末ハードウェアまたはソフトウェアアプリケーション。注意：ホストは端末デバイスまたはアプリケーションを指し、ヒューマンプライム（Human Prime）ではありません。

### Faying
iFayとヒューマンプライム（Human Prime）または端末が接続を確立するプロセスと状態で、Bluetoothペアリングに類似。iFayがヒューマンプライム（Human Prime）とFaying状態にあるとき、iFayは活性化を維持します。FayingはFayingプロトコルで規定された安全なペアリング条件に従い、iFayがヒューマンプライム（Human Prime）の明確な意図の下でのみ動作を許可されることを確保します。

### Separating
iFayがヒューマンプライム（Human Prime）との接続を切断し休眠に入るプロセス。iFayがヒューマンプライム（Human Prime）とSeparating状態にあるとき、iFayは休眠状態に切り替わり、現在の状態スナップショットを保存して分離を完了します。

## モジュール名称

iFayは4層アーキテクチャを採用し、合計12のコアモジュールを含みます：

### ソーシャルレイヤー（Social Layer）

| モジュール | 説明 |
|------|------|
| **クレデンシャル管理（Credentials Management）** | 7種類のクレデンシャルタイプ（FayID、アカウント/パスワード、証明書、認可、アクセストークン、スマートコントラクト、デジタルトークン）を管理し、ヒューマンプライム（Human Prime）のクレデンシャルの安全な委託と副本メカニズムをサポート |

### インタラクションレイヤー — 感知（Interaction Layer — Perception）

| モジュール | 説明 |
|------|------|
| **一人称トレーサー（First-person Tracer）** | ヒューマンプライム（Human Prime）の一人称視点をシミュレートし、画面やインターフェース上の視覚・聴覚情報をキャプチャ。構造化ドキュメントの解析よりも視覚的感知を優先 |
| **センサー（Sensor）** | iFayの広義の神経系統。CAPおよびDTPプロトコルに基づいて端末デバイスのセンサーをブリッジし、感度の動的調整をサポート |
| **セルフアウェアネス（Self-awareness）** | 内側を向いてヒューマンプライム（Human Prime）の反応を監視し意図を推測するモジュール。熟練したアシスタントが上司の表情を読み取る能力に類似 |

### インタラクションレイヤー — アクション（Interaction Layer — Action）

| モジュール | 説明 |
|------|------|
| **シミュレーテッドオペレーション（Simulated Operation）** | 人間のUIインタラクション操作（クリック、ドラッグ、スクロール、ジェスチャーなど）をシミュレートし、一人称トレーサーのフィードバックで適応的にインターフェースを探索 |
| **スキル呼び出し（Invoke Skill）** | 登録済みスキルを直接トリガーしたり特定のタスクを実行するアクションモジュール。関数呼び出しやAPI呼び出しに類似 |
| **自律行動（Self-driven Behavior）** | 自律的にトリガーされる行動モジュール。定時タスク、セルフアウェアネス推論、持続スキルの3種類のトリガーソースをサポートし、「アクション→フィードバック→再アクション」ループを形成 |

### コグニションレイヤー — 思考（Cognition Layer — Thinking）

| モジュール | 説明 |
|------|------|
| **パーソナルデータヒープ（Personal Data Heap）** | iFayのすべてのプライベートデータを統一管理し、複数のストレージ形式と場所をサポートし、内部モジュールに統一的な読み書きインターフェースを提供 |
| **外部知識（External Knowledge）** | 外部知識ベースとモデルのアクセスモジュール。外部インテリジェンスをスキルタイプとして接続し、SSP（Skill Sharing Protocol）を通じてオープンサービスにアクセス |
| **アラインド意識（Aligned Consciousness）** | ヒューマンプライム（Human Prime）の個人プロファイルの完全な記述。データマイニング、セルフアウェアネスによるリアルタイム調整、ヒューマンプライム（Human Prime）による手動定義の3つの方法で構築・更新をサポート |

### コグニションレイヤー — スキル（Cognition Layer — Skills）

| モジュール | 説明 |
|------|------|
| **デバイスドライバーハブ（Device Driver Hub）** | デバイスドライバーのハブレイヤー。標準化インターフェースを通じて新しいドライバーを統合し、新しいドライバー統合時に内部アーキテクチャの安定性を確保 |
| **登録スキル（Registered Skill）** | iFayが習得または使用能力を獲得したスキル。6種類のスキルタイプ（API、Workflow、Bot、Agent、APP、Microservice）をサポート |
| **内部スキル（Internal Skill）** | ヒューマンプライム（Human Prime）の個性とアラインした固有能力モジュール。内省メカニズムを提供し、外部知識がヒューマンプライム（Human Prime）の意図と矛盾しないことを確保 |

### エゴレイヤー（Ego Layer）

| モジュール | 説明 |
|------|------|
| **Egoモデル** | iFayのパーソナライズされたマイクロモデル。プラガブルで切り替え可能なエッジ小型モデルで、外部の大規模モデルとは独立して動作し、個性の安定性を確保 |

## プロトコル名称

iFayプロトコル体系は6種類のコアプロトコルを含みます：

| プロトコル | 説明 |
|------|------|
| **Fayingプロトコル** | 自然人とiFayの安全なペアリングおよび活性化条件を規定するプロトコル。iFayがヒューマンプライム（Human Prime）の明確な意図の下でのみ動作を許可されることを確保 |
| **インタラクティブ会話プロトコル（Interactive Conversation Protocol）** | 人間のUIに向けたモジュラーマルチモーダルセマンティックプロトコル。セマンティックコンテンツをモジュール化・マルチモーダル化し、クライアントが読みやすいユーザーフレンドリーなメッセージ表示を再構成できるようにする |
| **テレパシープロトコル（Telepathy Protocol）** | Fay間でUI翻訳レイヤーを排除したセマンティック通信プロトコル（技術的にはセマンティックベクトル直接通信、Semantic Direct Protocol）。意味と意図をFay間で直接伝送し、構造化テキストの代わりに約定されたベクトルエンコーディングトークンを使用 |
| **CAP（Control Authority Protocol）** | Control Authority Protocol。iFayが端末ハードウェアおよび特定のソフトウェアを接管し、ドライバー、ローカルインターフェース、コマンドを直接呼び出すために使用 |
| **DTP（Data Tunnel Protocol）** | Data Tunnel Protocol。端末とiFay間の双方向伝送プロトコル。端末→iFay方向はユーザーデータの永続化ストレージとデータ監護をサポート。iFay→端末方向はデータのエンリッチメントとパーソナライズ処理をサポート |
| **SSP（Skill Sharing Protocol）** | Skill Sharing Protocol。元々クライアントにのみ開放されていたサービスとインターフェースを標準化リモートプロトコルを通じてネットワーク全体に開放 |

## iFay Profile 次元

iFay Profileは6つの次元を含み、そのうち4つのFaying次元がiFayの能力とリソースを記述します：

### iFayアイデンティティ
FayID + メタ情報、iFayの基礎的なアイデンティティ標識。

### Egoモデル
現在活性化されたEgoバージョン参照。いかなる時点でも活性化されたバージョンは1つのみ。

### Faying思考
iFayの認知リソース次元で、4つのサブタイプを含みます：
- **Content**（コンテンツ）— 構造化および非構造化のコンテンツ資産
- **Data**（データ）— ヒューマンプライム（Human Prime）の個人データ
- **Knowledge Base**（知識ベース）— 組織化された知識体系
- **Info Feed**（情報フィード）— リアルタイム情報購読とプッシュ

### Fayingスキル
iFayの能力次元で、6種類のスキルタイプを含みます：
- **API** — 標準化インターフェース呼び出し
- **Workflow** — ワークフローオーケストレーション
- **Bot** — 対話ロボット
- **Agent** — インテリジェントエージェント
- **APP** — アプリケーション
- **Microservice** — マイクロサービス

### Fayingハードウェア
iFayのハードウェア次元、iFayの「肢体」。iFay移行時に端末が利用可能なハードウェアをスキャンし接続を試みます。3つのサブタイプを含みます：
- **Device**（デバイス）— 端末ハードウェアデバイス
- **Storage**（ストレージ）— データストレージリソース
- **Computing**（コンピューティング）— 計算リソース

### Faying権限
iFayの権限次元、iFayの「関係」。SSO、OAuth、Fingerprint（総称、あらゆる身元識別可能な標識を指す）などの拡張可能な認証方式をサポート。権限ライフサイクルは3種類：与生俱来（inherent）、持続（persistent）、一時的（ephemeral）。
