# 11.2 外部知識

## 一言定義

外部知識はiFayの**図書館と専門家アドバイザー**——パーソナルデータヒープがiFay自身の記憶なら、外部知識はiFayが参照できる図書館であり相談できる専門家である。iFayにヒューマンプライム（Human Prime）自身の能力を超える知識を持たせる。

## なぜ必要か

パーソナルデータヒープが保存するのはあなた自身のデータと記憶だが、あなたの知識には限りがある。外部知識モジュールはiFayが**自分の記憶の外に出て**、外部のナレッジベースを参照し、外部のインテリジェントモデルに相談できるようにする。

## アーキテクチャ内の位置

```
iFay 4層アーキテクチャ
├── コグニションレイヤー    ← 外部知識はここ
│   ├── 思考（Thought）
│   │   ├── パーソナルデータヒープ  ← 自分の記憶
│   │   ├── 👉 外部知識            ← 図書館と専門家アドバイザー
│   │   └── アラインド意識          ← あなたへの理解
│   └── スキル（Skill）
│       ├── 登録スキル             ← 外部知識ソースも一種の「スキル」
│       └── ...
└── エゴレイヤー
```

外部知識ソースはiFayの体系では一種の**スキルタイプ**として扱われる。外部知識の接続は他のスキルと同じ登録、認可、呼び出しフローに従う。

## 動作の仕組み

**1. 接続——外部知識ソースに接続**

知識ソースの種類：ナレッジベース、大規模言語モデル、エキスパートシステム、検索エンジン。これらはSSP（Skill Sharing Protocol）を通じてiFayに接続される。

**2. 照会——専門家に相談するように質問**

各照会結果には**確信度**が付与され、iFayは複数の知識ソースの回答を総合して最も信頼性の高い情報を選択できる。

**3. 統合——学んだ知識を自分のものに**

外部知識ソースから取得した情報はパーソナルデータヒープに統合され、iFay自身の記憶となる。

**知識ソースが利用不可の場合**

外部知識モジュールは**パーソナルデータヒープのキャッシュ知識にフォールバック**し、コグニションレイヤーに「現在キャッシュ知識を使用中、最新でない可能性あり」と通知する。

## 他モジュールとの関係

| 関連モジュール | 関係 | 人体の類比 |
|---------|------|---------|
| **パーソナルデータヒープ** | 外部知識が取得した情報を統合 | 図書館で学んだ知識 → ノートに記録 |
| **登録スキル** | 外部知識ソースはスキルタイプとして登録・管理 | 図書館の貸出カードも一種の「スキル証明」 |
| **内部スキル** | 内部スキルが外部知識とヒューマンプライム（Human Prime）の意図の矛盾をチェック | 自分の判断力で外部情報の信頼性を評価 |
| **SSPプロトコル** | 外部知識ソースはSSPプロトコルで標準化接続 | 図書館の統一貸出ルール |

## シナリオストーリー

### シナリオ1：健康診断レポートの解読

健康診断レポートの指標を理解するため、iFayがパーソナルデータヒープから過去3年の健康診断データを取得し、外部の医学ナレッジベースに照会して、パーソナライズされた健康アドバイスを提供する。

### シナリオ2：契約条項の起草

法律ナレッジベースに照会し、過去の契約テンプレートを参考にして、違約条項の草稿を生成する。同時に「正式署名前に専門弁護士のレビューを推奨」と注意喚起する。

## 開発者向け

外部知識は**フェーズ3（iFayを仮想世界のインターフェースに）**のコアモジュール。

- **要件番号**：要件12（外部知識接続）
- **インターフェース仕様**：`ExternalKnowledge`インターフェース、`query()`、`registerSource()`、`fallbackToCache()`の3つのコアメソッド
- **知識ソースタイプ**：`knowledge_base`、`llm`、`expert_system`、`search_engine`
- **関連プロトコル**：SSP（Skill Sharing Protocol）
- **フォールバック戦略**：外部知識ソース利用不可時、パーソナルデータヒープのキャッシュ知識に自動フォールバックし、コグニションレイヤーに通知
- **コンプライアンステスト**：iFACTS L1で知識照会とキャッシュフォールバック能力を検証、L2でパーソナルデータヒープとの知識統合インターフェースを検証、L3で「知識照会→統合→キャッシュ→フォールバック」の完全なリンクを検証
