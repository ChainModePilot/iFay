# 8. ソーシャルレイヤー

ソーシャルレイヤーはiFayの社会における**存在方式**を定義します——3つの根本的な問いに答えます：「私は誰か」（アイデンティティ）、「何をする権限があるか」（社会権限）、「私の貢献はどれだけの価値があるか」（社会貢献と発言権）。

ソーシャルレイヤーはiFayのセキュリティ体系の基盤であり、iFayが社会関係に融合するための前提条件です。

```
iFay 4層アーキテクチャ
├── 👉 ソーシャルレイヤー
│   ├── 8.1 アイデンティティ（FayID）    ← 私は誰か
│   ├── 8.2 社会権限                      ← 何をする権限があるか
│   │   ├── アカウント/パスワード
│   │   ├── 証明書（Certificate）
│   │   ├── 認可（Authorization）
│   │   ├── アクセストークン
│   │   └── スマートコントラクト
│   └── 8.3 社会貢献と発言権              ← 私の貢献の価値
│       ├── 公誉チェーン（GMChain）
│       └── メリトークン（MeriToken）
├── インタラクションレイヤー
├── コグニションレイヤー
└── エゴレイヤー
```

---

## 8.1 アイデンティティ（FayID）

### 一言定義

FayIDはiFayの**ID番号**——グローバルに一意、人間が読め、機械が読める永続的識別子であり、すべての社会的インタラクションにおいてiFayが識別・追跡される基盤です。

### なぜ必要か

FayIDの設計には2つの重要な考慮事項があります：
1. **統一識別**：iFayとcoFayは同一のFayID生成・管理メカニズムを使用します。
2. **アクティベーションバインディング**：FayIDはFayingプロトコルと深く結合し、iFayがヒューマンプライムの明確な認可の下でのみ動作することを保証します。

### コア特性

| 特性 | 説明 |
|------|------|
| **グローバル一意** | 世界人口を超える規模の識別子容量をサポート |
| **人間可読** | 人間が直感的に識別・記憶できる |
| **機械可読** | システムとFay間で効率的に解析・マッチング可能 |
| **永続性** | 一度割り当てられると生涯変わらない |
| **アクティベーション状態** | Fayingプロトコルと連動 |

### 開発者向け

- **独立プロジェクト**：FayIDは[独立オープンソースプロジェクト](https://github.com/ChainModePilot/FayID)
- **インターフェース仕様**：`FayIDRegistry`インターフェース、`generate()`、`resolve()`、`getActivationStatus()`を含む
- **コンプライアンステスト**：iFACTS L1でFayIDの一意性と永続性を検証、L2でFayingプロトコルとのアクティベーション状態連動を検証

## 8.2 社会権限

### 一言定義

社会権限はiFayの**キーチェーン**——iFayがあなたの代わりにサービスを利用するために必要なクレデンシャルを安全に管理します。これらのクレデンシャルは原本ではなく安全な副本です。

### 5種類の権限タイプ

#### 1. アカウント/パスワード（Account / Password）
最も一般的なクレデンシャルタイプ。副本メカニズムを通じて使用します。

#### 2. 証明書（Certificate）
デジタル証明書。暗号化通信と身元認証に使用。X.509等の標準フォーマットをサポート。

#### 3. 認可（Authorization）
OAuth 2.0等の従来のサードパーティ認可方式と互換。

#### 4. アクセストークン（Access Token）
有効期限付きの一時的なアクセスクレデンシャル。API呼び出し等に使用。

#### 5. スマートコントラクト（Smart Contract）
ブロックチェーン上のスマートコントラクトクレデンシャル。分散型サービスとFay間の自動化協力に使用。

### 副本メカニズムとセキュリティ

- **副本分離**：iFayは元のクレデンシャルを直接保持せず、権限制限付きの副本のみを保持
- **所有者マーキング**：各クレデンシャルの所有者がヒューマンプライムかiFay自身かを明示
- **監査追跡**：すべてのクレデンシャル使用を記録
- **異常検出**：異常使用パターン検出時に自動的に副本を取り消し
- **プライバシー保護**：プライバシーデータ照会時はブール結果のみを返却

### 開発者向け

- **インターフェース仕様**：`CredentialManager`インターフェース、`delegate()`、`authenticate()`、`revoke()`、`queryPrivacy()`、`getAuditLog()`を含む
- **5種類のクレデンシャルタイプ**：`account_password`、`certificate`、`authorization`、`access_token`、`smart_contract`
- **コンプライアンステスト**：iFACTS L1で5種類のクレデンシャル管理能力と副本メカニズムを検証、L4でプライバシー照会のブール返却制約と異常検出行動を検証

## 8.3 社会貢献と発言権（公誉チェーン（GMChain） と メリトークン（MeriToken））

### 一言定義

公誉チェーン（GMChain、Global Merit Chain）はiFayエコシステムの**社会貢献台帳**——すべてのFayと人間が社会に創造した価値を記録し、メリトークン（MeriToken）でこれらの貢献を定量化します。メリトークンは通貨ではなく、**発言権と信用の尺度**です。

### なぜ必要か

メリトークンの役割：
- **貢献の測定**：あなたのiFayが社会にどれだけの価値を創造したか
- **信用の構築**：貢献が多いほど信用が高く、協力ネットワークでより信頼される
- **発言権の獲得**：コミュニティ合意が必要な意思決定で、貢献者がより大きな発言権を持つ
- **協力のインセンティブ**：Fay間の協力時、貢献が公正に記録・報酬される

### コアメカニズム

**1. 貢献記録** — Fay間の協力タスク、情報組立サービス、API・スキルサービス、デバイス・計算力リソース等を記録

**2. 価格交渉** — Fayは協力タスク実行前にμ単位で価格を事前交渉

**3. メリトークン台帳（MeriToken Ledger）** — 累計貢献総量、貢献タイプ分布、信用スコア、発言権ウェイトを記録

### 従来の暗号通貨との本質的な違い

| 次元 | 従来の暗号通貨 | メリトークン |
|------|-------------|------------|
| **獲得方法** | 計算力消費（マイニング） | 社会的価値の創造 |
| **価値アンカー** | 市場需給と投機 | 社会貢献度 |
| **目的** | 金融資産 | 信用度量・発言権・協力インセンティブ |
| **法定通貨との関係** | 交換可能 | 交換不可 |

### 長期ビジョンと現在の位置づけ

> ⚠️ **重要説明**：公誉チェーン/メリトークンは完全AI化社会の長期ビジョン産物であり、ロードマップ第5フェーズのコアコンポーネントです。公誉チェーンは通貨注入を受け入れず、メリトークンは法定通貨と交換しません。

### 開発者向け

- **独立プロジェクト**：公誉チェーンは[独立オープンソースプロジェクト](https://github.com/ChainModePilot/Global-Merit-Chain)
- **インターフェース仕様**：`MeritLedger`インターフェース、`recordContribution()`、`getBalance()`、`getReputation()`、`negotiate()`を含む
- **計量単位**：μ（MU、Merit Unit）
- **コンプライアンステスト**：iFACTS L4で貢献記録の改ざん不可能性と価格交渉の公正性を検証
