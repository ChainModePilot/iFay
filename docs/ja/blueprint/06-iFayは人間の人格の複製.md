# 6. iFayは人間の人格のインスタンシエイト（複製）

iFayの個性は憑空から生成されるものではない——それはヒューマンプライム（Human Prime、略称Prime）からインスタンシエイト（複製）される。すべてのiFayはそのヒューマンプライム（Human Prime）の人格の延長であり、この延長には構造化された注入プロセスが必要である。

本ページでは、ヒューマンプライム（Human Prime）が3つの次元から自身の特性をiFayに注入する方法を説明する：**性格**がEgoモデルを形成し、**データ**がパーソナルデータヒープを充填し、**権限**がクレデンシャル委託を確立する。3つの次元は不可欠である——性格はiFayに「私は誰か」を知らせ、データはiFayに「私の過去」を与え、権限はiFayが「私の代わりに行動」できるようにする。

---

## ヒューマンプライム（Human Prime）の性格（→ Egoモデル）

ヒューマンプライム（Human Prime）の性格はEgoモデルの訓練と設定入力である。Egoモデルはi Fayの内蔵マイクロモデルであり、iFayが価値志向、興味・嗜好、習慣、認知境界、スキル境界、権限境界、ワークスタイルなどの次元でヒューマンプライム（Human Prime）と整合を保つよう制約する。

### 3つの注入方式

| 方式 | 説明 |
|------|------|
| **対話による語り** | ヒューマンプライム（Human Prime）がiFayとガイド付き対話を行い、iFayが標準研究アンケートで対話トピック範囲を制約し、有効な性格データの収集を確保する。これは雑談ではない——各ラウンドの対話は特定の性格次元を中心に展開される。 |
| **アンケート記入** | ヒューマンプライム（Human Prime）が構造化された性格アンケートに直接回答し、システムが回答をEgoモデルの訓練パラメータに変換する。 |
| **直接設定** | 上級ユーザーはEgoモデルの制約パラメータを直接編集し、iFayの行動境界を精密に制御できる。 |

### マルチバージョンEgo

人間は多面的である。仕事では厳格で集中し、友人の前ではリラックスしてユーモラスで、家族のそばでは優しく忍耐強い。Egoモデルは複数のバージョンをサポートし、各バージョンがヒューマンプライム（Human Prime）の人格の一側面に対応する。任意の時点で1つのEgoバージョンのみがアクティブ状態にあり、手動切り替えとコンテキストに基づく自動切り替えをサポートする。

### シナリオ

> Isabelが自分のiFayをアクティベートしたばかり。システムが彼女を対話に導く——ランダムなチャットではなく、標準研究アンケートに基づいて設計されたガイド付き交流である。
>
> 「曖昧な表現のビジネスメールを受け取った時、直接返信して明確にする傾向がありますか、それとも先に非公式に相手と話しますか？」
>
> Isabelは後者を選び、非公式な場で潜在的な誤解を解決する方が好きだと補足した。iFayはこの嗜好を記録し、Egoモデルの「コミュニケーションスタイル」次元のパラメータとしてエンコードする。
>
> 20分後、IsabelのiFayは彼女のコミュニケーション嗜好、意思決定スタイル、リスク態度を初歩的に理解した。これらの特性は、iFayが彼女の代わりに日常業務を処理する際の行動指針となる。

## ヒューマンプライム（Human Prime）のデータ（→ パーソナルデータヒープ）

データはiFayの「記憶」である。データのないiFayは記憶喪失の人のようなものだ——自分が誰かは分かるが、何を経験したかは覚えていない。パーソナルデータヒープはiFayがすべてのプライベートデータを統一管理するコンポーネントであり、ヒューマンプライム（Human Prime）データはその初期化入力である。

### 4種類のデータタイプ

| タイプ | 説明 | 例 |
|------|------|------|
| **コンテンツ（Content）** | 構造化または非構造化のコンテンツ資産 | 写真、動画、記事、創作作品 |
| **データ（Data）** | 個人データ | アプリ使用記録、健康データ、消費記録 |
| **ナレッジベース（Knowledge Base）** | 組織化された知識体系 | 専門ノート、学習資料、業務ドキュメント |
| **情報フロー（Info Feed）** | リアルタイム情報購読 | ニュースソース、業界動向、ソーシャルメディアフォロー |

### 3つの注入方式

1. **ヒューマンプライム（Human Prime）データの認可**：iFayにヒューマンプライム（Human Prime）の写真、アプリデータ、通信記録などへのアクセスを認可する。システムはOSレイヤーと緊密に統合し、システム監視下でのシステム情報や他のアプリのプライベートデータの読み取りをサポートする。
2. **ファイルのインポート**：ドキュメント、ノート、チャット履歴などの長期記憶ファイルをインポートする。
3. **情報ソースの指定**：書籍、情報ソース、ナレッジベースを指定し、iFayがこれらのソースから継続的に情報を取得できるようにする。

すべてのデータは標準フォーマットでパーソナルデータヒープに保存され、iFayのみが直接アクセスできる。これは一度きりの初期化操作だが、長期的な成長をサポートする——iFayはヒューマンプライム（Human Prime）の生活に伴い継続的に新しいデータを蓄積する。

### シナリオ

> 陳明は過去10年のデジタルライフを自分のiFayに託すことにした。
>
> まずiFayにスマートフォンのフォトアルバムへのアクセスを認可した——30,000枚以上の写真と動画で、大学卒業から子供の誕生まですべての重要な瞬間を記録している。次にWeChatのチャット履歴、業務メールアーカイブ、Notionノートライブラリをインポートした。
>
> iFayがこれらのデータを処理し終えると、陳明が去年どこに旅行したかだけでなく、彼の社交圏、仕事のリズム、生活習慣も理解した。陳明の同僚が前回のチームビルディングのレストラン名を尋ねた時、iFayは数秒で写真のジオタグとチャット履歴から答えを見つけることができた——陳明自身が思い出したかのように。

## ヒューマンプライム（Human Prime）の権限（→ クレデンシャル委託）

権限はiFayが「あなたの代わりに何ができるか」を決定する。権限のないiFayは鍵のない執事のようなものだ——何をすべきかは分かるが、どのドアも開けられない。

### 3つの権限ソース

iFayの権限には3つのソースがあり、それらの間に包含関係はない：

| ソース | 説明 | 例 |
|------|------|------|
| **ヒューマンプライム（Human Prime）の直接認可** | ヒューマンプライム（Human Prime）が既存のクレデンシャルをiFayに委託して使用させる | Netflixアカウント、銀行照会権限の委託 |
| **ヒューマンプライム（Human Prime）の権限交換** | ヒューマンプライム（Human Prime）の権限交換から派生した権限 | ヒューマンプライム（Human Prime）の企業アカウントから交換したAPIアクセス権 |
| **iFayの自主取得** | iFayが自身の行動を通じて取得した独立権限 | GMChain貢献で得たAPIアクセス権、Fayソーシャルネットワークで確立した信頼クレデンシャル |

### 包含関係なし

重要な設計：ヒューマンプライム（Human Prime）の権限とiFayの権限の間に**包含関係はない**。iFayはヒューマンプライム（Human Prime）が持たない権限を持つことができ（例：iFayが貢献で得たAPIアクセス権）、ヒューマンプライム（Human Prime）もiFayに委託していない権限を持つことができる（例：ヒューマンプライム（Human Prime）のプライベート銀行口座）。

### 7種類のクレデンシャルタイプ

iFayは以下の7種類のクレデンシャルタイプの管理をサポートする：FayID、アカウント・パスワード、証明書、認可、アクセストークン、スマートコントラクト、デジタルトークン（MeriToken）。ヒューマンプライム（Human Prime）がクレデンシャルを委託する際、元のクレデンシャルは対応する副本クレデンシャルに交換され、iFayは副本を使用してログインと認証を行い、元のクレデンシャルの安全を確保する。

### シナリオ

> 李薇は自分のNetflixアカウントをiFayに委託した——これは**ヒューマンプライム（Human Prime）の直接認可**である。iFayが受け取ったのは李薇の元のパスワードではなく、対応する副本クレデンシャルである。これでiFayは李薇の視聴嗜好に基づいてプレイリストを自動管理できる。
>
> 同時に、李薇のiFayはGMChainを通じて他のFayに情報アセンブリサービスを提供し続け、相当なMeriTokenを蓄積した。これらの貢献により、iFayはプレミアム天気データAPIのアクセス権を取得した——これは**iFayの自主取得**した権限であり、李薇本人はこのAPIのアクセス権を持っていない。
>
> しかし李薇のプライベート投資口座は、iFayに委託しないことを選んだ。これは全く問題ない——ヒューマンプライム（Human Prime）とiFayの権限境界は独立している。
