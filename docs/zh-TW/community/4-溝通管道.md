# 4. 溝通管道

iFay 社群提供多種溝通渠道，以滿足不同場景下的協作需求。

## GitHub Discussions

GitHub Discussions 是 iFay 社群的主要非同步討論平臺，適用於：

- SEP（規範增強提案）的公開討論
- 技術問題的深入探討
- 功能請求和設計方案討論
- 各工作組的日常交流

我們鼓勵將重要的技術討論放在 GitHub Discussions 上進行，以確保討論內容可追溯、可搜尋。

## GitHub Issues

GitHub Issues 用於具體的任務跟蹤和問題管理：

- Bug 報告
- 具體的開發任務
- 文件改進建議
- iFACTS 測試用例問題

提交 Issue 時，請儘量提供詳細的復現步驟或上下文資訊，以便維護者快速定位和處理。

## Discord

Discord 是 iFay 社群的實時溝通平臺。各工作組擁有獨立的頻道：

- `#general`：通用討論
- `#wg-fayid`：FayID 工作組
- `#wg-ego`：Ego 工作組
- `#wg-protocols`：協議工作組
- `#wg-fayger`：FayGer 工作組
- `#wg-ifacts`：iFACTS 工作組
- `#wg-docs`：文件工作組
- `#wg-community`：社群工作組

Discord 適合快速交流和非正式討論。但請注意，重要的技術決策應在 GitHub 上記錄，以確保透明和可追溯。

## 定期會議

- **核心維護者月度會議**：每月舉行一次，公開進行，所有社群成員均可旁聽。會議議程和紀要會在 GitHub 上釋出。
- **工作組會議**：各工作組可自行安排會議頻率和形式，建議至少每月一次同步進展。

## 郵件列表

郵件列表用於釋出重要公告和通知：

- 新版本釋出通知
- SEP 狀態變更通知
- 重大治理決策公告
- 社群活動通知

訂閱郵件列表可確保你不會錯過重要資訊。

## 行為準則

所有溝通渠道均遵循 iFay 行為準則。我們致力於營造一個友好、包容、專業的社群環境。請在所有互動中保持尊重和建設性。

## 語言

英語是 iFay 社群的主要工作語言，但我們歡迎使用其他語言參與討論。如果你更習慣使用中文、德語或其他語言，請放心使用——社群成員會協助翻譯，確保你的想法被準確傳達。

我們的目標是讓語言不成為參與 iFay 社群的障礙。
