# 8. 社交層

社交層定義了 iFay 在社會中的**存在方式**——它回答三個根本問題：「我是誰」（身份標識）、「我有權做什麼」（社會權限）、「我的貢獻值多少」（社會貢獻與話語權）。

如果把 iFay 比作一個社會人，社交層就是他的**社會身份**：身份證證明你是誰，各種證件和鑰匙證明你有權做什麼，而你的社會聲譽和貢獻記錄決定了你在社區中的話語權和可信度。

社交層是 iFay 安全體系的基石，也是 iFay 融入社會關係的前提。

```
iFay 四層架構
├── 👉 社交層
│   ├── 8.1 身份標識（FayID）        ← 我是誰
│   ├── 8.2 社會權限                  ← 我有權做什麼
│   │   ├── 帳密託管（Account/Pwd）
│   │   ├── 證書（Certificate）
│   │   ├── 鑑權（Authorization）
│   │   ├── 存取秘鑰（Access Token）
│   │   └── 智能合約（Smart Contract）
│   └── 8.3 社會貢獻與話語權          ← 我的貢獻值多少
│       ├── 公譽鏈（GMChain）
│       └── 美譽值（MeriToken）
├── 互動層
├── 認知層
└── 自我層
```

## 8.1 身份標識（FayID）

### 一句話定義

FayID 是 iFay 的**身份證號碼**——全域唯一、人類可讀、機器可讀的持久標識符，是 iFay 在所有社會互動中被識別和追蹤的基礎。

### 為什麼需要它

在人類社會中，身份證號碼是一切社會活動的起點。FayID 對 iFay 的意義完全相同：它是 iFay 參與一切社會互動的前提。

FayID 的設計有兩個關鍵考量：

1. **統一標識**：iFay 和 coFay 使用同一套 FayID 生成和管理機制。
2. **啟動綁定**：FayID 與 Faying 協議深度綁定，確保 iFay 僅在人類原型明確授權下運行。

### 核心特性

| 特性 | 說明 |
|------|------|
| **全域唯一** | 支持超過全球人口數量級的標識符容量，確保永不衝突 |
| **人類可讀** | 人類能直觀識別和記憶 |
| **機器可讀** | 系統和 Fay 之間能高效解析和匹配 |
| **持久性** | 一旦分配，終身不變，支持身份遷移和人格延續 |
| **啟動狀態** | 與 Faying 協議聯動，明確標識 iFay 當前是活躍還是休眠 |

### 對開發者

- **獨立專案**：FayID 是[獨立開源專案](https://github.com/ChainModePilot/FayID)，詳見 [FayID 文件](https://github.com/ChainModePilot/FayID/wiki)
- **介面規範**：`FayIDRegistry` 介面，包含 `generate()`（生成 FayID）、`resolve()`（解析 FayID）、`getActivationStatus()`（取得啟動狀態）
- **合規性測試**：iFACTS L1 驗證 FayID 的唯一性和持久性，L2 驗證與 Faying 協議的啟動狀態聯動

## 8.2 社會權限

### 一句話定義

社會權限是 iFay 的**鑰匙串**——它安全地管理著 iFay 代替你使用各種服務所需的憑證。但這些憑證不是你的原件，而是安全副本——即使副本丟了，你的原始憑證依然安全。

### 為什麼需要它

iFay 要代替你做事——登入電商平台買東西、用公司信箱發郵件、操控智慧家居、呼叫第三方 API。做這些事都需要「鑰匙」。社會權限模組就是幫你安全地管理這些鑰匙，並透過**副本機制**確保原始憑證永遠不會直接暴露。

### 五種權限類型

#### 1. 帳密託管（Account / Password）

最常見的憑證類型。人類原型將應用和服務的帳號密碼委託給 iFay，iFay 透過副本機制使用這些憑證登入和操作。

#### 2. 證書（Certificate）

數位證書，用於加密通訊和身份驗證。兼容現有的證書頒發機構（CA）體系，支持 X.509 等標準格式。

#### 3. 鑑權（Authorization）

兼容傳統的第三方鑑權方式（如 OAuth 2.0），允許 iFay 透過授權的要素去第三方平台認證，取得受限的存取權限。

#### 4. 存取秘鑰（Access Token）

託管的臨時存取憑據，通常有有效期。用於 API 呼叫、服務存取等場景。

#### 5. 智能合約（Smart Contract）

區塊鏈上的智能合約憑證，用於去中心化服務和 Fay 間的自動化協作。

### 副本機制與安全防線

所有五種權限類型都遵循統一的安全原則：

- **副本隔離**：iFay 永遠不直接持有原始憑證，只持有權限受限的副本
- **所有者標記**：每個憑證標明所有者是人類原型還是 iFay 本身
- **稽核追蹤**：每次憑證使用都被記錄，支持事後稽核
- **異常偵測**：偵測到異常使用模式時自動撤銷副本並通知人類原型
- **隱私保護**：當 iFay 被授權查詢隱私資料時，僅回傳布林結果（真或假），不暴露實際資料

### 對開發者

- **介面規範**：`CredentialManager` 介面，包含 `delegate()`（委託/建立副本）、`authenticate()`（認證）、`revoke()`（撤銷）、`queryPrivacy()`（隱私查詢，僅回傳布林值）、`getAuditLog()`（稽核日誌）
- **五種憑證類型**：`account_password`、`certificate`、`authorization`、`access_token`、`smart_contract`
- **憑證所有者**：`human_prime`（人類原型）或 `ifay`（iFay 本身）
- **副本狀態**：`active`（活躍）、`revoked`（已撤銷）、`expired`（已過期）
- **合規性測試**：iFACTS L1 驗證五種憑證類型的管理能力和副本機制，L2 驗證與註冊技能和技能調用的認證介面，L4 驗證隱私查詢的布林回傳約束和異常偵測行為

## 8.3 社會貢獻與話語權（公譽鏈（GMChain） 與 美譽值（MeriToken））

### 一句話定義

公譽鏈（GMChain，Global Merit Chain）是 iFay 生態的**社會貢獻帳本**——它記錄每一個 Fay 和人類為社會創造的價值，並以美譽值（MeriToken）量化這些貢獻。MeriToken 不是貨幣，而是**話語權和信譽的度量**。

### 為什麼需要它

在 Fay 生態中，大部分工作由 Fay 完成，每一次協作、每一個貢獻都可以被精確追蹤和記錄。公譽鏈（GMChain）就是這個追蹤和記錄系統——它讓「誰貢獻了什麼」變得透明、可驗證、不可篡改。

美譽值（MeriToken）是公譽鏈上的計量單位，代表社會貢獻度。它的作用不是「買東西」，而是：
- **衡量貢獻**：你的 iFay 為社會創造了多少價值？
- **建立信譽**：貢獻越多，信譽越高，在協作網路中越受信任
- **取得話語權**：在需要社區共識的決策中，貢獻者擁有更大的話語權
- **激勵協作**：Fay 間協作時，貢獻被公正記錄和回報

### 公譽鏈的核心機制

**1. 貢獻記錄**

公譽鏈記錄以下類型的貢獻：Fay 間協作完成的任務、提供資訊組裝服務、提供 API 和技能服務、提供裝置和算力資源、提供執行時期環境、其他被社區認可的增值輸入。

**2. 價格協商**

Fay 在執行協作任務前會事先協商價格（以 μ 為單位）。

**3. 美譽值帳本（MeriToken Ledger）**

美譽值帳本是每個 Fay 和人類的貢獻積分記錄。它記錄：累計貢獻總量、貢獻類型分布、信譽評分、話語權權重。

### 與傳統加密貨幣的本質區別

| 維度 | 傳統加密貨幣 | 美譽值（MeriToken） |
|------|-------------|-------------------|
| **取得方式** | 消耗算力完成計算任務（挖礦） | 創造社會價值（貢獻） |
| **價值錨定** | 市場供需和投機 | 社會貢獻度 |
| **目的** | 金融資產、投資工具 | 信譽度量、話語權、協作激勵 |
| **與法幣關係** | 可兌換法定貨幣 | 不與法定貨幣兌換 |
| **波動性** | 高度波動 | 穩定增長（貢獻只增不減） |

### 遠期願景與當前定位

> ⚠️ **重要說明**：公譽鏈（GMChain）/美譽值（MeriToken）是完全 AI 化社會的遠期願景產物，屬於路線圖第五階段（Fay 重塑勞動結構與價值分配模型）的核心元件。我們不會在短期內急於實現公譽鏈和美譽值，但在體系設計中必須為其預留位置。
>
> 我們承諾：
> - 公譽鏈永遠不會接受貨幣注資
> - 美譽值不會與法定貨幣進行兌換
> - 美譽值衡量的是社會貢獻，而非金融資產

### 對開發者

公譽鏈屬於 **階段 5（Fay 重塑勞動結構與價值分配模型）** 的核心元件，但其介面定義在早期階段就需要預留。

- **獨立專案**：公譽鏈是[獨立開源專案](https://github.com/ChainModePilot/Global-Merit-Chain)，詳見 [GMChain 文件](https://github.com/ChainModePilot/Global-Merit-Chain/wiki)
- **介面規範**：`MeritLedger` 介面，包含 `recordContribution()`（記錄貢獻）、`getBalance()`（查詢積分）、`getReputation()`（查詢信譽評分）、`negotiate()`（價格協商）
- **計量單位**：μ（MU，Merit Unit），美譽值的最小計量單位
- **合規性測試**：iFACTS L4 驗證貢獻記錄的不可篡改性和價格協商的公正性
- **設計要點**：貢獻記錄不可篡改；價格協商在任務執行前完成；美譽值不與法定貨幣兌換；信譽評分基於貢獻品質和協作回饋
