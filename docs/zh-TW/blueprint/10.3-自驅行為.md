# 10.3 自驅行為

## 一句話定義

自驅行為是 iFay 的**自主意志**——模擬操作是手，技能調用是嘴，自驅行為是 iFay 自己決定「什麼時候動手、什麼時候開口」。它讓 iFay 從被動執行者變成主動行動者。

## 為什麼需要它

想像你有一個非常優秀的助手。如果這個助手只會在你下達指令時才行動，那你每件事都得親自想到、親自吩咐——這其實並沒有真正減輕你的負擔，只是把「動手做」變成了「動嘴說」。

真正好的助手是什麼樣的？他會**主動**做事：
- 每週一早上自動幫你整理上週的工作總結（**排程任務**）
- 看到你皺眉頭、嘆氣，主動問你是不是遇到了什麼困難（**察言觀色後行動**）
- 你之前說過「每次收到快遞通知就幫我查一下物流」，他一直記著，每次都自動去查（**持續執行的習慣**）

這就是自驅行為的意義。它讓 iFay 擁有了「自主意志」——不再只是等你開口，而是自己判斷什麼時候該做什麼，然後主動去做。

但這裡有一個重要的安全機制：如果 iFay 自主做的事情結果不對，它會**暫停下來問你**。就像一個好助手，主動幫你做事，但遇到拿不準的情況會先確認，而不是自作主張到底。

## 它在架構中的位置

```
iFay 四層架構
├── 社交層
├── 互動層          ← 自驅行為在這裡
│   ├── 感知（Sense）
│   │   ├── 第一人稱追蹤器   ← 眼睛
│   │   ├── 感測器           ← 神經系統
│   │   └── 自我感知         ← 內心：推斷你的意圖 ──┐
│   └── 動作（Action）                              │
│       ├── 模擬操作         ← 雙手                  │
│       ├── 技能調用         ← 嘴巴                  │
│       └── 👉 自驅行為      ← 自主意志 ←────────────┘
├── 認知層
│   └── 技能
│       ├── 註冊技能         ← 持久技能來源之一
│       └── 內部技能         ← 持久技能來源之二
└── 自我層
```

自驅行為位於**互動層的動作子系統**中，是三種動作方式中最「高級」的一種。模擬操作和技能調用是「怎麼做」，自驅行為是「什麼時候做、要不要做」。

它有一個特別重要的輸入來源：**自我感知**模組。自我感知負責「讀懂你」，自驅行為負責「根據對你的理解主動行動」。兩者的配合，讓 iFay 從一個「你說我做」的工具，變成一個「我懂你、我主動幫你」的夥伴。

## 它是怎麼工作的

自驅行為有**三種觸發方式**，就像人類主動做事的三種原因：

**1. 排程任務——到點了就做**

這是最簡單的觸發方式，就像你設的鬧鐘。你告訴 iFay：「每週一早上 9 點幫我生成上週的工作週報。」iFay 就會在每個週一早上準時執行這個任務。

**2. 自我感知推斷——察覺到你需要，就主動做**

這是最「智能」的觸發方式。iFay 透過自我感知模組觀察你的狀態和環境，推斷出你可能需要什麼，然後主動行動。

比如：iFay 透過感測器資料發現你家冰箱裡的食材快用完了（智慧冰箱的感測器資料），結合你最近的飲食習慣，主動幫你下單補貨。你沒有說過「幫我買菜」，但 iFay 自己判斷出「該買菜了」。

**3. 持久技能——一直在背景運行的習慣**

有些技能不是一次性的，而是持續運行的。比如你讓 iFay「幫我監控這隻股票，跌破 50 塊就提醒我」。這個任務會一直在背景運行，直到條件觸發。

這些持久技能來自兩個地方：**註冊技能**（外部服務提供的持續監控能力）和**內部技能**（iFay 自身的固有能力）。

**行動→回饋→再行動的循環**

不管是哪種觸發方式，自驅行為都遵循一個核心循環：

1. **行動**：iFay 執行一個自主決定的任務
2. **回饋**：觀察執行結果，判斷是否符合預期
3. **再行動**：如果結果符合預期，繼續下一步；如果不符合，調整策略或暫停詢問

這個循環讓 iFay 的自主行為不是「一條路走到黑」，而是不斷自我修正的。

**安全閥：暫停並確認**

當 iFay 發現自主行為的結果和你的意圖不一致時，它會**暫停所有後續自主動作**，然後問你：「我剛才幫你做了 XX，但結果好像不太對，你看要不要繼續？」這個機制確保 iFay 的自主性不會失控。

## 與其他模組的關係

| 相關模組 | 關係 | 人體類比 |
|---------|------|---------|
| **自我感知** | 自我感知推斷人類原型意圖，觸發自驅行為 | 內心感受 → 驅動行動 |
| **模擬操作** | 自驅行為決定「做什麼」，模擬操作負責「用手做」 | 意志 → 手 |
| **技能調用** | 自驅行為決定「做什麼」，技能調用負責「打電話做」 | 意志 → 嘴 |
| **註冊技能** | 持久技能是自驅行為的觸發源之一 | 長期習慣驅動行動 |
| **內部技能** | 內部技能是自驅行為的另一個觸發源 | 本能驅動行動 |
| **對齊意識** | 對齊意識提供人類原型畫像，幫助自驅行為判斷「該不該做」 | 對你的了解決定行動邊界 |

## 場景故事

### 場景一：每週一自動生成工作週報（排程任務觸發）

週一早上 8:55，你還在捷運上。iFay 的排程任務觸發了：

1. iFay 自動從你的個人資料堆中提取上週的工作資料——郵件、日曆、專案管理工具中的任務完成情況、程式碼提交記錄
2. 透過技能調用，呼叫一個文件生成 API，將這些資料整理成一份結構清晰的週報
3. 週報生成後，iFay 檢查內容是否合理（比如有沒有遺漏重要專案）
4. 9:00 整，你到達辦公室，打開電腦，發現 iFay 已經把週報放在你的桌面上了，還附了一句：「上週你在 A 專案上花了最多時間，B 專案有兩個任務延期了，要不要我幫你調整本週計畫？」

你沒有說過「幫我寫週報」——這是 iFay 按照你之前設定的排程任務自動完成的。而且它不只是機械地彙總資料，還主動提出了建議。

### 場景二：主動幫你補貨（自我感知觸發）

週三晚上，iFay 透過你家智慧冰箱的感測器資料發現：牛奶只剩一盒了，雞蛋還有 3 個，蔬菜區幾乎空了。

iFay 結合以下資訊做出判斷：
- 你的飲食習慣（對齊意識中記錄的：你每天早上喝牛奶，每週大約用 12 個雞蛋）
- 你最近的購物記錄（個人資料堆中的資料：上次買菜是 5 天前）
- 當前時間（週三晚上，如果不補貨，週末可能就沒菜了）

iFay 主動行動：
1. 生成一份購物清單：牛奶 2 盒、雞蛋 1 盒、你常買的幾種蔬菜
2. 透過技能調用，呼叫生鮮配送 App 的 API 下單
3. 下單前，iFay 暫停了一下——因為總價超過了你平時的單次購物預算。它問你：「冰箱裡的東西快用完了，我幫你列了一份購物清單，總共 186 元，比你平時多了一點，因為加了你上週說想試試的有機蔬菜。要下單嗎？」
4. 你回覆「好的」，iFay 完成下單

注意最後那個「暫停並確認」的環節——iFay 發現金額超出了你的常規預算（結果與人類原型習慣不完全一致），所以沒有直接下單，而是先問了你。這就是自驅行為的安全機制在起作用。

## 對開發者

自驅行為屬於 **階段 4（iFay + coFay 全面擬人化）** 的核心模組。

- **需求編號**：需求 14（自驅行為）
- **介面規範**：`SelfDrivenBehavior` 介面，包含 `scheduleTask()`（排程任務）、`handleInference()`（處理自我感知推斷）、`pauseAndConfirm()`（暫停確認）和 `getLoopStatus()`（循環狀態）四個核心方法
- **三種觸發源**：`scheduled`（排程任務）、`self_awareness`（自我感知推斷）、`registered_skill` / `internal_skill`（持久技能）
- **核心機制**：動作→回饋→再動作的持續循環；執行結果與人類原型意圖不一致時暫停並請求確認
- **關聯模組**：自我感知（`SelfAwareness`）提供意圖推斷，模擬操作和技能調用負責實際執行，對齊意識（`AlignedConsciousness`）提供行為約束基準
- **合規性測試**：iFACTS L1 驗證三種觸發源的回應能力，L2 驗證與自我感知模組的介面對接，L4 驗證自主行為的安全約束（暫停確認機制）
