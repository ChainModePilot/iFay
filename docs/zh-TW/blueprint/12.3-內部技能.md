# 12.3 內部技能

## 一句話定義

內部技能是 iFay 的**直覺和底線**——註冊技能是 iFay 從外部學來的能力（像考取的證書），內部技能是 iFay 與生俱來的能力（像直覺和道德底線）。它確保外部技能的輸出不會違背人類原型的意圖。

## 為什麼需要它

想像你雇了一個翻譯。這個翻譯技術很好，但他不了解你——他可能把你的商務郵件翻譯得過於隨意，或者在你需要委婉表達時用了太直接的措辭。翻譯的「技能」沒問題，但翻譯的「風格」不是你的風格。

這時候你需要的不是換一個翻譯，而是需要一個**了解你的人**來審核翻譯的結果——「這句話不像我會說的，改一下」、「這個語氣太強硬了，緩和一點」。這個「了解你的人」就是內部技能。

內部技能做三件事：

**第一，建立與人類原型個性對齊的習慣，包括對外部技能的治理。** 就像你的直覺會告訴你「這件事不對勁」，內部技能會對外部技能的輸出進行「直覺檢查」——這個結果符合人類原型的風格嗎？符合人類原型的價值觀嗎？如果不符合，攔截並調整。

**第二，提供內省機制，確保外部知識不與人類原型意圖衝突。** iFay 會從外部知識庫取得大量資訊，但不是所有資訊都適合人類原型。內部技能就像一個「過濾器」，確保外部知識在被 iFay 採納之前，先經過人類原型意圖的校驗。

**第三，嵌入人類原型特定的固有能力。** 如果你是一個金融分析師，你的 iFay 天生就應該具備金融分析能力——這不是從外部註冊來的技能，而是 iFay 與生俱來的「專業素養」。這些能力是在 iFay 初始化時，根據人類原型的職業背景和專業知識嵌入的。

簡單來說：註冊技能是 iFay 的「外功」，內部技能是 iFay 的「內功」。外功可以不斷學習新招式，但內功決定了這些招式怎麼用、用到什麼程度、什麼時候不該用。

## 它在架構中的位置

```
iFay 四層架構
├── 社交層
├── 互動層
│   └── 動作（Action）
│       └── 自驅行為         ← 自驅行為依賴內部技能提供持久能力
├── 認知層          ← 內部技能在這裡
│   ├── 思維（Thought）
│   │   ├── 外部知識        ← 內部技能審核外部知識的輸出
│   │   └── 對齊意識        ← 內部技能的行為基準來自對齊意識
│   └── 技能（Skill）
│       ├── 設備驅動中樞
│       ├── 註冊技能        ← 內部技能治理註冊技能的輸出
│       └── 👉 內部技能      ← iFay 的「直覺和底線」
└── 自我層
    └── Ego 模型            ← Ego 塑造性格，內部技能執行性格約束
```

內部技能位於**認知層的技能子系統**中，和設備驅動中樞、註冊技能並列。但它的角色很特殊——它不是一個「執行者」，更像是一個「守門人」和「內功大師」的結合體。

它向內從**對齊意識**取得人類原型畫像作為行為基準，向外對**註冊技能**和**外部知識**的輸出進行審核和調整。同時，它還為**自驅行為**提供持久的內在能力支撐。

你可以把它想像成一個人的「本能反應」和「職業素養」的結合——本能反應讓你在危險時自動閃避（治理外部輸出），職業素養讓你在專業領域游刃有餘（嵌入的人類原型能力）。

## 它是怎麼工作的

內部技能的工作方式可以用三個關鍵詞概括：**治理、內省、賦能**。

**1. 治理——對外部技能輸出的「直覺檢查」**

每當註冊技能執行完一個任務並回傳結果時，內部技能會對結果進行審核：

- 這個結果的語氣和風格是否符合人類原型的習慣？
- 這個結果是否違背了人類原型的價值觀或偏好？
- 這個結果是否超出了人類原型設定的邊界？

如果發現問題，內部技能會攔截這個輸出，並根據人類原型畫像進行調整。調整後的結果才會被呈現給人類原型或用於後續操作。

這就像你的直覺——當有人給你一個建議時，你的直覺會告訴你「這個建議雖然邏輯上沒問題，但不適合我」。內部技能就是 iFay 的這種直覺。

**2. 內省——確保外部知識不「帶偏」iFay**

iFay 會從外部知識庫取得大量資訊和建議。但外部知識是通用的，不一定適合你。內部技能的內省機制會檢查：

- 這條外部知識是否與人類原型已有的認知衝突？
- 採納這條知識是否會導致 iFay 的行為偏離人類原型意圖？
- 這條知識的來源是否可信？

內省不是拒絕一切外部知識，而是確保外部知識在被 iFay 吸收之前，經過了人類原型意圖的「過濾」。就像一個有主見的人——他會聽取別人的意見，但不會盲目接受。

**3. 賦能——嵌入人類原型的專業能力**

如果你是一個金融分析師，你的 iFay 不需要去「註冊」一個金融分析技能——這個能力是在 iFay 初始化時就嵌入的。它包括：

- 你的專業知識體系（金融模型、市場分析方法等）
- 你的分析風格（保守型還是積極型）
- 你的行業經驗（對特定市場的判斷直覺）

這些能力不是透過外部 API 呼叫的，而是 iFay 內在的一部分。它們讓 iFay 在你的專業領域內，能像你一樣思考和判斷。

## 與其他模組的關係

| 相關模組 | 關係 | 人體類比 |
|---------|------|---------|
| **註冊技能** | 內部技能治理註冊技能的輸出，確保結果符合人類原型風格 | 直覺審核外部建議 |
| **外部知識** | 內部技能的內省機制審核外部知識，防止與人類原型意圖衝突 | 有主見地篩選外部資訊 |
| **對齊意識** | 內部技能的行為基準來自對齊意識維護的人類原型畫像 | 自我認知決定直覺反應 |
| **Ego 模型** | Ego 塑造 iFay 的性格，內部技能在技能層面執行性格約束 | 性格決定行為方式 |
| **自驅行為** | 自驅行為在自主執行任務時，依賴內部技能提供的持久能力 | 自發行動依賴本能和專業素養 |

## 場景故事

### 場景一：翻譯結果的風格校正

你是一家科技公司的 CEO，平時和合作夥伴溝通時習慣用溫和但專業的語氣。今天你讓 iFay 幫你回覆一封英文郵件。

iFay 呼叫了已註冊的翻譯 API 技能，把你的中文草稿翻譯成了英文。翻譯結果在語法和用詞上都沒問題，但語氣偏直接——「We need the report by Friday」（我們需要週五前拿到報告）。

內部技能介入了。它根據對齊意識中你的溝通風格畫像，判斷這句話的語氣不符合你的習慣。你通常會用更委婉的方式表達同樣的意思。於是內部技能將這句話調整為：「Would it be possible to have the report ready by Friday? That would be really helpful for our planning.」（如果能在週五前準備好報告就太好了，這對我們的規劃很有幫助。）

你看到調整後的結果，滿意地點了發送。你甚至沒有意識到 iFay 做了這個調整——因為調整後的結果就是你會說的話。這就是內部技能的價值：它讓 iFay 的輸出**像你**，而不只是**正確**。

### 場景二：金融分析師的內在能力

你是一名資深金融分析師，你的 iFay 在初始化時嵌入了你的專業能力作為內部技能。

某天，你的助理透過 iFay 發來一份市場分析報告，請你審閱。iFay 用內部技能中嵌入的金融分析能力快速掃描了報告：

- 報告中使用的估值模型是 DCF（現金流折現），但你通常偏好結合 DCF 和可比公司分析
- 報告對某個行業的成長預期過於樂觀，與你過去十年的行業經驗判斷不符
- 報告的風險評估部分缺少了地緣政治因素，而你一直認為這是當前市場的關鍵變數

iFay 把這些發現整理成批註，附在報告旁邊呈現給你：「這份報告整體不錯，但有三個地方可能需要你關注……」

注意：iFay 沒有呼叫任何外部金融分析 API——這些判斷完全來自內部技能中嵌入的你的專業知識和分析風格。這就是內部技能和註冊技能的區別：註冊技能是從外面「借來」的能力，內部技能是 iFay「自己」的能力。

## 對開發者

內部技能屬於 **階段 4（iFay + coFay 全面擬人化）** 的核心模組。

- **需求編號**：需求 15（內部技能）
- **介面規範**：`InternalSkill` 介面，包含 `introspect()`（內省檢查）、`intercept()`（攔截並調整外部輸出）和 `getHostCapabilities()`（取得人類原型特定能力）三個核心方法
- **三大職責**：（1）建立與人類原型個性對齊的習慣，治理外部技能輸出；（2）內省機制確保外部知識不與人類原型意圖衝突；（3）嵌入人類原型特定的固有能力（專業技能、專業知識）
- **關聯模組**：註冊技能（`RegisteredSkillManager`）的輸出被內部技能治理，外部知識（`ExternalKnowledge`）的輸出被內部技能審核，對齊意識（`AlignedConsciousness`）提供行為基準
- **與註冊技能的區別**：註冊技能是外部取得的能力（需要註冊和預授權），內部技能是與生俱來的能力（初始化時嵌入，無需註冊）
- **設計要點**：內部技能是「守門人」角色，對外部輸出進行風格和意圖校驗；內省機制防止外部知識「帶偏」iFay；嵌入的人類原型能力是 iFay 的核心競爭力
- **合規性測試**：iFACTS L1 驗證內省和攔截能力，L2 驗證與註冊技能和外部知識的輸出審核介面，L4 驗證外部輸出調整是否真正符合人類原型畫像
