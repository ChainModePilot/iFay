# 6. iFay 是人類人格的複刻

iFay 的個性不是憑空生成的——它複刻（Instantiate）自人類原型（Human Prime，簡稱 Prime）。每一個 iFay 都是其人類原型的人格延伸，而這種延伸需要一個結構化的注入過程。

本頁描述人類原型如何從三個維度將自身特徵注入 iFay：**性格**塑造 Ego 模型、**資料**填充個人資料堆、**權限**建立憑證委託。三個維度缺一不可——性格讓 iFay 知道「我是誰」，資料讓 iFay 擁有「我的過去」，權限讓 iFay 能夠「替我行動」。

---

## 人類原型性格（→ Ego 模型）

人類原型的性格是 Ego 模型的訓練和設定輸入。Ego 模型是 iFay 的內嵌微型模型，它約束 iFay 在價值取向、興趣偏好、習慣、認知邊界、技能邊界、權限邊界和工作風格等維度與人類原型保持對齊。

### 三種注入方式

| 方式 | 說明 |
|------|------|
| **對話敘述** | 人類原型與 iFay 進行引導式對話，iFay 透過標準研究問卷約束對話主題範圍，確保採集到有效的性格資料。這不是閒聊——每一輪對話都圍繞特定的性格維度展開。 |
| **問卷填寫** | 人類原型直接填寫結構化的性格問卷，系統將答案轉化為 Ego 模型的訓練參數。 |
| **直接配置** | 進階使用者可以直接編輯 Ego 模型的約束參數，精確控制 iFay 的行為邊界。 |

### 多版本 Ego

人是多面的。你在工作中嚴謹專注，在朋友面前輕鬆幽默，在家人身邊溫柔耐心。Ego 模型支援多個版本，每個版本對應人類原型人格的一個側面。任一時刻只有一個 Ego 版本處於活躍狀態，支援手動切換和基於上下文的自動切換。

### 場景

> Isabel 剛剛啟動了她的 iFay。系統引導她進入一段對話——不是隨意的聊天，而是圍繞標準研究問卷設計的引導式交流。
>
> 「當你收到一封措辭模糊的工作郵件時，你傾向於直接回覆澄清，還是先私下找對方溝通？」
>
> Isabel 選擇了後者，並補充說她更喜歡在非正式場合解決潛在的誤解。iFay 記錄下這個偏好，將其編碼為 Ego 模型中「溝通風格」維度的一個參數。
>
> 二十分鐘後，Isabel 的 iFay 已經初步理解了她的溝通偏好、決策風格和風險態度。這些特徵將成為 iFay 替她處理日常事務時的行為準則。

## 人類原型資料（→ 個人資料堆）

資料是 iFay 的「記憶」。沒有資料的 iFay 就像一個失憶的人——知道自己是誰，卻不記得自己經歷過什麼。個人資料堆是 iFay 統一管理所有私有資料的元件，人類原型資料是它的初始化輸入。

### 四種資料類型

| 類型 | 說明 | 範例 |
|------|------|------|
| **內容（Content）** | 結構化或非結構化的內容資產 | 照片、影片、文章、創作作品 |
| **資料（Data）** | 個人資料 | 應用使用記錄、健康資料、消費記錄 |
| **知識庫（Knowledge Base）** | 組織化的知識體系 | 專業筆記、學習資料、工作文件 |
| **資訊流（Info Feed）** | 即時資訊訂閱 | 新聞源、產業動態、社群媒體關注 |

### 三種注入方式

1. **授權人類原型資料**：授權 iFay 存取人類原型的照片、應用資料、通訊記錄等。系統與作業系統層緊密整合，支援在系統監督下讀取系統資訊和其他應用的私有資料。
2. **匯入檔案**：匯入文件、筆記、聊天記錄等長期記憶檔案。
3. **指定資訊源**：指定書籍、資訊源和知識庫，讓 iFay 持續從這些來源獲取資訊。

所有資料以標準格式儲存到個人資料堆中，僅可被 iFay 直接存取。這是一次性初始化操作，但支援長期成長——iFay 會隨著人類原型的生活持續累積新的資料。

### 場景

> 陳明決定把過去十年的數位生活交給他的 iFay。
>
> 他首先授權 iFay 存取手機相簿——超過 30,000 張照片和影片，記錄了從大學畢業到孩子出生的每一個重要時刻。然後他匯入了 LINE 聊天記錄、工作郵件存檔和 Notion 筆記庫。
>
> 當 iFay 處理完這些資料後，它不僅知道陳明去年去了哪裡旅行，還理解了他的社交圈、工作節奏和生活習慣。當陳明的同事問起上次團建的餐廳名字時，iFay 能在幾秒內從照片的地理標籤和聊天記錄中找到答案——就像陳明自己回憶起來一樣。

## 人類原型權限（→ 憑證委託）

權限決定了 iFay 能「替你做什麼」。沒有權限的 iFay 就像一個沒有鑰匙的管家——知道該做什麼，卻打不開任何一扇門。

### 三種權限來源

iFay 的權限有三種來源，它們之間沒有包含關係：

| 來源 | 說明 | 範例 |
|------|------|------|
| **人類原型直接授權** | 人類原型將已有憑證委託給 iFay 使用 | 委託 Netflix 帳號、銀行查詢權限 |
| **人類原型權限換取** | 由人類原型的權限交換而來的衍生權限 | 人類原型的企業帳號換取的 API 存取權 |
| **iFay 自主獲得** | iFay 透過自身行為獲得的獨立權限 | 透過 GMChain 貢獻獲得的 API 存取權、在 Fay 社交網路中建立的信任憑證 |

### 無包含關係

一個關鍵設計：人類原型權限與 iFay 權限之間**沒有包含關係**。iFay 可以擁有人類原型沒有的權限（比如 iFay 透過貢獻獲得的 API 存取權），人類原型也可以擁有未委託給 iFay 的權限（比如人類原型的私人銀行帳戶）。

### 七種憑證類型

iFay 支援管理以下七種憑證類型：FayID、帳號密碼、憑證、授權、存取權杖、智慧合約、數位代幣（MeriToken）。人類原型委託憑證時，原始憑證會被交換為對應的副本憑證，iFay 使用副本進行登入和認證，確保原始憑證的安全。

### 場景

> 李薇把她的 Netflix 帳號委託給了 iFay——這是**人類原型直接授權**。iFay 收到的不是李薇的原始密碼，而是一個對應的副本憑證。現在 iFay 可以根據李薇的觀影偏好自動管理她的播放清單。
>
> 與此同時，李薇的 iFay 一直在透過 GMChain 為其他 Fay 提供資訊組裝服務，累積了可觀的 MeriToken。憑藉這些貢獻，iFay 獲得了一個高級天氣資料 API 的存取權——這是**iFay 自主獲得**的權限，李薇本人並不擁有這個 API 的存取權。
>
> 但李薇的私人投資帳戶，她選擇不委託給 iFay。這完全沒問題——人類原型和 iFay 的權限邊界是獨立的。
