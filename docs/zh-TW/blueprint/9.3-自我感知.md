# 9.3 自我感知

## 一句話定義

自我感知是 iFay 的**情商**——它不看螢幕、不感知環境，而是**向內觀察你**，透過你的反應推斷你的感受和意圖，就像一個善於察言觀色的老朋友。

## 為什麼需要它

iFay 的感知子系統有三個模組，各自看向不同的方向：
- **第一人稱追蹤器**向外看——看螢幕上的內容
- **感測器**感知環境——感受溫度、位置、運動
- **自我感知**向內看——觀察**你**

為什麼需要向內看？

想像兩種助手。第一種助手，你說什麼他做什麼，像一台自動販賣機——你按按鈕，它出飲料。第二種助手，他注意到你今天說話語速比平時快、眉頭微微皺著、午餐只吃了一半，於是他主動把下午的會議推遲了半小時，給你倒了杯熱水，把你最喜歡的輕音樂打開了。

第一種是沒有自我感知的 iFay——一個聽命行事的機器人。
第二種是有自我感知的 iFay——一個**懂你**的夥伴。

自我感知讓 iFay 從「你說我做」升級為「你不說我也懂」。它觀察你的微表情、你的操作習慣變化、你的情緒波動，然後推斷出你可能需要什麼——甚至在你自己意識到之前。

這就是自動販賣機和貼心朋友之間的區別。

## 它在架構中的位置

```
iFay 四層架構
├── 社交層
├── 互動層          ← 自我感知在這裡
│   ├── 感知（Sense）
│   │   ├── 第一人稱追蹤器   ← 向外看，看螢幕
│   │   ├── 感測器           ← 感知環境
│   │   └── 👉 自我感知      ← 向內看，讀懂你
│   └── 動作（Action）
│       ├── 模擬操作
│       ├── 技能調用
│       └── 自驅行為         ← 自我感知的推斷會觸發自驅行為
├── 認知層
│   └── 思維
│       └── 對齊意識         ← 自我感知即時調整對齊意識
└── 自我層
```

自我感知位於**互動層的感知子系統**中，是三個感知模組中最「內向」的一個。它不關心螢幕上顯示了什麼（那是第一人稱追蹤器的事），也不關心環境溫度是多少（那是感測器的事）。它只關心一件事：**你現在怎麼樣？你想要什麼？**

自我感知的特殊之處在於，它的輸出會同時流向兩個方向：
1. **向下**傳遞給**自驅行為模組**——觸發 iFay 的主動行動
2. **向內**傳遞給**對齊意識模組**——即時更新 iFay 對你的理解

## 它是怎麼工作的

自我感知的工作方式，就像一個和你相處多年的老朋友讀懂你的過程：

**1. 觀察你的反應**

自我感知會持續監測你與 iFay 互動時的各種訊號：
- 你的操作速度變化（突然操作變快可能是著急，變慢可能是猶豫）
- 你的瀏覽行為（在某段文字上停留很久可能是感興趣，快速滑過可能是不感興趣）
- 你的表情和語氣（如果裝置有攝影機和麥克風）
- 你對 iFay 建議的接受或拒絕模式
- 你的日常習慣是否出現異常

**2. 推斷你的意圖**

基於觀察到的訊號，自我感知會推斷出你當前的狀態和可能的意圖。這不是簡單的「如果 A 則 B」的規則，而是綜合多種訊號的智能推斷。

比如：你在看一篇關於機器學習的文章，捲動速度在某一段明顯變慢，還回捲了兩次——自我感知推斷你對這個段落特別感興趣，可能想深入了解。

**3. 傳遞推斷結果**

當自我感知推斷出一個新的意圖時，它會做兩件事：

- **告訴自驅行為模組**：「人類原型可能對這個話題感興趣，要不要主動找相關資料？」自驅行為模組收到後，可能會自動去搜尋相關文章。
- **告訴對齊意識模組**：「人類原型對機器學習領域表現出興趣，更新一下人類原型畫像。」對齊意識模組收到後，會把「對機器學習感興趣」加入你的個人畫像，讓 iFay 以後的行為更貼合你。

**4. 即時調整**

自我感知不是一次性的判斷，而是持續運行的。它會根據你的最新反應不斷修正自己的推斷。如果它推斷錯了（比如你拒絕了它基於推斷提供的建議），它會學習並調整。

## 與其他模組的關係

| 相關模組 | 關係 | 人體類比 |
|---------|------|---------|
| **自驅行為** | 自我感知的推斷觸發自驅行為的主動行動 | 情商 → 主動關心（「你看起來累了，我幫你……」） |
| **對齊意識** | 自我感知即時調整對齊意識中的人類原型畫像 | 對你的了解隨著相處不斷加深 |
| **認知層** | 推斷結果傳遞給認知層進行更深層的理解和決策 | 直覺 → 理性思考 |
| **第一人稱追蹤器** | 第一人稱追蹤器向外看，自我感知向內看，互補 | 眼睛看世界 vs 心靈讀懂人 |
| **感測器** | 感測器提供的環境資料可以輔助自我感知的推斷 | 身體感覺輔助情緒判斷（比如心跳加速 → 可能緊張） |

## 場景故事

### 場景一：你在讀文章，iFay 主動找資料

週末下午，你在平板上瀏覽一篇關於「可持續建築設計」的長文。自我感知注意到：

- 你在前半部分快速滑動（不太感興趣）
- 但到了「被動式房屋節能技術」這一節，你的捲動速度明顯變慢
- 你還回捲了一次，重新看了一張圖表
- 你在這一段停留了將近兩分鐘

自我感知推斷：你對「被動式房屋節能技術」這個話題特別感興趣。

它把這個推斷傳遞給自驅行為模組。自驅行為模組主動搜尋了幾篇相關的深度文章和一個 YouTube 影片教學，悄悄放在你的閱讀列表裡。當你讀完這篇文章時，iFay 輕輕提示：「我找到了幾篇關於被動式房屋的深度資料，要看看嗎？」

你沒有說過「幫我找資料」，但 iFay 已經做好了。

### 場景二：視訊會議中的貼心助手

你正在參加一個視訊會議。自我感知透過攝影機捕捉到你的面部微表情變化：

- 會議前 20 分鐘，你表情放鬆，偶爾點頭
- 當某位同事開始討論專案預算削減時，你的眉頭微微皺起，嘴角略微下拉
- 你的身體姿勢從放鬆變為前傾，手指開始無意識地敲桌面

自我感知推斷：你對當前討論的話題感到不舒服，可能是不同意預算削減方案，或者這個話題讓你有壓力。

它把這個推斷傳遞給自驅行為模組。自驅行為模組悄悄準備了兩件事：
1. 一份你之前做過的專案預算分析報告（以防你需要用資料反駁）
2. 一條禮貌的藉口訊息草稿（「抱歉，我有個緊急電話需要接」），以防你想暫時離開

這些都靜靜地準備好了，不會打擾你。只有當你需要時，iFay 才會呈現給你。

## 對開發者

自我感知模組屬於 **階段 4（iFay + coFay 全面擬人化）** 的核心模組，是 iFay 從「工具」進化為「夥伴」的關鍵。

- **需求編號**：需求 13（自我感知）
- **介面規範**：`SelfAwareness` 介面，包含 `inferIntent()`（推斷意圖）、`monitorHostReaction()`（監測人類原型反應）和 `adjustAlignment()`（調整對齊意識）三個核心方法
- **關聯模組**：自驅行為模組（`SelfDrivenBehavior`）接收推斷結果並觸發主動行動；對齊意識模組（`AlignedConsciousness`）接收推斷結果並更新人類原型畫像
- **關聯協議**：無直接協議依賴，但可利用感測器模組（透過 CAP/DTP）取得輔助資料（如心率、面部表情）
- **合規性測試**：iFACTS L1 驗證意圖推斷能力，L2 驗證與自驅行為和對齊意識的介面對接，L4 驗證推斷準確性和隱私保護
- **設計要點**：推斷結果必須同時傳遞給自驅行為模組和認知層；支持即時調整對齊意識；推斷錯誤時應能從人類原型回饋中學習修正
