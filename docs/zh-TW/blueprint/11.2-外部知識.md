# 11.2 外部知識

## 一句話定義

外部知識是 iFay 的**圖書館和專家顧問**——個人資料堆是 iFay 自己的記憶，外部知識是 iFay 可以查閱的圖書館和諮詢的專家。它讓 iFay 擁有超越人類原型自身能力的知識。

## 為什麼需要它

你的記憶再好，也不可能記住所有事情。當你遇到不懂的問題時，你會怎麼做？去圖書館查資料，或者諮詢專業人士。

比如你收到一份體檢報告，上面寫著「三酸甘油酯 2.8 mmol/L」。你知道這是什麼意思嗎？大多數人不知道。這時候你需要的不是「回憶」（個人資料堆），而是「查閱」——去查醫學知識庫，或者問一個醫生。

iFay 也是如此。個人資料堆儲存的是你自己的資料和記憶，但你的知識是有限的。外部知識模組讓 iFay 能夠**走出自己的記憶**，去查閱外部的知識庫、諮詢外部的智能模型——就像一個人走進圖書館或撥通專家的電話。

這讓 iFay 的能力不再局限於「你知道什麼」，而是擴展到「世界上有什麼知識可以幫到你」。

## 它在架構中的位置

```
iFay 四層架構
├── 社交層
├── 互動層
│   └── ...
├── 認知層          ← 外部知識在這裡
│   ├── 思維（Thought）
│   │   ├── 個人資料堆      ← 自己的記憶
│   │   ├── 👉 外部知識      ← 圖書館和專家顧問
│   │   └── 對齊意識         ← 對你的理解
│   └── 技能（Skill）
│       ├── 註冊技能         ← 外部知識源也是一種「技能」
│       └── ...
└── 自我層
```

外部知識位於**認知層的思維子系統**中，和個人資料堆、對齊意識並列。它的獨特之處在於：外部知識源在 iFay 的體系中被視為一種**技能類型**——就像翻譯 API 是一種技能一樣，醫學知識庫也是一種技能。這意味著外部知識的接入遵循和其他技能相同的註冊、授權和呼叫流程。

外部知識和個人資料堆的關係就像「圖書館」和「個人筆記本」：你從圖書館借來的書讀完後，重要的內容會被你記在筆記本裡。同樣，iFay 從外部知識源取得的資訊，會被整合到個人資料堆中，變成 iFay 自己的「記憶」。

## 它是怎麼工作的

外部知識的工作方式可以用三個關鍵詞概括：**接入、查詢、整合**。

**1. 接入——連接外部知識源**

外部知識源的種類很多：
- **知識庫**：醫學知識庫、法律知識庫、百科全書
- **大型語言模型**：GPT、Claude 等通用 AI 模型
- **專家系統**：特定領域的專業推理系統
- **搜尋引擎**：網際網路上的公開資訊

這些知識源透過 **SSP（技能共享協議）** 接入 iFay。SSP 是一個標準化的協議，讓原本只對特定用戶端開放的服務和介面向全網開放——iFay 可以像存取任何其他技能一樣存取這些知識源。

**2. 查詢——像諮詢專家一樣提問**

當 iFay 需要外部知識時，它會向對應的知識源發送查詢請求。每個查詢結果都帶有**置信度**——告訴 iFay 這個答案有多可靠。iFay 可以綜合多個知識源的回答，選擇最可靠的資訊。

**3. 整合——學到的知識變成自己的**

從外部知識源取得的資訊不會只是「用完就扔」。重要的知識會被整合到個人資料堆中，成為 iFay 自己的記憶。下次遇到類似問題時，iFay 可以直接從自己的記憶中找到答案，不需要再次查詢外部知識源。

**如果知識源不可用怎麼辦？**

網路可能斷開，知識源可能下線。這時候，外部知識模組會**降級使用個人資料堆中的快取知識**。就像圖書館關門了，你還可以翻翻自己之前做的筆記。同時，iFay 會通知認知層「當前使用的是快取知識，可能不是最新的」。

## 與其他模組的關係

| 相關模組 | 關係 | 人體類比 |
|---------|------|---------|
| **個人資料堆** | 外部知識取得的資訊整合到個人資料堆中 | 從圖書館學到的知識 → 記在筆記本裡 |
| **註冊技能** | 外部知識源作為技能類型註冊和管理 | 圖書館的借書證也是一種「技能憑證」 |
| **對齊意識** | 對齊意識幫助篩選與人類原型相關的外部知識 | 你的興趣決定你去圖書館借什麼書 |
| **內部技能** | 內部技能檢查外部知識是否與人類原型意圖衝突 | 用自己的判斷力評估外部資訊的可信度 |
| **SSP 協議** | 外部知識源透過 SSP 協議標準化接入 | 圖書館的統一借閱規則 |

## 場景故事

### 場景一：幫你看懂體檢報告

你收到了年度體檢報告，裡面有一堆看不懂的指標：三酸甘油酯 2.8 mmol/L、低密度脂蛋白 3.9 mmol/L、尿酸 480 μmol/L……

你對 iFay 說：「幫我看看這份體檢報告，有沒有什麼需要注意的？」

iFay 開始工作：
1. 先從個人資料堆中調取你過去三年的體檢資料（自己的記憶）
2. 然後查詢外部的**醫學知識庫**（圖書館）：每個指標的正常範圍是多少？超標意味著什麼？
3. 綜合分析後告訴你：

「你的三酸甘油酯偏高（正常值應低於 1.7，你是 2.8），低密度脂蛋白也偏高（正常值應低於 3.4，你是 3.9）。這兩項指標和去年相比都有上升。結合你最近三個月的飲食記錄（個人資料堆中的資料），你外送點餐的頻率明顯增加了，而且偏好高油高鹽的食物。」

「建議你關注血脂健康，考慮調整飲食習慣。要不要我幫你預約一個心臟內科的門診？」

在這個場景中，iFay 結合了**自己的記憶**（你的歷史體檢資料和飲食記錄）和**外部知識**（醫學知識庫中的指標解讀），給出了個性化的健康建議。這就是外部知識的價值——讓 iFay 擁有你不具備的專業知識。

### 場景二：幫你起草合約條款

你是一個小企業主，需要和供應商簽一份採購合約。你對合約法不太熟悉，但又不想每次都花錢請律師。

你對 iFay 說：「幫我起草一個採購合約的違約條款，供應商如果延遲交貨超過 15 天，我們有權解除合約。」

iFay 開始工作：
1. 查詢外部的**法律知識庫**：關於採購合約違約條款的法律規定和常見寫法
2. 參考個人資料堆中你之前簽過的合約範本（自己的記憶）
3. 綜合生成一段違約條款草稿，包含法律術語和保護性條款

iFay 給你呈現草稿：「這是根據《民法》相關規定起草的違約條款。我參考了法律知識庫中的標準範本，並結合了你之前和 XX 公司簽的那份合約的風格。你看看是否需要調整？」

同時 iFay 提醒你：「這是基於法律知識庫生成的草稿，建議在正式簽署前讓專業律師審核。」

這個場景展示了外部知識的另一個重要用途：讓 iFay 在你不擅長的領域也能提供專業級的輔助。iFay 不是律師，但它可以查閱法律知識庫，就像你去圖書館查法律書籍一樣。

## 對開發者

外部知識屬於 **階段 3（iFay 作為虛擬世界的介面）** 的核心模組。

- **需求編號**：需求 12（外部知識接入）
- **介面規範**：`ExternalKnowledge` 介面，包含 `query()`（查詢外部知識）、`registerSource()`（註冊知識源）和 `fallbackToCache()`（降級到快取）三個核心方法
- **知識源類型**：`knowledge_base`（知識庫）、`llm`（大型語言模型）、`expert_system`（專家系統）、`search_engine`（搜尋引擎）
- **關聯協議**：SSP（技能共享協議）用於標準化接入外部知識源
- **關聯模組**：個人資料堆（`PersonalDataHeap`）整合外部知識，註冊技能（`RegisteredSkillManager`）管理知識源註冊，內部技能（`InternalSkill`）檢查知識衝突
- **降級策略**：外部知識源不可用時，自動降級使用個人資料堆中的快取知識，並通知認知層
- **合規性測試**：iFACTS L1 驗證知識查詢和快取降級能力，L2 驗證與個人資料堆的知識整合介面，L3 驗證「知識查詢→整合→快取→降級」的完整鏈路
