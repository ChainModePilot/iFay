# 3. 設計原則

iFay（Individual Fay）是與自然人深度綁定的 AI 數位化身。它不是工具，而是你的人格延伸——融合你的性格、記憶、偏好與數位能力，代替你執行機械性、重複性和危險性勞動，放大你的社會價值。iFay 採用 CPE+M 框架（Context 上下文、Protocol 協議、Environment 環境、Merit 貢獻度量），由社交層、互動層、認知層和自我層四層架構組成，覆蓋從模擬人類操作到重塑勞動結構與價值分配的五個演進階段。

---

## 設計原則

iFay 的設計遵循五項核心原則，它們貫穿整個系統的架構與實作：

| # | 原則 | 一句話說明 |
|---|------|-----------|
| 1 | **生態友善的漸進式採納** | 生態夥伴無需實作完整 iFay 即可發布產品——一個只控制無人機的 iFay，只需滿足所需部件子集即可上線。 |
| 2 | **宣告式極簡組裝** | 透過一個 FayManifest 宣告檔案定義所需部件、協議和配置，像寫 package.json 一樣組裝 iFay。 |
| 3 | **靈活的部件組合** | 部件鬆耦合、可混搭，不同廠商的實作只要符合介面契約即可自由組合。 |
| 4 | **人格化而非工具化** | iFay 不是 Agent——每個 iFay 都有獨特的個性、記憶和偏好，是人類原型的複刻，甚至可在人類原型過世後延續人格。 |
| 5 | **場景驅動的直覺設計** | 每個功能模組都能用一個具體的生活場景來解釋，讓讀者直觀想像有了 iFay 後的生活。 |

5. [FayManifest 宣告式組裝](./13-FayManifest聲明式組裝)

    FayManifest 是 iFay 的「package.json」——開發者只需在一個 JSON 宣告檔案中列出所需的部件、協議、控制模式和驅動配置，FayGer 執行時期自動解析依賴並組裝 iFay 實例。一個週末就能從零搭建一個專用 iFay。

---

6. [iFACTS 合規性驗證](./14-iFACTS合規性驗證)

    iFACTS（iFay Architecture Conformance Test Suite）是標準化合規性測試套件，覆蓋協議互動、模組介面、安全行為等關鍵規範點。廠商實作必須通過 iFACTS 測試才能聲稱「iFay 可用」，測試分為 L1 單部件合規、L2 介面合規、L3 整合合規、L4 行為合規四個層級。

---

7. [監護與人格延續](./15-監護與人格延續)

    iFay 是人格的數位載體。當人類原型無法管理 iFay 時，監護人可透過助記詞或預指定身分認證接管管理權；當人類原型過世後，iFay 可在數位墓園沙箱中以獨立身份繼續存在，人格不滅。
