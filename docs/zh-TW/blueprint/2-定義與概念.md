# 2. 定義與概念

## 產品專有名詞

### iFay
Individual Fay。附屬於自然人（稱為**人類原型，Human Prime，簡稱 Prime**）的數位化身，致力於在價值觀、個性和習慣方面與其所附屬的自然人（人類原型）保持對齊。iFay 是人類原型人格的複刻（Instantiate），融合了人類原型的人格特徵與數位能力，旨在代替人類原型執行機械性、重複性、危險性勞動和繁瑣輔助工作，提升人類原型的安全保障、健康和生活品質，並放大人類原型的社會價值。

### coFay
Common Fay，承擔公共角色的數位化身，大致等同於當前 Agent 的概念。它在某一領域擁有通用知識和技能，可以透過對話為多人服務。coFay 是獨立專案，iFay 規範僅在必要時引用。

### Fay
iFay 和 coFay 的統稱，擬人化 AI 代理的通用術語。它可能是新型社會關係中的重要節點。

### FayID
分配給每個 Fay 的全域唯一持久識別碼，人類可識別且機器可讀。FayID 對 iFay 和 coFay 使用統一的識別碼生成和管理機制，支援超過全球人口數量級的識別碼容量。

### FayGer
為 Fay（iFay 和 coFay）提供虛擬執行時期環境的容器，類似 Docker。這確保了 Fay 可以跨平台附身（Inhabit）於不同的硬軟體，使硬軟體具備人類原型的個人意志。FayGer 提供跨語言建構工具和跨平台執行容器，使任何 Fay（無論開發語言）都能被打包並執行。

### Ego 模型
iFay 的個性化微型模型，與特定個人的畫像對齊。Ego 是可插拔、可切換的邊緣小模型，可載入到任何終端裝置上使該裝置擁有人類原型個性。Ego 約束 iFay 在價值取向、興趣偏好、習慣、認知邊界、技能邊界、權限邊界、工作風格等維度與人類原型對齊，獨立於外部大型模型運行，確保個性穩定性。支援多版本管理與人格切換，以適應人類原型在不同場景下的多面人格需求。

> ⚠️ **倫理約束**：Ego 版本切換必須遵循透明性原則。所有 Ego 版本必須共享同一套核心價值觀，差異僅限於表達風格和互動偏好。切換時應在互動元資料中標註當前活躍的 Ego 版本標識，確保可稽核。

### iFay Profile
iFay 的統一資料模型，語意可解釋的屬性表。用於人類和系統識別 iFay，以及 Fay 間相互識別，是 iFay 的完整「身分證」。包含六個維度：iFay 身份（FayID + 元資訊）、Ego 模型（當前活躍 Ego 版本引用）、Faying 思維、Faying 技能、Faying 硬體、Faying 權限。人類原型畫像（對齊意識）是 Profile 的子集和輸入源。

### FayManifest
iFay 的宣告式組裝設定檔，類似 package.json。宣告 iFay 實作所需的部件、協議、控制模式（指令控制/Ego 控制/自主控制）和配置。開發者只需宣告「需要什麼」而非「如何實作」，支援最小化宣告——只宣告一個部件子集即可上線，系統自動補充必需的基礎設施依賴。

### GMChain
**全球貢獻鏈**（Global Merit Chain），一種透過衡量可識別社會貢獻來獲取數位代幣的區塊鏈。與大多數區塊鏈專案不同，該專案完全依賴社會價值的認定來獲取代幣，而非透過完成專案內預定義的、無爭議的計算任務。同時，為了與現實世界的價值認定方式對接，它也接受以法定貨幣、債券、資產證書、黃金等抵押的定向發行方式。

### MeriToken
數位代幣，一種對應社會貢獻度的量化數位憑證。獲取方式基於創造社會價值，而非透過消耗算力完成區塊鏈技術工作。

### μ（MU）
Merit Unit，MeriToken 的計量單位。

### iFACTS
iFay Architecture Conformance Test Suite，標準化合規性測試套件。覆蓋協議互動、模組介面、安全行為等關鍵規範點，包含四個測試層級：L1 單部件合規、L2 介面合規、L3 整合合規、L4 行為合規。廠商實作必須通過 iFACTS 測試才能聲稱其產品「iFay 可用」。

### iFay Ready
應用產品被 iFay 操控所需滿足的認證標準，分為三個等級：
- **Bronze**：支援 iFay 透過模擬操作（第一人稱追蹤器 + 模擬操作）操控應用
- **Silver**：支援 iFay 透過 CAP 協議直接控制應用，支援 DTP 協議資料交換
- **Gold**：支援 iFay 透過 SSP 協議共享技能，支援完整的 C/F/S 架構整合

### 監護人（Guardian）
人類原型過世或無法管理 iFay 時，被指定接管 iFay 管理權的自然人。注意：不使用「繼承人」一詞。監護人透過助記詞啟動或人類原型預先指定的身分認證方式完成身分驗證。

### 監護（Guardianship）
人類原型將 iFay 的管理權轉移給監護人的過程和狀態。監護權轉移完成後，iFay 與新的監護人綁定，並更新 Faying 協議的配對關係。

### 數位墓園（Digital Cemetery）
人類原型過世後，iFay 以獨立身份繼續運行的專屬沙箱環境。數位墓園限制 iFay 的行為範圍，防止 iFay 延續執行人類原型預設任務而造成混亂和損失，確保安全性和隔離性。

## 通用概念澄清

### 上下文（Context）
Fay 所處的外部環境，包括通訊對象、互動模式、傳達的訊息、訊息表達的含義、可控的硬軟體、可呼叫的資源和技能等。

### 協議（Protocol）
包括資料儲存、訊息體、參數和介面結構的註解定義，以及針對特定目的的互動流程約定。

### 憑證（Credentials）
廣義概念，指用於唯一標識人、事、物的任何數位識別碼，不可篡改或否認。包括 FayID、帳號密碼、憑證、授權、存取權杖、智慧合約和數位代幣（MeriToken）。

### 人類原型（Human Prime）
iFay 所複刻的自然人使用者。iFay 與人類原型深度綁定，複刻人類原型的人格特徵並融合數位能力。

### 複刻（Instantiate）
從人類原型生成與其對齊的 iFay 的過程。複刻不是簡單的資料複製，而是將人類原型的性格、資料和權限結構化注入 iFay 的過程。

### 附身（Inhabit）
將 iFay 注入到某個終端硬體或軟體應用上，以便 iFay 可以控制該終端的過程。附身後，終端成為 iFay 的「肢體」，iFay 可以透過 CAP 協議直接控制終端的硬體和軟體功能。

### 宿主（Host）
被 iFay 附身（Inhabit）並控制的終端硬體或軟體應用。注意：宿主指的是終端裝置或應用，而非人類原型。

### Faying
iFay 與人類原型或終端建立連線的過程和狀態，類似藍牙配對。當 iFay 與人類原型處於 Faying 狀態時，iFay 保持啟動。Faying 需遵循 Faying 協議規定的安全配對條件，確保 iFay 僅在人類原型明確意圖下被授權運行。

### Separating
iFay 與人類原型斷開連線進入休眠的過程。當 iFay 與人類原型處於 Separating 狀態時，iFay 切換為休眠狀態，儲存當前狀態快照並完成分離。

## 模組名稱

iFay 採用四層架構，共包含 12 個核心模組：

### 社交層（Social Layer）

| 模組 | 說明 |
|------|------|
| **憑證管理（Credentials Management）** | 管理七種憑證類型（FayID、帳號密碼、憑證、授權、存取權杖、智慧合約、數位代幣），支援人類原型憑證的安全委託與副本機制 |

### 互動層 — 感知（Interaction Layer — Perception）

| 模組 | 說明 |
|------|------|
| **第一人稱追蹤器（First-person Tracer）** | 模擬人類原型第一人稱視角，擷取螢幕或介面上的視覺和聽覺資訊，優先使用視覺感知而非解析結構化文件 |
| **感測器（Sensor）** | iFay 的廣義神經系統，基於 CAP 和 DTP 協議橋接終端裝置感測器，支援動態調節靈敏度 |
| **自我感知（Self-awareness）** | 向內監測人類原型反應以推斷意圖的模組，類似熟練助手解讀老闆面部表情的能力 |

### 互動層 — 動作（Interaction Layer — Action）

| 模組 | 說明 |
|------|------|
| **模擬操作（Simulated Operation）** | 模擬人類 UI 互動操作（點擊、拖曳、捲動、手勢等），透過第一人稱追蹤器回饋自適應探索介面 |
| **技能調用（Invoke Skill）** | 直接觸發已註冊技能或執行特定任務的動作模組，類似函式呼叫或 API 呼叫 |
| **自驅行為（Self-driven Behavior）** | 自主觸發的行為模組，支援排程任務、自我感知推斷和持久技能三種觸發源，形成「動作→回饋→再動作」循環 |

### 認知層 — 思維（Cognition Layer — Thinking）

| 模組 | 說明 |
|------|------|
| **個人資料堆（Personal Data Heap）** | 統一管理 iFay 所有私有資料，支援多種儲存格式和位置，向內部模組提供統一讀寫介面 |
| **外部知識（External Knowledge）** | 外部知識庫和模型的接入模組，將外部智慧作為技能類型接入，透過 SSP 協議存取開放服務 |
| **對齊意識（Aligned Consciousness）** | 人類原型個人畫像的完整描述，支援從資料探勘、自我感知即時調整和人類原型手動定義三種方式建立和更新 |

### 認知層 — 技能（Cognition Layer — Skills）

| 模組 | 說明 |
|------|------|
| **設備驅動中樞（Device Driver Hub）** | 設備驅動的中樞層，透過標準化介面整合新驅動，確保新驅動整合時內部架構穩定 |
| **註冊技能（Registered Skill）** | iFay 已掌握或獲得使用能力的技能，支援六種技能類型（API、Workflow、Bot、Agent、APP、Microservice） |
| **內部技能（Internal Skill）** | 與人類原型個性對齊的固有能力模組，提供內省機制，確保外部知識不與人類原型意圖衝突 |

### 自我層（Ego Layer）

| 模組 | 說明 |
|------|------|
| **Ego 模型** | iFay 的個性化微型模型，可插拔、可切換的邊緣小模型，獨立於外部大型模型運行，確保個性穩定性 |

## 協議名稱

iFay 協議體系包含六種核心協議：

| 協議 | 說明 |
|------|------|
| **Faying 協議** | 規定自然人與 iFay 安全配對及啟動條件的協議，確保 iFay 僅在人類原型明確意圖下被授權運行 |
| **互動對話協議（Interactive Conversation Protocol）** | 面向人類 UI 的模組化多模態語意協議，將語意內容模組化和多模態化，使用戶端能重構易讀的使用者友善訊息展示 |
| **心靈感應協議（Telepathy Protocol）** | Fay 間去除 UI 翻譯層的語意通訊協議（技術上即語意向量直接通訊，Semantic Direct Protocol），允許意義和意圖在 Fay 之間直接傳輸，使用約定的向量編碼權杖替代結構化文字 |
| **控制權限協議（CAP）** | Control Authority Protocol，用於 iFay 接管終端硬體和特定軟體，直接呼叫驅動程式、本機介面和命令 |
| **資料隧道協議（DTP）** | Data Tunnel Protocol，終端與 iFay 間的雙向傳輸協議。終端→iFay 方向支援持久化使用者資料儲存和資料監護；iFay→終端方向支援資料豐富化和個性化處理 |
| **技能共享協議（SSP）** | Skill Sharing Protocol，將原本僅對用戶端開放的服務和介面透過標準化遠端協議向全網開放 |

## iFay Profile 維度

iFay Profile 包含六個維度，其中四個 Faying 維度描述 iFay 的能力和資源：

### iFay 身份
FayID + 元資訊，iFay 的基礎身份標識。

### Ego 模型
當前活躍 Ego 版本引用，任一時刻僅有一個活躍版本。

### Faying 思維
iFay 的認知資源維度，包含四個子類型：
- **Content**（內容）— 結構化和非結構化的內容資產
- **Data**（資料）— 人類原型的個人資料
- **Knowledge Base**（知識庫）— 經過組織的知識體系
- **Info Feed**（資訊流）— 即時資訊訂閱與推送

### Faying 技能
iFay 的能力維度，包含六種技能類型：
- **API** — 標準化介面呼叫
- **Workflow** — 工作流編排
- **Bot** — 對話機器人
- **Agent** — 智慧代理
- **APP** — 應用程式
- **Microservice** — 微服務

### Faying 硬體
iFay 的硬體維度，是 iFay 的「肢體」。iFay 遷移時終端掃描並嘗試連線可用硬體。包含三個子類型：
- **Device**（裝置）— 終端硬體裝置
- **Storage**（儲存）— 資料儲存資源
- **Computing**（算力）— 計算資源

### Faying 權限
iFay 的權限維度，是 iFay 的「關係」。支援 SSO、OAuth、Fingerprint（統稱，指一切可識別身份的標識）等可擴充鑑權方式。權限生命週期分為三種類型：與生俱來（inherent）、持久（persistent）、臨時（ephemeral）。
